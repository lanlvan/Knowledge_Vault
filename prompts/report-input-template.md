# Report Input Template

## 1. Purpose

用于在生成日报、周报、月报、阶段汇报或会议准备材料前，先收集本次输出所需的最小输入信息。

本模板不承载业务事实，只用于辅助用户整理输入，供任务型 prompt 读取和使用。

## 2. Required Input

```text
输出类型：
日报 / 周报 / 月报 / 阶段汇报 / 会议准备 / 其他

时间范围：

重点主题：

需要特别关注的项目：

需要特别关注的个人工作判断：

本周期新增动作：

本周期已完成事项：

当前推进中事项：

待确认问题：

风险或卡点：

需要同步或向上反馈的内容：

需要规避的表达：
```

## 3. Optional Input

```text
关联文件：

关联项目：

关联 decisions：

关联 pending：

是否需要读取 source：

是否有 expired / superseded / uncertain 材料：

是否存在 correction：
```

## 4. Usage Rules

- 如果缺少“输出类型 / 时间范围 / 重点主题”，不应直接生成正式正文。
- 如果缺少“本周期新增动作”，应在输出中标明材料不足。
- 如果待确认问题未形成结论，不得写成已完成。
- 如果材料来自历史记录，应先判断 active / expired / superseded / uncertain。
- 如果用户无法提供完整输入，可以先生成“输入缺口清单”，而不是生成正式汇报。
- 本模板只作为输入整理工具，不作为事实源。
