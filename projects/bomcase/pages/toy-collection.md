# Toy Collection Page Brief

## 1. 来源

* 当前 source：`../sources/toy-collection-handoff.md`
* 如需核对事实，以对应 source 为准；
* 本文件是工作摘要，不替代 source；
* README/index 不作为事实源。

## 2. 页面目标

潮玩图鉴页用于展示收藏进度、SSR 统计、系列卡组与单卡状态信息。  
按当前 source，页面目标聚焦：

* 展示全站收藏进度与 SSR 顶级单卡统计；
* 支持分类、系列、单卡拥有状态筛选；
* 展示系列卡组与卡组内单卡状态；
* 展示隐藏款与全站唯一 SSR 标签信息；
* 在无卡组数据时提供空状态承接。

## 3. 页面结构

* 顶部导航区（潮玩图鉴高亮）；
* 顶部全站统计区（收藏进度卡 + SSR 统计卡）；
* 分类 Tab 与筛选区（分类/系列/拥有状态）；
* 系列卡组列表区（支持横向滚动与懒加载）；
* 集齐兑换区域（当前未开放）；
* 单卡列表区域（已拥有/未拥有/隐藏款/全站唯一 SSR）；
* 空状态区域（当前系列暂无卡组）。

## 4. 核心表达

图鉴页核心表达是“收藏可视化 + 筛选浏览 + 单卡状态识别”：

* 顶部统计负责全站层面的收藏与 SSR 概览；
* 页面主体负责系列卡组与单卡状态浏览；
* 筛选用于快速定位分类/系列/拥有状态；
* 未开放能力（如兑换）明确保持占位说明，不提前扩展规则。

本节仅保留页面表达，不扩展为完整业务规则。

## 5. 关键页面判断

基于当前 source 可沉淀为：

* 顶部 SSR 统计不随分类/系列/拥有状态筛选变化；
* 分类数量口径是“该分类下 SSR 顶级单卡数量”；
* 拥有状态筛选对象是单卡，不是卡组；
* 收藏完成度分子为已解锁卡牌数量，分母为全部卡牌数量；
* 收藏完成度展示为整数百分比，整数取整规则仍待确认；
* SSR 统计刷新频率跟后端缓存策略一致；
* SSR 统计接口异常时展示 0，不提供重试入口；
* 当前页面验收以产品规则为准，图片仅作为示意图；
* 分类会影响系列筛选结果；
* 筛选只影响下方卡牌/卡组列表，不影响顶部统计；
* 多个筛选条件同时存在时按 AND 关系计算；
* 卡组默认顺序以后端返回顺序为准；
* 卡组列表无收起状态，全部显示；
* 同系列多卡组按配置顺序展示；
* 加载失败时展示空态；
* 无筛选结果时展示无结果文案，并提供重置筛选入口；
* 异常与无结果场景的具体最小文案由后续 UI / 文案阶段处理；
* 单卡数量格式为 xN；
* 单卡状态枚举包含已解锁、未解锁、拥有、隐藏；
* 隐藏款未解锁时不展示真实名称，展示“隐藏款”；
* 隐藏款未解锁时图片展示剪影 / 占位图；
* 上述隐藏款展示规则适用于后续同类隐藏款；
* 单卡点击进入已有功能弹窗，弹窗细节不在本页展开；
* 集齐兑换区当前为未开放且不可点击；
* 卡组为空时展示“去商城看看”承接入口。
