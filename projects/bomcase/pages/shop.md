# Shop Page Brief

## 1. 来源

* 当前 source：`../sources/shop-handoff.md`
* 如需核对事实，以对应 source 为准；
* 本文件是工作摘要，不替代 source；
* README/index 不作为事实源。

## 2. 页面目标

商城页用于展示商品列表与对应军需权益，并承接商品详情入口与已有下单入口。  
按当前 source，页面目标聚焦：

* 展示商品信息、权益信息与基础筛选信息；
* 支持权益选中切换（在可切换权益类型下）；
* 承接“商品浏览 -> 详情/下单”的入口路径；
* 表达“买商品 · 赠军需权益”的页面语义。

## 3. 页面结构

* 顶部导航（已有功能范围）；
* 顶部说明横幅（主副文案 + 查看规则入口）；
* 分类 / 排序 / 搜索区域；
* 商品列表区域（最多双排）；
* 商品卡片区域（主图、标题、标签、价格、销量、权益区、CTA）；
* 底部信任保障区域。

## 4. 核心表达

商城页核心表达是“商品为主体、权益为附加承接”：

* 商品卡片承接浏览与购买入口；
* 权益展示用于说明下单可获军需权益；
* 固定权益与 3 选 1 权益均保持选中态语义；
* 页面强调“买商品赠权益”，不表达为“直接购买军需”。

本节仅保留页面表达，不扩展为完整业务规则。

## 5. 关键页面判断

基于当前 source 可沉淀为：

* 商品列表展示边界明确：最多双排，不滚动、不分页；
* 单商品 Tag 仅展示一个；
* 商品主图区域可由商品主图 + 当前选中权益图组合表达；
* 商品图/标题进入详情，CTA 进入已有下单流程；
* 分类切换会联动列表并恢复搜索、排序默认；
* 规则弹窗、搜索、下单流程、导航等均属于已有能力边界。

## 6. 待确认项

暂无明确待确认项（以当前 source 为准）。

## 7. 与其它页面 / 流程关系

* 商城页通过商品图/标题承接到商品详情页；
* 商城页通过 CTA 承接到已有下单流程；
* 商品权益展示与后续军需相关玩法形成表达衔接；
* “查看规则”承接已有规则弹窗能力。

本节只描述入口关系，不定义支付、库存、订单、发货、资产、权益结算规则。
