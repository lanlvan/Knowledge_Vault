# Open Box Result Page Brief

## 1. 来源

* 当前 source：`../sources/open-box-result-handoff.md`
* 如需核对事实，以对应 source 为准；
* 本文件是工作摘要，不替代 source；
* README/index 不作为事实源。

## 2. 页面目标

结果页用于在本次开启结算完成后，展示正常掉落结果，并补充关键反馈信息。  
按当前 source，页面目标聚焦：

* 正常掉落饰品展示（主内容）；
* 顶部保底状态反馈；
* 触发保底时的保底奖励区；
* 有沉淀时的水晶能量提示；
* 底部继续操作入口承接（已有能力）。

## 3. 页面结构

* 顶部标题区（含顶部保底反馈条件展示）；
* 正常掉落饰品区（核心展示区）；
* 保底奖励区（条件展示，不占正常掉落位）；
* 底部按钮区（返回/再来）；
* 水晶能量沉淀提示（低权重条件提示）；
* 关闭按钮（已有功能范围）。

## 4. 核心表达

结果页核心表达是“正常掉落为主，保底与沉淀为补充反馈”：

* 正常掉落饰品始终是主视觉主体；
* 保底奖励是额外反馈，不混入正常掉落卡片；
* 顶部保底文案只做状态信号，不承担奖励详情；
* 水晶提示只表达本次沉淀结果，不展开资源总览。

本节仅保留页面表达，不扩展为完整业务规则。

## 5. 关键页面判断

基于当前 source 可沉淀为：

* 保底奖励不作为第 6 张正常掉落卡片；
* 保底奖励区仅在触发保底并额外发放奖励时展示；
* 保底奖励区使用统一军需箱 icon，不按奖励名切换专属视觉；
* 触发保底并发放额外奖励时，结果页必须让用户感知保底奖励已到账；
* 上述到账感知只约束结果表达目标，具体视觉 / UI 表现由对应方案承接，不在本 brief 中固化为动效、样式或单一实现；
* 顶部保底反馈仅在特定状态展示，普通进度更新不默认展示；
* 水晶提示与保底状态独立判断；
* 结果页展示以后端最终结算结果为准。
