# Open Box Page Brief

## 1. Source Policy

* 本文件是 Open Box 当前页面事实压缩视图。
* `active_source` 是当前 Open Box 的唯一详细事实载体。
* Delivery Output 生成时必须读取 page brief、active source、`projects/bomcase/pending.md` 与 `projects/bomcase/decisions.md`。
* 历史 source 信息只记录在 active source 的 Source History 中。
* `projects/bomcase/sources/_archive/` 下文件仅用于人工历史回溯，不作为当前交付输出事实源。
* `projects/bomcase/outputs/handoff/` 下的交付物不得反向作为事实源。

## 2. 页面目标

开箱页用于承接军需开启操作，当前确认重点是三栏职责清晰化与保底进度可读化。  
按当前 source，页面目标聚焦：

* 左侧展示资源总览；
* 中部承接主判断与主操作；
* 右侧承接卡牌填充选择；
* 在中部展示保底进度及常驻规则说明；
* 说明下方饰品区的保底范围标记关联。

## 3. 页面结构

* 顶部导航（已有功能范围）；
* 左侧资源概览区；
* 中部猎户军需核心区（X1/X5、满足度、资源构成、CTA）；
* 中部保底进度模块（条件展示）；
* 右侧卡牌背包区（半展开摘要态 / 全展开操作态 / 全展开底部固定汇总区）；
* 下方饰品区保底标记与底部掉落信息 Tab（已有功能边界）。

## 4. 核心表达

开箱页核心表达是“资源总览、操作判断、选卡填充三者分层协同”：

* 左侧只做资源概览，不直接承担开启判断；
* 中部是开启判断与主操作区；
* 右侧是卡牌填充选择器，不是资产管理区；
* 保底进度是辅助展示模块，不直接主导开启按钮状态；
* 页面展示以后端返回结果为准，不以前台自行推导替代。

本节只保留页面表达，不扩展为完整业务规则。

## 5. 关键页面判断

基于当前 source 可沉淀为：

* 中部为主判断区，左侧/右侧为支撑信息与操作；
* 右侧半展开与全展开应共用同一组件骨架；
* 右侧顶部不展示未确认口径计数（如 `[0/3]`、`3/300`）；
* 保底模块仅在配置保底玩法时展示，未配置不占位；
* 保底模块不参与开启满足度计算，不进入资源构成 icon 区；
* 切换 X1/X5 会触发已选卡牌清空与状态重算；
* 能量溢出值以后端返回为准；
* 选卡变化、X1/X5 切换、清除已选、开启后，均需刷新能量溢出提示；
* 能量溢出相关百分比按整数展示；
* 页面展示值允许作为预估值，最终以后端结算结果为准；
* 当已选卡牌能量超过当前开启所需补足能量时，右侧卡牌背包全展开态底部固定汇总区轻量展示能量溢出提示；无能量溢出时不展示；
* 能量溢出提示不进入中部满足度、资源构成 icon 区或左侧资源概览，不改变中部主操作区职责与开启按钮主状态；
* 上述能量溢出提示为 v1.2 引入并在 v1.3 延续的页面展示口径。
* 喜欢卡允许被选中并参与消耗，但需要二次确认；
* 页面表达重点应放在“当前已选卡会被消耗”，避免用户将喜欢态误解为不可消耗保护态；
* 即将过期能量的最小展示信息为即将过期数量；
* PC 端即将过期能量提示以 hover 方式触发。
* 中部保底 info 定位为常驻规则说明。

## 7. 与其它页面 / 流程关系

* 本页为开箱主操作页，结果反馈承接到 `open-box-result`；
* 触发保底后，开箱页需要告知用户已触发保底，并直接进入 `open-box-result`；
* 不要求用户在开箱页明确停留确认后再进入结果页；
* 右侧无可用卡牌时提供“前往获取”入口（按已有能力承接）；
* 下方饰品区与保底模块通过“保底范围标记”建立表达关联；
* 右侧卡牌入口与左侧资源入口形成联动。

本节仅描述页面表达与入口关系，不定义完整资产、权益、消耗、奖励、结算规则。
