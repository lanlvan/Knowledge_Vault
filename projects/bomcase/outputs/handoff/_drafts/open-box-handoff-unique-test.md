## 1. 本次变更背景

本次为研发向交接文档的阅读结构优化与表达增强，目标是让研发在同一份文档内快速定位变更点并获得完整上下文，不新增页面规则。

本期确认状态：已确认。  
本次输出类型：变更型交接文档。  
本次输出模式：增量。

## 2. 本期范围

### 2.1 【本次调整】范围说明

本期范围聚焦交接文档表达层优化：

1. 优化研发阅读路径与版本更新说明；
2. 前置不变更范围声明；
3. 对高影响模块补充最小实现口径；
4. 将待确认项按“阻塞开发启动 / 不阻塞开发启动但影响验收”分层；
5. 延续能量溢出提示展示口径，不新增产品事实。

### 2.2 【本次调整】不变更范围声明（前置）

除本期明确标注 `【本次调整】` / `【本次新增】` / `【本次延续】` / `【待确认】` 的内容外，其余章节均沿用上一版口径。

本次不改变：

- 展示条件（含保底模块展示条件）；
- 状态文案主口径；
- 开启判断与按钮主状态边界；
- 验收口径主结构。

## 3. 继承规则

### 3.1 页面目标与结构继承

Open Box 延续“三栏协同”定位：

- 左侧：资源总览；
- 中部：开启判断与主操作；
- 右侧：卡牌填充选择；
- 中部保底进度：辅助展示，不主导开启按钮状态；
- 下方饰品区：承接保底范围标记表达。

### 3.2 关键边界继承

- 右侧半展开与全展开共用同一组件骨架；
- 右侧顶部不展示未确认计数口径（如 `[0/3]`、`3/300`）；
- 保底模块仅在配置保底玩法时展示，未配置不占位；
- 切换 X1/X5 触发已选卡牌清空与状态重算；
- 展示以后端返回结果为准，前端不自行推导跨轮保底结果。

## 4. 本次变更点

### 4.1 【本次调整】研发阅读路径与版本说明

- 将阅读引导合并在文档顶部版本更新说明，不新增独立“阅读顺序”章节；
- 用“章节编号 + 章节标题”作为主要查看索引；
- 在研发影响章节内补充高影响模块的实现最小口径，减少跳读损耗。

### 4.2 【本次延续】右侧卡牌背包与能量溢出提示

当本次已选卡牌能量超过当前开启所需补足能量时，仅在右侧卡牌背包全展开态底部固定汇总区轻量展示：

`已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额`

延续边界：

- 无溢出时不展示该提示；
- 不进入中部满足度、资源构成 icon 区或左侧资源概览；
- 不改变中部主结构与开启按钮主状态；
- 仅为展示口径补充，不构成资源计算规则变更。

## 5. 不变更范围

- 页面目标、页面结构、组件职责不变；
- 保底模块“仅展示机制进度、不参与开启判断”的边界不变；
- 右侧半展开“摘要态”与全展开“操作态”定位不变；
- CTA 推荐态规则不变；
- 开启后保底最终展示以后端结算结果为准的口径不变；
- 待确认项仍保持待确认，不上升为已确认事实。

## 6. 研发影响

### 6.1 【本次调整】研发侧快速结论

- 本次以交付表达优化为主，不新增产品规则；
- 研发实施可沿用既有实现边界；
- 需重点关注第 7 节的分层待确认项，其中 A 组会阻塞开发启动。

### 6.2 【本次新增】高影响模块最小实现口径

#### 模块 A：中部保底进度模块

- 模块职责：展示当前保底进度、剩余积累次数、保底奖励名称与规则说明；
- 已确认展示口径：仅配置保底玩法时展示；未配置不展示且不占位；
- 关键状态/交互：累计中、新一轮稳定态、获得保底范围饰品结果态、触发保底结果态；info 为 hover；
- 研发影响侧：前后端；
- 待确认阻塞点：无（可按既有口径启动开发）；
- 验收关注点：不参与开启满足度，不影响一键填充/开启按钮，不进入资源构成 icon。

#### 模块 B：右侧卡牌背包全展开态底部固定汇总区

- 模块职责：展示已选张数、填充能量、清除入口及已选卡牌消耗说明；
- 已确认展示口径：有溢出时展示“溢出 +Y% 存入能量余额”，无溢出不展示；
- 关键状态/交互：未选态、已选无溢出、已选有溢出、清除已选后回退；
- 研发影响侧：前后端；
- 待确认阻塞点：`Y%` 的值来源与刷新时机；
- 验收关注点：溢出提示仅在全展开底部汇总区出现，不进入半展开、中部、左侧。

## 7. 待确认项

### 7.1 A. 【待确认】阻塞开发启动

1. 能量溢出值 `Y%` 的来源（后端直返 / 前端可计算）；
2. 能量溢出值刷新时机（选卡变化、X1/X5 切换、清除已选、开启后）；
3. 能量溢出值展示精度（取整/小数位/百分比规则）；
4. 溢出展示值与最终能量余额沉淀结果的一致性口径。

### 7.2 B. 【待确认】不阻塞开发启动，但影响验收

1. 右侧收起/展开 icon 的方向、动效、样式；
2. 喜欢卡与选中态并存时的 icon 排布、层级与避让细节；
3. 保底标记 icon 的视觉样式；
4. 左侧卡牌背包入口在右侧已全展开时，采用“抖动”还是“高亮”及持续时间；
5. 即将过期能量 hover 弹层内容、字段、样式、触发延迟；
6. 保底结果态展示时长、消失方式、是否过渡动效；
7. 保底 info 的位置、宽度、样式与动效细节。

- 保底模块与开启满足度、按钮状态边界不变；

- 切换 X1/X5 后已选卡牌清空并重算相关状态。
