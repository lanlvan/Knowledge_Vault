## 3. 研发影响标记

### 3.1 中间主操作区（前后端）

- 模块职责：承接开启次数选择、满足度展示、资源构成展示、主操作按钮。
- 已确认展示口径：中间区为页面主判断区；展示值以后端返回结果为准。
- 关键状态/交互：X1/X5 切换会清空已选卡牌并重算满足度与资源构成。
- 研发影响侧：前后端
- 待确认阻塞点：无新增阻塞；需保证中间区与右侧选卡联动一致。
- 验收关注点：按钮推荐态、满足度展示、资源构成与选卡联动同步正确。

### 3.2 右侧卡牌背包（前端）

- 模块职责：承担本次开箱流程中的卡牌填充选择，不作为资产管理区。
- 已确认展示口径：半展开为摘要态，全展开为操作态；二者共用同一组件骨架。
- 关键状态/交互：全展开底部固定汇总区支持“已选/填充/清除已选/溢出轻提示”。
- 研发影响侧：前端
- 待确认阻塞点：溢出值来源、刷新时机、展示精度、与最终余额沉淀一致性。
- 验收关注点：半展开不展示溢出提示；溢出提示仅在全展开底部固定汇总区出现。

### 3.3 保底进度模块（前后端）

- 模块职责：展示保底机制进度与奖励说明，不参与开启满足度判断。
- 已确认展示口径：仅在配置保底玩法时展示；未配置时不展示且不占位。
- 关键状态/交互：结果态与新一轮进度以后端结算返回为准，前端不自行推导跨轮结果。
- 研发影响侧：前后端
- 待确认阻塞点：结果态时长/消失方式/过渡动效，info 展示细节。
- 验收关注点：保底模块不影响按钮主状态，不进入中间资源构成 icon 区。

## 4. 页面目标

- 左侧展示账号资源总览信息，不承担开启判断。
- 中间承接当前开启判断与主操作。
- 右侧承接选卡填充操作与汇总反馈。
- 中间补充保底进度与规则说明入口，提升可读性。
- 下方饰品区域通过保底标记与保底进度表达建立关联。

## 5. 页面结构

1. 顶部导航（已有能力边界，本次不展开内部规则）。
2. 左侧资源概览区（资源状态与入口信息）。
3. 中间猎户军需核心区（X1/X5、满足度、资源构成、保底进度、CTA）。
4. 右侧卡牌背包区（半展开摘要态 / 全展开操作态 / 底部固定汇总区）。
5. 下方饰品区保底标记与底部掉落信息 Tab（已有能力边界）。

## 7. 核心页面口径

- 左侧只做资源总览，不承担开启判断与选卡操作。
- 中间为主判断与主操作区，满足度与资源构成以后端返回为准。
- 中间资源构成仅展示“本次实际参与资源”，未参与资源不展示且不占位。
- 右侧为选卡填充选择器，半展开用于摘要，全展开用于操作。
- 保底进度模块仅用于机制展示，不参与满足度计算，不影响按钮主状态。
- 能量溢出提示仅在右侧全展开底部固定汇总区展示。
- 溢出提示不进入中间满足度、中间资源构成 icon 区、左侧资源概览。
- X5 批量开启的保底最终结果以后端结算返回为准，前端不推导跨轮结果。

## 11. 待确认项

### A. 阻塞开发启动

1. 能量溢出值 Y% 的来源口径（后端直接返回或前端计算）。
2. 能量溢出值刷新时机（选卡变化、X1/X5 切换、清除已选、开启后）。
3. 能量溢出值展示精度（取整/小数位/百分比展示规则）。
4. 溢出展示值与最终能量余额沉淀结果一致性口径。

### B. 不阻塞开发启动，但影响验收

1. 右侧收起/展开 icon 的方向、动效、样式。
2. 喜欢卡与选中态并存时的 icon 排布、层级与避让细节。
3. 保底标记 icon 的视觉样式。
4. 左侧背包入口在右侧已全展开时采用“抖动”还是“高亮”以及持续时间。
5. 即将过期能量 hover 弹层的内容、字段、样式与触发延迟。
6. 保底结果态展示时长、消失方式、是否过渡动效。
7. 保底 info 的位置、宽度、样式与动效细节。

## 12. 验收口径

### 12.1 页面结构验收

1. 页面必须包含左侧资源概览、中间核心区、右侧卡牌背包三个主区域。
2. 保底模块位置必须在中间资源构成 icon 区下方、CTA 区上方。
3. 仅配置保底玩法的军需箱展示保底模块；未配置时不展示且不占位。
4. 右侧必须支持半展开与全展开，并共用同一组件骨架。
5. 全展开态为 2 列卡牌布局；仅卡牌列表可滚动，顶部与底部固定。

### 12.2 字段展示验收

1. 左侧“赠品开箱次数”可展示 0，作为 L1 主资源。
2. 左侧“能量余额”仅表示账号已有能量，不等于本次选卡填充能量。
3. 右侧“共 N 张卡牌”表示用户拥有总数，不等于筛选后列表数量。
4. 中间资源构成仅展示本次参与资源；未参与资源不展示、无空占位。
5. 右侧已选态必须展示“已选 N 张｜填充 +X%｜清除已选”。
6. 已选且无溢出时展示“已选卡牌将在开启时消耗”。
7. 已选且有溢出时展示“已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额”。
8. 半展开态不得展示“溢出 +Y% 存入能量余额”。
9. 溢出提示不得进入中间满足度、中间资源构成、左侧资源概览。

### 12.3 状态与交互验收

1. X1/X5 切换后，已选卡牌必须清空。
2. X1/X5 切换后，中间满足度与资源构成必须重算。
3. X1/X5 切换后，右侧底部已选态与溢出提示必须同步清空。
4. 一键填充遇到喜欢卡时必须跳过，并出现轻提示。
5. 未发生“跳过喜欢卡”时不得展示该轻提示。
6. 未配置保底玩法时，保底模块不得展示。
7. 保底结果态与批量跨门槛结果展示必须以后端返回为准。
8. 结果态后页面稳定态回到新一轮累计展示。
9. 待确认项在未确认前不得按已确认规则验收。

### 12.4 范围控制验收

1. 不在左侧会员区域新增 info 或点击交互。
2. 不在右侧顶部展示未确认计数（如 `[0/3]`、`3/300`）。
3. 不在半展开态展示筛选区、卡牌列表、底部固定汇总、溢出提示。
4. 不在本文内重定义资源扣减优先级、完整能量计算、完整结算发放规则。
5. 不因溢出提示新增独立弹窗、领取按钮或额外结算流程。

## 13. 不覆盖范围

- 完整资源扣减优先级规则。
- 完整能量计算与结算规则。
- 完整奖励发放与资产权益系统规则。
- 结果页详细表现规则。
- 顶部导航与底部掉落信息 Tab 的既有内部规则。
- 未经确认的新增交互、新增状态、新增字段口径。
