### 4.1 后端需要确认的事项

1. 开启满足度与资源构成的返回口径与展示口径是否一致。
2. 当前军需箱是否配置保底玩法，以及未配置时的展示控制信号。
3. 保底进度 `X/Y`、剩余积累次数 `N`、保底奖励名称返回口径。
4. 开启后的保底结算结果：自然命中、触发保底、新一轮进度。
5. X5 等批量开启场景的最终保底结果与前端展示边界。
6. 能量溢出值 `Y%` 的来源、刷新时机、展示精度与沉淀一致性。

### 4.2 不作为后端改动依据的内容

1. 左侧信息层级排版与局部展示层次。
2. 右侧半展开/全展开的具体视觉表现与动效样式。
3. 按钮推荐态的具体颜色、光效、图标样式。
4. icon 造型、描边、阴影、圆角、间距等具体视觉参数。
5. info 弹层与卡牌选中态的具体视觉实现细节。

## 五、页面目标

1. 左侧展示账号资源总览，不承担开启判断。
2. 中部作为开启判断与主操作区，承接满足度、资源构成、CTA。
3. 右侧作为卡牌填充选择器，支持半展开摘要态与全展开操作态。
4. 中部展示保底进度与规则说明入口，下方饰品区承接保底范围标记关联。
5. 页面展示以后端返回结果为准，前端不自行推导替代结算结果。

## 八、核心页面口径

### 8.1 卡牌数量口径

1. 左侧 `N张` 与右侧半展开 `共 N 张卡牌` 均表示用户拥有卡牌总数。
2. 右侧全展开列表是选择器，展示数量可受筛选和可用状态影响，不等同拥有总数。
3. 喜欢卡保护不影响列表展示数量，只影响一键填充自动选择逻辑。

### 8.2 能量口径

| 位置 | 展示内容 | 口径 |
| --- | --- | --- |
| 左侧资源概览 | 能量余额 18% | 账号已有能量 |
| 右侧半展开/全展开汇总 | 填充 +X% | 本次已选卡牌新增能量，不含左侧已有能量 |
| 右侧全展开底部补充 | 溢出 +Y% 存入能量余额 | 本次填充超过当前补足所需时，超出部分沉淀到余额 |
| 中部资源构成 | 卡牌能量 +X% | 本次计入开启判断的卡牌能量，以返回结果为准 |

补充边界：
- 中部满足度与资源构成以后端返回结果为准。
- 溢出提示仅归属右侧全展开底部固定汇总区。
- 不在左侧、中部满足度区、资源构成 icon 区重复展示溢出提示。

### 8.3 资源参与口径

| 场景 | 中部显示 | 中部不显示 |
| --- | --- | --- |
| 纯赠品路径 | 赠品次数×N | 卡牌能量、扳手、VIP权益 |
| 纯能量路径 | 卡牌能量+X%、扳手×N、VIP权益+X%（如参与） | 赠品×0 |
| 混合路径 | 赠品次数×N、卡牌能量+X%、扳手×N、VIP权益+X%（如参与） | 未参与资源 |
| 无资源参与 | 不显示资源构成 icon | 全部资源项 |

### 8.4 保底模块口径

1. 保底模块仅展示机制进度与规则说明，不参与开启满足度计算。
2. 保底模块不影响一键填充、不影响开启军需按钮状态、不进入资源构成 icon 区。
3. 未配置保底玩法时，模块不展示且不占位。
4. 保底进度、目标进度、奖励名称、结算结果以后端配置和返回为准。
5. 批量开启跨轮结果以后端最终结算返回为准，前端不自行推导。

### 8.5 X1/X5 切换口径

- 切换后必须清空已选卡牌。
- 切换后必须重算满足度与资源构成。
- 右侧底部已选汇总与溢出提示同步清空。
- 保底进度不因切换动作直接变化，仅在开启结算后按结果刷新。

### 9.4 CTA 按钮状态表（表达约束 + 判定标准）

| 场景 | 一键填充 | 开启军需 | 主推荐操作 | 判定标准 |
| --- | --- | --- | --- | --- |
| 赠品已满足 | 非推荐态 | 推荐态 | 开启军需 | 用户无需阅读长说明即可识别“直接开启”为当前主操作；同一时刻仅一个主推荐操作 |
| 能量已满足 | 非推荐态 | 推荐态 | 开启军需 | 与上同 |
| 可填充但未满足 | 推荐态 | 非推荐态 | 一键填充 | 用户可快速识别“先补足再开启”的操作顺序；非推荐态不抢主操作注意力 |
| 无可用填充且不满足 | 不可推荐/弱化态 | 非推荐态（但保持可点击） | 无明确推荐 | 一键填充不作为推荐；开启军需保持可触达并触发既有能力 |

补充边界：
- 保底模块与能量溢出提示不改变 CTA 主状态判定。
- 文档定义的是表达约束与可验证标准，不固化具体颜色、动效、阴影、圆角、间距等实现。

## 十三、验收口径

### 13.1 页面结构验收

1. 页面应包含左中右三栏与下方饰品区，结构职责清晰。
2. 保底模块位于中部资源构成下方、CTA 上方，仅配置保底玩法时展示。
3. 右侧半展开与全展开为同一组件骨架；全展开仅列表滚动，顶部与底部固定。
4. 能量溢出提示仅出现在右侧全展开底部固定汇总区。

### 13.2 字段展示验收

1. 左侧赠品、能量、扳手、卡牌入口、会员等级字段按层级展示，会员不新增 info。
2. 左侧和右侧半展开的 `N张/共 N 张卡牌` 口径一致，均表示拥有总数。
3. 中部资源构成只展示参与资源；未参与资源与 `×0` 不展示。
4. 保底主文案、结果态文案、info 入口均可按状态正确展示。
5. 右侧全展开已选态必须同时覆盖主文案、消耗说明、溢出说明（按条件展示）。

### 13.3 状态与交互验收

1. 半展开四种状态（赠品满足、能量满足、已有填充、无可用卡牌）可完整切换。
2. 全展开状态切换与底部汇总保持一致，清除已选后相关说明清空。
3. X1/X5 切换触发“清空已选 + 重算 + 清空溢出提示”。
4. 一键填充跳过喜欢卡时必须出现轻提示，未跳过时不出现。
5. 保底模块状态与开启后结算结果一致，批量开启以后端最终结果为准。

### 13.4 CTA 状态验收

1. 赠品满足/能量满足时，开启军需为唯一主推荐操作。
2. 可填充未满足时，一键填充为唯一主推荐操作。
3. 无可用填充且不满足时，一键填充不作为推荐；开启军需保持可点击并触发既有能力。
4. 保底模块与溢出提示不改变 CTA 主状态判定。

### 13.5 待确认项验收边界

1. 待确认项不得作为已确认规则进行通过/失败判定。
2. 涉及 A 类待确认的场景应标记为“待产品/研发确认后补验”。
3. 涉及 B 类待确认的视觉实现以最终设计结论补充验收。

## 十四、不覆盖范围

1. 不在本文档定义完整资源扣减优先级、完整能量计算规则、完整奖励结算系统规则。
2. 不展开顶部导航、底部掉落信息 Tab 等已有功能内部实现细则。
3. 不新增未确认功能（如独立溢出弹窗、额外结算流程、新按钮流程）。
4. 不固化具体视觉实现（颜色、图标造型、动效曲线、描边、阴影、圆角、间距等）。
5. 结果页详细呈现规则由结果页文档承接。
