## 3. 研发影响标记

### 3.1 【本次新增】高影响模块最小实现口径：保底进度模块

- 模块职责：展示当前军需箱保底进度、剩余积累次数、保底奖励名称与规则说明入口；
- 已确认展示口径：仅在配置保底玩法时展示，未配置不展示且不占位；
- 关键状态 / 交互：累计中、新一轮稳定态、结果态展示；info 为 hover；
- 研发影响侧：前后端；
- 待确认阻塞点：无；
- 验收关注点：不参与开启满足度、不影响一键填充/开启按钮状态、不进入资源构成 icon 区。

### 3.2 【本次新增】高影响模块最小实现口径：右侧底部固定汇总区（能量溢出提示）

- 模块职责：展示已选卡牌汇总、填充能量、清除入口与消耗说明；
- 已确认展示口径：有溢出时展示“溢出 +Y% 存入能量余额”，无溢出不展示；
- 关键状态 / 交互：未选态、已选无溢出、已选有溢出、清除已选回退；
- 研发影响侧：前后端；
- 待确认阻塞点：`Y%` 来源、刷新时机、展示精度；
- 验收关注点：提示仅在右侧全展开态底部固定汇总区，不进入中部或左侧。

## 4. 页面目标

开箱页用于承接军需开启操作，页面表达聚焦三栏职责清晰化与保底进度可读化：

- 左侧展示资源总览；
- 中部承接主判断与主操作；
- 右侧承接卡牌填充选择；
- 中部展示保底进度及规则说明入口；
- 下方饰品区承接保底范围标记关联表达。

## 5. 页面结构

- 顶部导航（已有功能范围）；
- 左侧资源概览区；
- 中部猎户军需核心区（X1/X5、满足度、资源构成、CTA）；
- 中部保底进度模块（条件展示）；
- 右侧卡牌背包区（半展开摘要态 / 全展开操作态 / 全展开底部固定汇总区）；
- 下方饰品区保底标记与底部掉落信息 Tab（已有功能边界）。

## 6. 组件与字段

- 左侧资源概览：赠品开箱次数、能量余额、机械扳手、卡牌背包入口、会员等级；
- 中部核心区：军需箱主视觉、X1/X5、开启满足度、资源构成 icon、保底进度模块、一键填充、开启军需；
- 右侧卡牌背包：半展开摘要、全展开筛选与列表、底部固定汇总区；
- 下方饰品区：保底范围饰品标记（仅表达关联，不扩展既有规则）。

## 7. 核心页面口径

- 中部为主判断区，左侧/右侧为支撑信息与操作；
- 右侧半展开与全展开共用同一组件骨架；
- 右侧顶部不展示未确认口径计数（如 `[0/3]`、`3/300`）；
- 保底模块仅在配置保底玩法时展示，未配置不占位；
- 保底模块不参与开启满足度计算，不进入资源构成 icon 区；
- 切换 X1/X5 触发已选卡牌清空与状态重算；
- 【本次延续】当已选卡牌能量超过当前开启所需补足能量时，仅在右侧卡牌背包全展开态底部固定汇总区轻量展示能量溢出提示；无能量溢出时不展示；
- 【本次延续】能量溢出提示不进入中部满足度、资源构成 icon 区或左侧资源概览，不改变中部主操作区职责与开启按钮主状态。

## 8. 页面状态

- 右侧分为半展开摘要态与全展开操作态；
- 有卡牌且无需填充时，半展开展示“查看背包”路径；
- 已有填充时，半展开展示“已选 N 张，填充 +X%”摘要；
- 无可用卡牌时，半展开展示空态与“前往获取”入口；
- 全展开态底部固定汇总区按已选与溢出状态切换文案；
- 保底结果态展示后，页面稳定态回到新一轮进度展示。

## 9. 交互规则

- X1/X5 切换后，已选卡牌清空，开启满足度与相关状态重算；
- 一键填充用于补足动作，开启军需为最终动作；
- 一键填充跳过喜欢卡时展示轻提示；
- 右侧全展开态支持选中、反选、清除已选；
- 开启后保底最终展示以后端结算结果为准，前端不自行推导跨轮结果；
- 保底 info 与相关说明入口采用 hover 触发口径。

## 10. 空状态 / 异常状态

- 右侧无可用卡牌时展示空状态与“前往获取”入口；
- 保底模块未配置时不展示且不占位；
- 无能量溢出时不展示“溢出 +Y% 存入能量余额”；
- 本期不新增未确认的 loading、权限、异常恢复等规则。

## 11. 待确认项

### 11.1 A. 【本次调整】【待确认】阻塞开发启动

1. 能量溢出值 `Y%` 的来源（后端直接返回 / 前端可计算）；

4. 溢出展示值与最终能量余额沉淀结果的一致性口径。

### 11.2 B. 【本次调整】【待确认】不阻塞开发启动，但影响验收

1. 右侧收起/展开 icon 的方向、动效、样式；
2. 喜欢卡与选中态并存时的 icon 排布、层级与避让细节；
3. 保底标记 icon 的视觉样式；
4. 左侧卡牌背包入口在右侧已全展开时反馈采用“抖动”还是“高亮”及持续时间；
5. 即将过期能量 hover 弹层内容、字段、样式与触发延迟；
6. 保底结果态展示时长、消失方式、是否过渡动效；
7. 保底 info 的位置、宽度、样式与动效细节。
