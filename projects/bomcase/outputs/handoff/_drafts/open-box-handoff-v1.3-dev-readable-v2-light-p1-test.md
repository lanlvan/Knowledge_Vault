## 7. 核心页面口径【本次延续】

### 7.1 全局研发边界

- 前端不自行推导：中间满足度、资源 icon、保底命中/触发/跨轮结果、批量开启优先级。
- 产品定义展示口径与验收边界；接口字段与计算方式由后端 / 研发内部确认。
- 最终展示以后端配置与服务端返回为准。

### 7.2 能量与资源参与

- 中间资源 icon 仅展示本次参与资源；未参与、×0 不展示。
- `溢出 +Y%` 不进入左侧、中间满足度、中间资源 icon、本期保底卡片。
- 不扩展能量计算公式、扣减顺序、有效期系统规则。

### 7.3 保底模块

| 项目 | 口径 |
|---|---|
| 参与开启满足度 / 影响 CTA / 进入资源 icon | 均不参与 / 不影响 / 不进入 |
| 展示条件 | 仅配置保底玩法；未配置不展示不占位 |
| 自然命中焦点区 | 本轮完成并进入新一轮；不额外触发保底奖励 |
| 触发保底 | 服务端按配置发放 1 个奖励；不发放军需箱、不要求二次开启 |
| 批量开启 | 最终展示以服务端返回为准 |

### 7.4 X1 / X5 切换

- 切换后：清空已选、重算满足度与资源 icon、重置右侧状态、清空溢出提示。
- 保底卡片不因切换直接变化；进度在开启结算后按服务端返回刷新。
- X5 且赠品不足 5 次时，赠品可部分计入，剩余由卡牌能量等补足。

## 8. 页面状态【本次延续】

### 8.1 CTA 推荐态

| 状态 | 一键填充 | 开启军需 |
|---|---|---|
| 赠品足够 / 能量足够 | 非推荐 | 推荐 |
| 可填充未满足 | 推荐 | 非推荐 |
| 都没有资源 | 非推荐 | 可点击常规态（触发已有功能） |

### 8.2 保底状态

| 状态 | 文案 | 说明 |
|---|---|---|
| 未配置 | 不展示 | 不占位 |
| 常规累计 | `进度 X/Y` | Y≤10 分段条；Y>10 连续条 |
| 临近完成 | `再开启 N 次触发` | N 以服务端返回为准 |
| 自然命中 | `已获得焦点区卡牌，新一轮已开启` | 不额外触发保底奖励 |
| 结算触发 | `奖励发放中...` | 瞬时态，建议最多 2 秒 |
| 新一轮延续 | `上一轮已完成，新一轮已开启` | 最终回到常规累计 |

### 8.3 右侧背包状态

| 场景 | 半展开 | 全展开底部 |
|---|---|---|
| 赠品/能量已满足 | 对应状态文案 + `查看背包 ›` | 对应非填充文案 |
| 已有填充 | `已选 N 张，填充 +X%` + `查看背包调整 ›` | `已选 N 张｜填充 +X%｜清除已选` + 消耗说明 |
| 无卡牌 | `暂无可用卡牌` + `前往获取 ›` | — |
| 半展开限制 | 不展示筛选、列表、底部汇总、溢出 | 列表区可滚动；顶/底固定 |

## 9. 交互规则【本次延续】

| 触发对象 | 条件 | 结果 |
|---|---|---|
| 资源概览 / 能量余额 info | hover | 展示对应说明 |
| 即将过期 X% | hover | 展示过期说明弹层（后续迭代） |
| 卡牌背包 N张 › | 点击 | 右侧进入全展开；已全展开则抖动或高亮（方式待确认） |
| X1 / X5 | 点击未选项 | 切换并清空已选，重算状态 |
| 一键填充 | 可填充且需补能量 | 自动选卡；遇喜欢卡跳过并轻提示 |
| 开启军需 | 满足 / 不满足 | 执行开启 / 触发已有功能 |
| 保底 info | hover | 展示保底规则 |
| 普通卡 | 点击 | 选中/反选，立即计入填充 |
| 喜欢卡 | 未选中点击 | 二次确认后加入 |
| 清除已选 | 已选 > 0 | 清空并回退底部文案 |
| 批量开启 X5 | 配置保底 | 最终展示以服务端返回为准 |
| 焦点区/保底标记 | — | 不新增点击或 hover |

- 离开页面：已选与溢出提示不持久化。
- 一键填充满足引导条件时，右侧可自动进入全展开并展示已选。

## 12. 验收口径【本次延续】

> 本节为可验证断言，不展开完整 QA checklist；详细用例可另出 Acceptance Draft。

### 12.1 结构与范围

- 三栏 + 左下方保底卡片结构完整；右侧半展开/全展开为同一组件骨架。
- 溢出提示仅归属右侧全展开底部；不得进入左侧、中间、保底卡片。
- 焦点区标记与保底奖励标记按配置展示，不得混用。

### 12.2 文案与状态

- 第 6 章全部用户可见文案在对应场景可正确展示。
- X1/X5 切换清空已选、重算满足度、清空溢出提示。
- 半展开不展示溢出；全展开已选且有溢出时展示完整溢出文案。
- 保底六态可识别；结算触发态为瞬时态。

### 12.3 交互与服务端边界

- 选卡、反选、清除、一键填充、喜欢卡保护行为符合第 9 章。
- 中间满足度、资源 icon、保底结果、批量开启结果均以服务端返回为准，前端不自行推导。
- 待补充项（第 11 章）不得作为已确认规则验收。

## 13. 不覆盖范围【本次延续】

- 不扩展完整能量计算、资源扣减、资产入账、奖励发放、库存、异常补偿规则。
- 不定义后端随机算法、保底后台配置流程、跨系统结算机制。
- 不新增保底奖励领取按钮、二次开启流程、奖励详情页。
- 不展开顶部导航、掉落 Tab 等既有功能内部规则。
- 不把示例值、待补充边界写成固定系统规则。
