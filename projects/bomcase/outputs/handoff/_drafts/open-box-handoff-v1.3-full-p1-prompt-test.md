# Open Box 研发交接文档

## 1. 文档版本说明

本版本为当前页面口径的完整研发交接，不新增产品规则。

本版本基于当前已确认页面事实生成，覆盖研发实现、联调、UI 承接和测试验收所需信息。

页面目标是稳定三栏职责（左侧资源概览 / 中间主判断与主操作 / 右侧选卡填充），并在页面左下方承载「本期保底」状态表达。最终开启满足度、保底进度与结果状态以服务端返回为准。

## 4. 页面目标

当前范围覆盖：

1. 左侧资源概览区域（账号资源总览）；
2. 中间核心操作区域（开启判断、资源构成、主操作）；
3. 右侧卡牌背包区域（选卡填充入口与操作）；
4. 页面左下方本期保底卡片（范围、进度、结果态）；
5. 下方饰品区域焦点区标记与保底奖励标记（仅说明保底关联）。

本次不改变资源消耗优先级、能量计算规则、库存/发放等后端系统规则。

## 7. 核心页面口径

### 7.1 关键概念

| 概念 | 口径 |
| --- | --- |
| 焦点区卡牌 | 用户侧保底命中范围表达 |
| 保底范围饰品 | 研发/服务端理解口径，与焦点区卡牌指向同一配置对象 |
| 填充 +X% | 本次已选卡牌带来的新增能量，不含账号已有能量 |
| 卡牌能量 +X% | 本次开箱计入满足度的卡牌能量值，具体以服务端返回为准 |
| 溢出 +Y% | 本次填充超过补足需求后的沉淀部分，仅在右侧全展开底部汇总区表达 |

### 7.2 能量与资源参与口径

- 左侧能量余额是账号已有能量；右侧填充 +X% 是本次新增能量；中间卡牌能量 +X% 以服务端返回为准。
- 前端不得用“左侧已有 + 右侧新增”自行推导中间卡牌能量或开启满足度。
- 中间资源构成只展示本次参与资源；未参与资源、×0 不展示。
- 本期保底卡片和能量溢出提示都不进入中间资源构成 icon 区。

### 7.3 本期保底口径

- 本期保底卡片仅展示保底状态，不参与开启满足度计算、不影响一键填充和开启军需按钮状态。
- 自然命中焦点区：本轮完成并开启新一轮，不额外触发保底奖励发放。
- 连续未命中达到门槛：服务端按配置发放 1 个保底奖励并开启新一轮。
- 批量开启场景下的最终保底结果以后端结算返回为准，前端不做跨轮推导。

### 7.4 X1 / X5 切换口径

- 选择 X5 且赠品不足 5 次时，赠品可部分计入，剩余可由卡牌能量补足。
- 切换 X1/X5 后：已选卡牌清空，满足度重算，资源构成重算，右侧状态重判，底部汇总与溢出提示清空。

## 8. 页面状态

### 8.1 左侧资源概览状态

- L1 主资源：赠品次数；L2 次主资源：账号能量余额；L3 辅助信息：扳手、卡牌入口、会员等级。
- 即将过期能量仅在存在时展示；正常状态不展示。
- info 类交互统一 hover；会员等级不新增 info 或点击交互。

### 8.2 中间核心区状态

| 状态 | 一键填充 | 开启军需 | 说明 |
| --- | --- | --- | --- |
| 赠品足够 | 非推荐 | 推荐 | 推荐直接开启 |
| 能量足够 | 非推荐 | 推荐 | 推荐直接开启 |
| 可填充但未满足 | 推荐 | 非推荐 | 推荐先填充 |
| 都没有 | 非推荐 | 非推荐 | 开启保持可点击，触发已有功能 |

统一表达边界（适用于上表全部状态）：需有明确主次区分，具体视觉表现以最终设计稿为准。

### 8.3 本期保底卡片状态

| 状态 | 触发条件 | 展示文案 |
| --- | --- | --- |
| 未配置保底 | 当前军需箱未配置保底玩法 | 不展示 |
| 常规累计态 | 配置保底且未临近触发 | 进度 X/Y |
| 临近完成态 | 距离触发较近 | 再开启 N 次触发 |
| 自然命中态 | 开启结果获得任一焦点区卡牌 | 已获得焦点区卡牌，新一轮已开启 |
| 结算触发态 | 连续未命中达到门槛 | 奖励发放中... |
| 新一轮延续态 | 上一轮完成后 | 上一轮已完成，新一轮已开启 |

### 8.4 右侧卡牌背包状态

- 半展开态仅做摘要，不展示筛选、卡牌列表、底部汇总和溢出提示。
- 全展开态展示完整操作结构与底部汇总。
- 喜欢卡与选中态需共存可识别，不得互相覆盖。
- 一键填充跳过喜欢卡时展示轻提示；非该场景不展示。

| 5 | 保底结果态 | 自然命中态/新一轮延续态时长与消失方式 | 结果态体验与验收 | 产品确认后 UI/前端承接 |

## 12. 验收口径

### 12.1 页面结构验收

- 三栏结构与左下方本期保底卡片完整存在。
- 右侧支持半展开与全展开，且为同一组件骨架。
- 全展开采用 2 列卡牌布局，顶部与底部固定，列表区滚动。
- 能量溢出提示仅归属右侧全展开底部汇总区。

### 12.2 字段展示验收

- 左侧 `N张` 与右侧半展开 `共 N 张卡牌` 表示用户拥有卡牌总数。
- 中间资源构成只展示本次参与资源，不展示未参与资源和 ×0。
- 本期保底关键文案与状态文案完整可展示。
- 右侧底部文案需覆盖未选、已选、无溢出、有溢出四类关键场景。
- 喜欢卡轻提示仅在一键填充跳过场景出现。

### 12.3 状态与交互验收

- X1/X5 切换后必须清空已选并重算满足度，且清空溢出提示。
- 可填充未满足时，一键填充为推荐；已满足时，开启军需为推荐。
- 清除已选后，底部不再显示消耗说明与溢出提示。
- 批量开启保底相关最终展示以后端返回结果为准。

### 12.4 范围控制验收

- 不在左侧展示会员权益加成值。
- 不在半展开态展示筛选、列表、底部汇总、溢出提示。
- 不在中间或左侧展示溢出提示。
- 不新增保底奖励领取按钮、二次开箱流程或未确认能力。

## 13. 不覆盖范围

- 不补完整资源扣减、结算、库存、支付、到账、补偿等系统规则。
- 不定义后端奖励随机算法、保底配置后台流程。
- 不把示例值写成固定系统规则。
- 不把待补充边界写成已确认产品事实。
- 不新增保底奖励领取按钮、弹窗或详情页。
