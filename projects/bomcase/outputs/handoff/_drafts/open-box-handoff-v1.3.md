# open box 研发交接文档 v1.3

## 1. 页面目标

本页面用于用户开启军需，当前确认范围覆盖：

1. 左侧「资源概览」区域：展示用户当前与开箱相关的账号资源状态。
2. 中间「猎户军需」核心操作区域：展示军需箱主视觉、开启次数选择、开启满足度、资源构成与主操作按钮。
3. 右侧「卡牌背包」区域：作为当前开箱流程中的卡牌填充选择器，用于支持卡牌填充能量操作。
4. 页面左下方「本期保底」运营活动卡牌：展示本期保底状态、焦点区卡牌范围、累计进度与结果状态。
5. 下方饰品区域中的焦点区标记与保底奖励标记：仅说明其与本期保底卡片的关联，不展开饰品区域其它已有规则。

开启条件可由赠品次数、账号已有能量、卡牌填充能量及其他实际参与资源共同满足。所有 UI 示意图仅用于辅助说明，不作为像素级规范；如示意图与本文档存在差异，以本文档为准。

本次重点：

- 明确左侧资源概览、中间主操作、右侧卡牌填充选择器的职责分离。
- 强化中间区域作为当前页面主判断区和主操作区。
- 在页面左下方展示「本期保底」运营活动卡牌，展示当前军需箱的保底状态、焦点区卡牌范围与规则说明。
- 优化右侧卡牌背包在开箱流程中的半展开摘要态与全展开操作态。
- 明确卡牌填充、喜欢卡保护、资源构成 icon、本期保底卡片与按钮推荐态之间的边界。
- 在右侧卡牌背包全展开态底部固定汇总区，轻量提示本次填充能量溢出后的余额沉淀结果。

## 2. 研发关注边界

本页面大部分内容为 UI / 前端展示与交互调整。明确需要研发确认的内容集中在「本期保底」卡片及其开启后的结算结果展示；本文档只标记可能涉及后端确认的产品变化，不指定接口结构、字段命名、表结构、服务拆分或技术实现方式。

### 2.1 涉及后端确认的内容

| 事项 | 说明 |
|---|---|
| 当前军需箱是否配置保底玩法 | 决定是否展示页面左下方「本期保底」卡片；未配置时不展示、不占位。 |
| 当前保底进度 X/Y | 用于展示当前进度、目标进度和进度条。 |
| 剩余开启次数 N | 用于展示「再开启 N 次触发」。 |
| 焦点区卡牌范围 | 用于前台展示本期保底命中范围；研发口径对应当前箱子内带保底标记的指定范围卡牌 / 饰品。 |
| 焦点区标记 | 用于标识当前箱子中属于本期保底命中范围的卡牌。 |
| 保底奖励标记 | 用于标识触发保底后可能发放的奖励范围。 |
| 开启后的保底结算结果 | 包括是否自然命中焦点区、是否触发保底、新一轮进度等。 |
| X5 等批量开启后的最终保底结果 | 前端不逐次计算命中与进度变化，以后端结算结果为准。 |
| 本次填充能量溢出值 Y% | 用于右侧卡牌背包全展开态底部固定汇总区展示“溢出 +Y% 存入能量余额”。若不存在溢出，不展示该提示。 |

### 2.2 不作为后端改动依据的内容

- 左侧资源概览信息层级。
- 中间开启满足度视觉表达。
- 本次资源构成 icon 区排列方式。
- 一键填充与开启军需按钮推荐态。
- 右侧卡牌背包半展开态 / 全展开态。
- 右侧卡牌背包底部能量溢出轻提示展示样式。
- 喜欢卡保护轻提示展示。
- 本期保底卡片的视觉样式、info 样式、焦点区标记样式、保底奖励标记样式。
- 按钮颜色、光效、动效、卡片样式和布局间距。

### 2.3 研发边界说明

- 产品只确认页面展示口径、状态边界和验收口径。
- 产品不定义接口字段、数据表、服务拆分和后端计算方式。
- 前端涉及保底结果展示时，不自行推导焦点区命中、保底触发、保底奖励、新一轮进度或批量开启跨轮结果。
- 前端不通过左侧账号资源与右侧已选卡牌自行推导中间开启满足度、资源构成 icon 或保底结果。
- 能量溢出轻提示仅展示后端或既有开启判断结果可确认的溢出值，不重新定义资源扣减优先级和能量计算规则。
- 最终页面展示以后端配置与接口返回为准。

## 3. 页面结构

### 3.1 顶部导航区域

| 项目 | 说明 |
|---|---|
| 区域名称 | 顶部导航 |
| 区域位置 | 页面顶部 |
| 包含元素 | BOMCASE Logo、导航菜单、用户相关入口、道具 / 消息 / 头像等入口 |
| 已确认说明 | 顶部导航为已有功能，本次不展开 |

### 3.2 左侧资源概览区域

| 项目 | 说明 |
|---|---|
| 区域名称 | 资源概览 |
| 区域位置 | 页面左侧常驻面板 |
| 包含元素 | 区域标题、副说明、资源概览 info、赠品开箱次数、能量余额、能量余额 info、即将过期能量提示、机械扳手、卡牌背包入口、会员等级 |
| 已确认说明 | 左侧区域只做账号资源概览，不承担本次是否可开启判断，也不承担卡牌选择操作 |

左侧资源概览展示账号当前资源总览，可展示数量为 0 的资源。信息层级分为三层：

| 层级 | 内容 | 说明 |
|---|---|---|
| L1 主资源 | 赠品开箱次数 | 最高优先级展示。 |
| L2 次主资源 | 账号已有能量余额 | 中等优先级展示。 |
| L3 辅助资源 / 入口 / 身份信息 | 机械扳手、卡牌背包、会员等级 | 弱于 L1 / L2，不抢主视觉。 |

会员等级只展示身份和等级进度，不展示会员权益加成数值，不展示会员等级 info，不新增会员等级点击交互。能量即将过期提示保留在「能量余额」卡片内展示，不额外增加左侧底部状态提示区。

### 3.3 中间猎户军需核心区域

中间区域负责本次开启判断和主操作，核心是展示后端确认的满足度，不自行推导资源结果。

| 项目 | 说明 |
|---|---|
| 区域名称 | 猎户军需核心操作区 |
| 区域位置 | 页面中间主视觉与主操作区 |
| 包含元素 | 页面标题、军需箱主视觉、X1 / X5 开启次数选择、开启满足度、本次资源构成 icon 区、CTA 按钮区 |
| 已确认说明 | 中间区域是当前页面的主判断区和主操作区 |

中间区域负责表达当前选择开启次数、当前开启满足度、本次实际参与资源和当前推荐操作。中间区域不做账号资源总览，不展示完整卡牌列表，不展示本次卡牌填充溢出说明。

中间「本次资源构成 icon 区」仅展示本次实际参与本次开启判断的资源；未参与资源、数量为 0 的资源不在中间 icon 区展示。中间区域展示的开启满足度与资源构成 icon 以后端返回的本次计入结果为准。

### 3.4 本期保底卡片

「本期保底」卡片展示当前保底状态、焦点区卡牌范围、累计进度与结果状态，核心是不参与开启满足度，也不影响按钮状态。

| 项目 | 说明 |
|---|---|
| 区域名称 | 保底模块 / 本期保底卡片 |
| 区域位置 | 页面左下方区域 |
| 展示形式 | 运营活动卡牌 |
| 前台标题 | 本期保底 |
| 范围标签 | 焦点区卡牌 |
| 包含元素 | 固定奖励视觉资源、模块标题、范围标签、info icon、进度文案、进度条、状态文案 |
| 展示条件 | 仅配置保底玩法的指定军需箱展示 |
| 未配置处理 | 未配置保底玩法的军需箱不展示该模块，不占位 |
| 已确认说明 | 常态展示本期保底状态、焦点区卡牌范围、累计进度与结果状态；完整规则放入 info |

本期保底卡片仅展示当前军需箱的保底相关状态与规则说明，不参与开启满足度计算，不影响一键填充逻辑，不影响开启军需按钮状态，不进入中间资源构成 icon 区，不直接影响 X1 / X5 切换，不影响右侧卡牌背包。

保底进度、目标进度、剩余开启次数、焦点区卡牌范围、焦点区标记、保底奖励标记、批量开启最终保底结果，均以后端配置与接口返回为准。固定奖励视觉资源仅用于运营活动卡牌表达，不代表最终实际发放奖励，不根据实际奖励内容动态替换。

### 3.5 右侧卡牌背包区域

右侧卡牌背包用于本次选卡填充，核心是临时选择器，不是完整资产管理区；半展开态只展示摘要，溢出提示仅在全展开态底部展示。

| 项目 | 说明 |
|---|---|
| 区域名称 | 卡牌背包 |
| 区域位置 | 页面右侧面板 |
| 包含元素 | 半展开态摘要面板、全展开态卡牌选择面板、右侧顶部收起 / 展开切换 icon、卡牌筛选、2 列卡牌列表、卡牌选中态、喜欢卡标识、一键填充跳过喜欢卡轻提示、底部固定汇总区 |
| 已确认说明 | 右侧「卡牌背包」保留资产语义，但实际定位是当前开箱流程中的卡牌填充选择器，不是资产管理区 |

右侧面板分为半展开态和全展开态。半展开态是当前状态摘要，全展开态是操作详情。两种展示态应使用同一组件骨架，区别只在内容展示，不应表现为两个不同组件。

右侧顶部展示收起 / 展开切换 icon，不展示 `[0/3]`、`3/300` 等未确认口径计数。右侧底部固定汇总区在已选卡牌状态下，负责展示本次已选卡牌数量、填充能量、清除入口与消耗说明。当本次存在填充能量溢出时，右侧底部固定汇总区轻量补充展示溢出能量沉淀提示。

### 3.6 下方饰品区域与底部掉落信息 Tab

| 区域 | 说明 |
|---|---|
| 下方饰品区域 | 为已有功能，本次仅说明焦点区标记、保底奖励标记与本期保底卡片的关联。 |
| 掉落信息 Tab | 精彩掉落、最近掉落、个人掉落为已有功能，本次不展开。 |

下方饰品区域中，焦点区标记用于标识本期保底命中范围，保底奖励标记用于标识触发保底后的可能奖励范围。两类标记指向同一批卡牌，但表达含义不同，前端按后端配置展示，不自行合并、不自行推导、不用单一标记替代两类标记，除非产品和 UI 另行确认。

## 5. 核心口径

### 5.1 卡牌数量口径

左侧「卡牌背包 N张」与右侧半展开态「共 N 张卡牌」表示用户当前拥有的卡牌总数。

右侧全展开列表是卡牌选择器，展示数量可受筛选、可用状态影响，不等同于用户拥有卡牌总数。喜欢卡保护不影响右侧列表展示，仅影响一键填充自动选择逻辑。设计图中的右侧列表展示数量仅作为布局示例，不作为卡牌数量口径。

### 5.2 能量口径

| 位置 | 展示内容 | 口径说明 |
|---|---|---|
| 左侧资源概览 | 能量余额 18% | 账号已有能量余额。 |
| 右侧底部汇总 / 半展开填充态 | 填充 +X% | 本次已选卡牌带来的新增能量，不包含账号已有能量。 |
| 右侧底部汇总补充说明 | 溢出 +Y% 存入能量余额 | 本次已选卡牌能量超过当前开启所需补足能量时，超出部分在开启后沉淀至账号能量余额。仅存在溢出时展示。 |
| 中间资源构成 icon | 卡牌能量 +X% | 本次开箱计入满足度的卡牌能量值，具体数值以后端返回为准。 |

前端不通过“左侧账号已有能量 + 右侧本次选卡新增能量”自行计算中间「卡牌能量」展示值。中间「开启满足度」与资源构成均以后端返回的本次开箱判断结果为准。

右侧「填充 +X%」表示本次已选卡牌带来的新增能量，不包含账号已有能量，也不因是否溢出而改变其展示口径。当本次已选卡牌能量超过当前开启所需补足能量时，超出部分作为溢出能量沉淀至账号能量余额。

右侧底部汇总区可轻量展示：`已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额`。“溢出 +Y%”仅在存在溢出能量时展示；不存在溢出时，仍展示原文案：`已选卡牌将在开启时消耗`。

“溢出 +Y%”不进入中间资源构成 icon 区，不展示在中间开启满足度区域，不展示在左侧资源概览中。示意图中的 `+18%`、`+68%`、`+95%` 均为状态示例，具体值以后端返回为准。本文档仅约束展示口径，不展开已有能量计算规则、实际扣减逻辑或资源消耗优先级。

### 5.3 资源参与口径

左侧资源概览展示账号资源总览；中间资源构成 icon 区只展示本次实际参与本次开启判断的资源。

| 路径 | 中间显示 icon | 中间不显示 |
|---|---|---|
| 纯赠品路径 | 赠品次数 ×N | 卡牌能量、扳手、VIP |
| 纯能量路径 | 卡牌能量 +X%、扳手 ×N、VIP权益 +X%（如有） | 赠品 ×0 |
| 混合路径 | 赠品次数 ×N、卡牌能量 +X%、扳手 ×N、VIP权益 +X%（如有） | 未参与资源 |
| 都没有资源参与 | icon 区不显示任何内容 | 全部资源 icon |

赠品 ×0 不展示。未参与资源不展示。不显示空占位。当仅有 1 个资源 icon 时，使用完整资源名称文案，例如：`赠品次数 ×1`。右侧底部的“溢出 +Y% 存入能量余额”不属于中间资源构成 icon 区内容。

### 5.4 保底模块口径

本期保底卡片仅作为当前军需箱的保底状态展示，不参与本次开启条件判断。

| 项目 | 口径 |
|---|---|
| 是否参与开启满足度计算 | 不参与 |
| 是否影响一键填充逻辑 | 不影响 |
| 是否影响开启军需按钮状态 | 不影响 |
| 是否进入资源构成 icon 区 | 不进入 |
| 是否直接影响 X1 / X5 切换 | 不直接影响 |
| 是否影响右侧卡牌背包 | 不影响 |
| 是否影响卡牌填充能量 | 不影响 |
| 是否影响资源消耗优先级 | 不影响 |
| 数据来源 | 以后端配置与接口返回为准 |

| 概念 | 口径 |
|---|---|
| 焦点区卡牌 | 面向用户的保底命中范围表达。研发口径中，焦点区卡牌对应当前箱子内带保底标记的指定范围卡牌 / 饰品。 |
| 保底范围饰品 | 研发 / 后端理解口径，表示当前箱子中被配置为保底命中范围的指定卡牌 / 饰品。 |
| 焦点区标记 | 用于标识当前箱子中属于本期保底命中范围的卡牌。 |
| 保底奖励标记 | 用于标识触发保底后可能发放的奖励范围。 |

焦点区卡牌与保底范围饰品指向同一组配置对象，不是两个奖励池。前台主表达统一使用「焦点区卡牌」，研发 / 后端口径可使用「保底范围饰品」。

焦点区标记与保底奖励标记指向同一批卡牌，但表达含义不同。前端按后端配置展示两类标记，不自行合并、不自行推导、不用单一标记替代两类标记，除非产品和 UI 另行确认。

| 类型 | 说明 |
|---|---|
| 自然命中焦点区 | 用户开启结果中获得任一焦点区卡牌，视为命中保底范围，本轮保底完成，进度重置，并进入新一轮，不额外触发保底奖励发放。 |
| 未命中焦点区 | 用户开启结果中未获得任一焦点区卡牌，本轮保底进度 +1。 |
| 触发保底 | 连续未命中焦点区达到保底门槛后，触发保底。奖励由后端按当前箱子的保底配置，在指定范围卡牌中随机发放 1 个奖励；本轮保底完成，进度重置，并进入新一轮。 |

自然命中、触发保底及新一轮进度等结果判断以后端结算返回为准，前端不自行推导。

保底奖励发放边界：

- 触发保底后不再发放军需箱，不再要求用户二次开启。
- 触发保底后的奖励在后端结算完成时即发放。
- 保底奖励发放后自动开启新一轮保底进度。
- 保底奖励发放不依赖前端结果页播放状态。
- 结果页仅作为告知展示，不作为奖励发放的触发节点。

批量开启说明：

- X5 等批量开启场景下，前端不逐次计算命中与进度变化。
- X5 等批量开启的保底最终展示以后端结算结果为准。
- 前端不自行推导跨轮保底结果。
- 批量开启中如同时发生自然命中焦点区、触发保底奖励、新一轮进度变化等结果，前端不自行判断展示优先级，最终结果态文案与新一轮进度以后端返回为准。

达到保底门槛后的展示说明：

- 达到保底门槛后的展示以最终结算结果为准。
- 不要求前台停留展示 `Y/Y` 过程态。
- 触发保底、自然命中焦点区等结果态用于本次开启后的结果反馈或刷新后的短期展示。
- 页面最终稳定态以结算后的新一轮保底进度为准。
- 结果态具体展示时长、消失方式、是否有过渡动效待确认。

### 5.5 X1 / X5 与赠品部分抵扣规则

X1 / X5 为开启次数选择。当选择 X5 且赠品次数不足 5 次时，赠品次数可计入本次开启满足度，剩余部分可通过卡牌能量等资源补足。

切换 X1 / X5 后：

- 已选卡牌清空。
- 开启满足度重新计算。
- 中间资源构成 icon 重新计算。
- 右侧状态重新判断。
- 右侧底部固定汇总区清空已选态与能量溢出提示。
- 本期保底卡片不因 X1 / X5 切换直接变化；保底进度仅在开启结算后按后端结果刷新。

### 5.6 已有功能边界

资源消耗优先级、实际扣减逻辑、能量计算规则、开启条件不足时的已有提示逻辑，均沿用现有开箱规则，本次优化不调整。

保底进度、目标进度、剩余开启次数、焦点区卡牌范围、焦点区标记、保底奖励标记、批量开启最终保底结果，均以后端配置与接口返回为准。

本次填充能量溢出值用于前台轻提示展示，具体计算和返回方式以后端或既有开启判断结果为准。本文档不重新定义能量计算、扣减顺序、余额沉淀规则或有效期规则。

### 6.3 本期保底卡片状态

默认状态：

```text

焦点区卡牌

```

| 状态 | 触发条件 | 前台展示文案 | 展示说明 |
|---|---|---|---|
| 未配置保底 | 当前军需箱未配置保底玩法 | 不展示 | 模块不展示、不占位。 |

| 临近完成态 | 距离保底触发较近，例如剩余 N 次 | 再开启 N 次触发 | 用于强化即将达成感，N 以后端返回或后端计算结果为准。 |

| 新一轮延续态 | 上一轮已完成，新一轮已开启 | 上一轮已完成，新一轮已开启 | 承接自然命中或触发保底后的重置说明，展示新一轮初始进度。 |

前台不使用“重置 / 清零”作为主表达。常态保持轻表达，完整规则放入 info。达到保底门槛后，不要求前台停留展示 `Y/Y` 过程态。自然命中焦点区或触发保底后，页面最终稳定态均回到常规累计态，展示后端返回的新一轮进度。

保底 info icon hover 后展示规则说明。建议 info 内容：

## 7. 右侧卡牌背包状态

### 7.1 半展开态

半展开态定义为当前状态摘要，只承载当前状态摘要与进入全展开态的入口，不承载具体选卡操作，不展示筛选、卡牌列表、底部固定汇总，不展示 `[0/3]`、`3/300` 等未确认计数，不展示“溢出 +Y% 存入能量余额”。

半展开展示内容只包括：

- 卡牌背包标题
- 当前摘要信息
- 当前状态说明
- 入口文案

有卡牌且赠品足够时展示：

```text
卡牌背包

共 N 张卡牌
赠品次数已满足，暂无需填充

查看背包 ›
```

有卡牌且能量已满时展示：

```text
卡牌背包

共 N 张卡牌
开启能量已满足，暂无需填充

查看背包 ›
```

有卡牌且已有填充时展示：

```text
卡牌背包

共 N 张卡牌
已选 N 张，填充 +X%

查看背包调整 ›
```

无卡牌时展示：

```text
卡牌背包

暂无可用卡牌

前往获取 ›
```

无卡牌状态指当前开箱流程中无可用于填充的卡牌，不等同于账号历史上没有任何卡牌。无卡牌摘要态展示空状态插图，插图样式以最终设计稿为准。有卡牌但当前无需填充时，不使用空状态插图，不显示“暂无可用卡牌”。

### 7.2 全展开态

全展开态定义为操作详情。展示结构：

```text
卡牌背包
选卡填充能量

[能量由高到低] [全部等级]

2 列卡牌列表

动态轻提示

底部固定汇总区
```

顶部区域展示 `卡牌背包` 与 `选卡填充能量`，保留半展开 / 全展开切换 icon，不展示 `[0/3]`、`3/300` 等未确认口径计数。

筛选区展示 `能量由高到低`、`全部等级`，不展示“清除”。“清除已选”仅在底部汇总区展示。

全展开态卡牌采用 2 列布局，用于快速多选填充，不作为卡牌详情页。每张卡展示稀有度 / 等级、卡牌名、卡牌图片、能量值、喜欢标识、选中勾选。全展开态仅卡牌列表区域可滚动；顶部标题 / 筛选区与底部固定汇总区不随列表滚动。

| 状态 | 视觉表现 | 说明 |
|---|---|---|
| 普通卡 · 未选 | 未选中态正常展示，需与已选中态有明确视觉区分。 | 点击直接选中，无需确认。 |
| 普通卡 · 已选 | 展示已选中状态标识，需与未选中态有明确视觉区分。 | 再次点击反选撤回。 |
| 当前操作项 | 选中瞬间需有可识别反馈。 | 稳定后回到已选中态。 |
| 喜欢卡 · 未选 | 常驻喜欢卡标识，正常展示，不弱化；具体 icon 样式以最终设计稿为准。 | 点击触发二次确认。 |
| 喜欢卡 · 已选 | 需要同时表达喜欢保护与已选中状态。 | 已确认后正常计入填充结果，可反选。 |
| 一键填充跳过喜欢卡 | 喜欢卡不自动选中。 | 展示动态轻提示说明已跳过。 |

卡牌同时具备喜欢保护与已选中状态时，需要同时表达两种状态。不允许因选中态覆盖而丢失喜欢保护表达。

喜欢卡轻提示文案：

```text
喜欢卡已保护，未自动填充
```

该提示不是常驻信息，仅在用户触发一键填充后，系统跳过喜欢卡时出现。如果未发生一键填充跳过喜欢卡，不展示该提示。该提示为轻提示，不作为右侧主要状态文案。

底部固定汇总区固定在右侧面板底部，不随卡牌列表滚动。

| 场景 | 底部文案 |
|---|---|
| 需要补能量，未选卡 | 已选 0 张｜选择卡牌后填充能量 |
| 赠品已满足，未选卡 | 赠品次数已满足，暂无需填充 |
| 能量已满足，未选卡 | 开启能量已满足，暂无需填充 |
| 已有选择 | 已选 N 张｜填充 +X%｜清除已选 |
| 已有选择 · 无溢出补充说明 | 已选卡牌将在开启时消耗 |
| 已有选择 · 有溢出补充说明 | 已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额 |

“填充 +X%”表示本次已选卡牌带来的新增能量，不包含左侧账号已有能量。“清除已选”仅在已选卡牌数量大于 0 时展示。“已选卡牌将在开启时消耗”仅在已选卡牌数量大于 0 且不存在能量溢出时展示。

当本次已选卡牌存在能量溢出时，底部补充说明展示“已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额”。“溢出 +Y%”表示本次开启消耗后沉淀至账号能量余额的部分。“溢出 +Y% 存入能量余额”为轻提示文案，不使用强按钮、弹窗或独立模块表达。

| 系统判断用户可通过一键填充补足并开启当前最小开启次数 | 右侧默认全展开 | 用于引导用户完成卡牌选择或一键填充。 |
| 其他非引导场景 | 右侧默认摘要模式 | 不默认展开完整卡牌列表。 |
| 点击查看背包 / 查看背包调整 | 右侧全展开 | 展示操作详情。 |
| 点击一键填充 | 自动选择可填充卡牌，并更新中间开启满足度与右侧选中结果 | 仅当本次一键填充结果满足“可开启最少次数”的引导条件时，右侧自动进入全展开态并展示已选卡牌；其他场景保持或回到摘要态。 |
| 一键填充后存在能量溢出 | 右侧全展开态底部固定汇总区展示溢出提示 | 展示“已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额”。 |
| 一键填充跳过喜欢卡 | 右侧展示轻提示 | 提示“喜欢卡已保护，未自动填充”。 |
| 切换 X1 / X5 | 已选卡牌清空，右侧和中间重新计算状态 | 已确认；底部汇总区与能量溢出提示同步清空。 |
| 完成开启结算 | 本期保底卡片根据后端最终结算结果刷新 | 仅配置保底玩法时适用。 |
| 离开页面 | 已选清空 | 选卡是当前会话预览，不持久化。 |

## 11. 验收口径

### 11.1 页面结构验收

- 页面应包含左侧资源概览、中间核心操作区、右侧卡牌背包三个主要区域。
- 左侧资源概览区域应包含：赠品开箱次数、能量余额、机械扳手、卡牌背包入口、会员等级。
- 中间核心区域应包含：军需箱主视觉、X1 / X5、开启满足度、资源构成 icon、一键填充、开启军需。
- 本期保底卡片应位于页面左下方区域。
- 本期保底卡片应以运营活动卡牌形式表达。
- 仅配置保底玩法的指定军需箱展示本期保底卡片。
- 未配置保底玩法时，不展示本期保底卡片，不保留空位。
- 右侧卡牌背包区域应支持半展开态和全展开态。
- 半展开和全展开应使用同一组件骨架，不应表现为两个不同组件。
- 全展开态应使用 2 列卡牌布局。
- 全展开态仅卡牌列表区域可滚动；顶部标题 / 筛选区与底部固定汇总区不随列表滚动。
- 能量溢出提示仅归属右侧卡牌背包全展开态底部固定汇总区，不新增独立模块。
- 下方卡牌 / 饰品区需要支持展示焦点区标记。
- 下方卡牌 / 饰品区需要支持展示保底奖励标记。

### 11.2 字段展示验收

- 左侧赠品开箱次数展示为 L1 主资源，可展示 0。
- 左侧能量余额展示为 L2 次主资源，仅展示账号已有能量。
- 左侧即将过期提示保留在能量余额卡片内。
- 左侧机械扳手、卡牌背包、会员等级展示为 L3 信息。
- 左侧会员等级不展示 info，不新增点击交互。
- 左侧不展示会员权益加成数值。
- 左侧卡牌背包展示为“卡牌背包 N张 ›”形式。
- 左侧“N张”表示用户当前拥有卡牌总数。
- 右侧半展开“共 N 张卡牌”表示用户当前拥有卡牌总数。
- 右侧全展开列表展示数量可受筛选、可用状态影响，不等同于用户拥有卡牌总数。
- 喜欢卡保护不影响右侧列表展示，仅影响一键填充自动选择逻辑。
- 中间开启满足度应展示百分比大字和进度条。
- 中间资源构成 icon 只展示本次实际参与资源。
- 中间「卡牌能量 +X%」以后端返回的本次计入满足度的卡牌能量值为准，前端不自行根据左侧与右侧相加推导。
- 中间未参与资源不展示，不显示 ×0。
- 本期保底卡片应展示前台标题“本期保底”。
- 本期保底卡片应展示范围标签“焦点区卡牌”。
- 常规累计态展示轻文案：`进度 X/Y`。
- 临近完成态展示：`再开启 N 次触发`。
- 自然命中焦点区态展示：`已获得焦点区卡牌，新一轮已开启`。
- 结算触发瞬时态展示：`奖励发放中...`。
- 新一轮延续态展示：`上一轮已完成，新一轮已开启`。
- 本期保底卡片应展示 info icon。
- Y ≤ 10 时展示分段进度条，Y > 10 时展示连续进度条。
- 固定奖励视觉资源仅用于活动卡片表达，不代表最终实际发放奖励。
- 保底结果态文案应以后端返回为准展示，不由前端自行推导。
- 右侧顶部展示收起 / 展开切换 icon，不展示 `[0/3]`、`3/300` 等未确认口径计数。
- 右侧半展开填充态必须展示“已选 N 张，填充 +X%”。
- 右侧半展开态不展示“溢出 +Y% 存入能量余额”。
- 右侧全展开已选态必须展示“已选 N 张｜填充 +X%｜清除已选”。
- 右侧全展开已有选择时，底部固定汇总区应展示“已选卡牌将在开启时消耗”。
- 当本次已选卡牌存在能量溢出时，右侧全展开底部固定汇总区应展示“已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额”。
- 当不存在能量溢出时，不展示“溢出 +Y% 存入能量余额”。
- “溢出 +Y%”表示本次开启消耗后沉淀至账号能量余额的部分。
- “溢出 +Y%”不应展示在中间开启满足度、资源构成 icon 区或左侧资源概览中。
- 右侧“填充 +X%”表示本次已选卡牌带来的新增能量，不包含左侧账号已有能量。
- 卡牌能量值选中态与未选中态需有明确视觉区分，具体颜色以最终设计稿为准。
- 下方卡牌 / 饰品区应按后端配置展示焦点区标记与保底奖励标记。

### 11.3 状态验收

- 有卡牌且赠品次数满足时，右侧半展开展示：`赠品次数已满足，暂无需填充`。
- 有卡牌且开启能量已满时，右侧半展开展示：`开启能量已满足，暂无需填充`。
- 有卡牌且已有填充时，右侧半展开展示：`已选 N 张，填充 +X%`。
- 无可用卡牌时，右侧半展开展示：`暂无可用卡牌` 和 `前往获取`。
- 无可用卡牌时，右侧半展开应展示空状态插图。
- 一键填充跳过喜欢卡时，右侧展示轻提示：`喜欢卡已保护，未自动填充`。
- 非一键填充跳过喜欢卡场景，不展示该提示。
- 喜欢卡提示不是默认常驻说明，示例图为一键填充后的提示态。
- 可填充但未满足时，一键填充为推荐态。
- 已满足开启条件时，开启军需为推荐态。
- 都没有资源时，中间资源构成 icon 区不展示任何内容。
- 都没有资源时，中间进度条下方不展示文案。
- 无可填充卡牌时，一键填充不作为当前推荐操作，但状态仍需可识别；具体视觉表现以最终设计稿为准。
- 当选择 X5 且赠品次数不足 5 次时，赠品次数可计入本次开启满足度，剩余部分可通过卡牌能量等资源补足。
- X1 / X5 切换后，开启满足度和资源构成 icon 应重新计算。
- X1 / X5 切换后，已选卡牌清空，右侧底部固定汇总区不再展示已选态、消耗说明或能量溢出提示。
- 已选卡牌且不存在能量溢出时，右侧全展开态底部展示“已选卡牌将在开启时消耗”。
- 已选卡牌且存在能量溢出时，右侧全展开态底部展示“已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额”。
- 清除已选后，右侧全展开态底部不展示“已选卡牌将在开启时消耗”或“溢出 +Y% 存入能量余额”。
- 未配置保底玩法时，不展示本期保底卡片。
- 常规累计态、临近完成态、自然命中焦点区态、结算触发态、新一轮延续态均需可识别。
- 本次开启未获得任一焦点区卡牌时，本轮保底进度 +1。
- 本次开启获得任一焦点区卡牌时，本轮保底完成，进度重置，并进入新一轮。
- 自然命中焦点区态不触发额外保底奖励。
- 连续未命中焦点区达到保底门槛时，由后端按当前箱子的保底配置，在指定范围卡牌中随机发放 1 个奖励。
- 触发保底后，本轮保底完成，进度重置，并进入新一轮。
- 批量开启时，保底最终展示以后端返回结果为准。
- 批量开启中如同时发生自然命中焦点区、触发保底奖励、新一轮进度变化等结果，前端不自行判断展示优先级。
- 结算触发态仅作为瞬时态，不作为长期稳定态。
- 结算触发态产品建议最多展示 2 秒；如后端结果返回较快，可不强制停满 2 秒。
- 新一轮开启后，展示后端返回的新一轮进度。

### 11.4 交互验收

- info 类统一 hover 触发。
- 左侧「卡牌背包 N张 ›」点击后，右侧应进入全展开态。
- 当右侧已处于全展开态时，再点击左侧「卡牌背包 N张 ›」，右侧应通过抖动或高亮等方式进行区域提示，具体方式待确认。
- X1 / X5 切换后，已选卡牌应清空，并重新计算开启满足度。
- X1 / X5 切换后，能量溢出提示应同步清空。
- 右侧「前往获取 ›」点击打开 CTA 下方建议弹窗。
- 普通卡点击后直接选中，并立即计入当前填充结果。
- 已选普通卡再次点击后反选撤回。
- 喜欢卡未选中时点击应触发二次确认，已有功能，本次不展开。
- 喜欢卡确认加入后正常计入填充结果。
- 一键填充时应跳过喜欢卡。
- 一键填充跳过喜欢卡时，右侧应展示轻提示。
- 一键填充后如存在能量溢出，右侧全展开态底部固定汇总区应展示溢出提示。
- 清除已选后，右侧已选状态清空。
- 清除已选后，右侧底部不再展示能量溢出提示。
- 开启条件满足时，点击开启军需执行开启。
- 开启条件不满足时，点击开启军需触发已有功能，本次不展开。
- 离开页面后，已选卡牌不持久化。
- 离开页面后，能量溢出提示不持久化。
- 即将过期能量提示 hover 后展示说明弹层，该弹层为后续迭代能力。
- 保底 info icon hover 后展示保底规则说明。
- 本期保底卡片本体不新增点击规则。
- 固定奖励视觉资源不新增点击规则。
- 进度条不新增点击规则。
- 焦点区标记不新增点击规则。
- 保底奖励标记不新增点击规则。

### 11.5 保底 info 验收

保底 info 至少包含以下说明：

```text
焦点区卡牌为本期保底范围。
未获得焦点区卡牌时，保底进度增加。
获得焦点区卡牌或触发保底后，本轮保底完成并开启新一轮。
触发保底后，将直接获得焦点区范围内的 1 个奖励。
```

info 内容用于前台规则说明，不替代后端实际结算规则。实际保底进度、门槛、奖励、范围以后端配置与接口返回为准。

### 11.6 标记验收

- 焦点区标记用于命中判断范围识别。
- 保底奖励标记用于触发保底后的奖励范围识别。
- 焦点区标记与保底奖励标记指向同一批卡牌，但表达含义不同。
- 前端不得将焦点区标记与保底奖励标记强行合并为同一标记。
- 前端不得只展示其中一个标记来替代另一个标记，除非产品和 UI 另行确认。
- 前端不得根据卡牌位置、稀有度、图片或名称自行判断是否属于焦点区或保底奖励范围。
- 标记展示以后端配置为准。

### 11.7 范围控制验收

- 不在左侧展示会员权益加成数值。
- 不在左侧会员等级上新增 info 或点击交互。
- 不把右侧卡牌背包做成资产管理区。
- 不在半展开态展示筛选、卡牌列表和底部固定汇总。
- 不在半展开态展示“溢出 +Y% 存入能量余额”。
- 不在右侧筛选区展示“清除”。
- 不在右侧顶部展示未确认口径的 `[0/3]`、`3/300` 等数量。
- 不在无确认情况下新增 loading、分页、权限、异常状态。
- 不在本次文档中重新定义资源消耗优先级、实际扣减逻辑、能量计算规则。
- 不因能量溢出提示新增独立能量沉淀模块、弹窗、领取按钮或结算前确认流程。
- 不把能量溢出提示放入中间开启满足度、资源构成 icon 区、本期保底卡片或左侧资源概览。
- 不新增累计返赠、任务、领取按钮、活动中心入口等未确认功能。
- 历史讨论中曾出现“累计返赠”命名，本次统一确认模块前台标题为“本期保底”，不使用“累计返赠”。
- 不展开资源概览、卡牌背包、底部掉落 Tab、CTA 按钮等已有功能规则。
- 不额外设计保底 loading、异常态、无权限态、按钮置灰态、未确认空状态。
- 保底常态保持轻表达，规则说明放入 info。
- 本期保底卡片不放回中间开启满足度区域。
- 本期保底卡片不参与开启满足度计算，不影响一键填充、开启军需按钮状态。
- 本期保底卡片不进入中间资源构成 icon 区。
- 不新增保底奖励领取按钮。
- 不新增保底奖励二次开箱流程。
- 不展示“获得军需箱后再开启”的旧规则表达。
- 不把固定设计资源当作实际发放奖励。
- 不把「焦点区卡牌」与「保底范围饰品」理解为两个奖励池。
- 不将「焦点区标记」与「保底奖励标记」混用。
- 能量溢出轻提示不影响一键填充、开启军需按钮状态。
- 对已有功能但本次未说明的内容，不展开流程和规则。
- UI 示意图仅作为辅助理解，不作为最终规则口径；最终以本文档为准。
