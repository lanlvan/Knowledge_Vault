测试草稿：仅用于人工评审对比。

## 首屏摘要

- 本次目标：优化研发阅读与执行效率，不新增页面规则。
- 本次变化类型：阅读与表达优化。
- 不变更范围：展示条件、按钮状态逻辑、资源计算口径、验收口径基线保持不变。
- 研发优先关注模块：保底进度模块、能量溢出提示、X1/X5 切换重算链路。
- 待确认项已按“阻塞开发启动 / 不阻塞开发启动但影响验收”分组。

## 1. 本次变更背景

当前开箱页规则已稳定，本稿用于把既有口径整理成更易执行的研发交接表达，降低实现与验收误读风险。

## 2. 本期范围

- 聚焦开箱页主流程：左侧资源概览、中部主判断区、右侧卡牌背包与保底信息表达。
- 仅调整交接文档阅读结构与模块呈现顺序。
- 不新增页面功能、不新增交互口径、不新增验收项。

## 3. 继承规则

- 开箱页沿用“三栏职责分离”：
  - 左侧：资源概览；
  - 中部：开启判断与主操作；
  - 右侧：选卡填充。
- 保底进度模块为辅助展示，不参与开启满足度计算。
- 能量溢出提示仅在右侧全展开底部固定汇总区展示。
- 页面最终展示以后端返回结果为准。

## 4. 本次变更点

- 新增首屏摘要，研发可在短时间内识别重点与边界。
- 高影响模块改为“最小实现口径”格式，避免只写关注点。
- 待确认项按阻塞程度分层，便于排期与风险管理。
- 保留不变更范围，降低“新需求误判”。

## 5. 不变更范围

- 不变更页面结构定义；
- 不变更展示条件；
- 不变更状态文案口径；
- 不变更资源参与与满足度计算边界；
- 不变更既有验收基线。

## 6. 研发影响（高影响模块最小实现口径）

### 6.1 保底进度模块

- 模块职责：在中部展示当前保底进度与规则说明，服务于开箱理解，不主导主操作状态。
- 已确认展示口径：仅在配置保底玩法时展示；未配置时不展示且不占位。
- 关键状态 / 交互：累计中、获得保底范围饰品、触发保底、批量开启跨门槛四类结果态。
- 研发影响侧：前后端（后端提供最终结果，前端负责状态映射与展示）。
- 待确认阻塞点：无阻塞开发启动的新增点。
- 验收关注点：模块不参与满足度计算；不影响一键填充与开启按钮主状态。

### 6.2 能量溢出提示

- 模块职责：提示已选卡牌能量超出本次补足需求后的沉淀结果。
- 已确认展示口径：仅在右侧全展开底部固定汇总区展示；无溢出时不展示。
- 关键状态 / 交互：随选卡、反选、清除已选、X1/X5 切换触发刷新或清空。
- 研发影响侧：前后端（展示依赖口径、刷新依赖事件链）。
- 待确认阻塞点：溢出值来源、刷新时机、与最终沉淀一致性口径。
- 验收关注点：不得进入中部满足度区、资源构成区或左侧资源概览。

### 6.3 X1 / X5 切换与状态重算

- 模块职责：切换开启次数并触发本次开箱状态重算。
- 已确认展示口径：切换后清空已选卡牌；满足度与资源构成重算；底部溢出提示同步清空。
- 关键状态 / 交互：X1 与 X5 切换后统一走清空与重算链路。
- 研发影响侧：前端（状态机与 UI 同步）、后端（返回值口径稳定性）。
- 待确认阻塞点：无新增阻塞项。
- 验收关注点：切换后状态应一致回收，不遗留已选态与溢出提示。

## 7. 待确认项

### A. 阻塞开发启动

1. 能量溢出值来源（后端直接返回或前端计算口径）  
2. 能量溢出值刷新时机（选卡变化、X1/X5 切换、清除已选、开启后）  
3. 溢出展示值与最终能量余额沉淀结果一致性口径  
4. 需产品确认优先级：左侧入口在右侧已全展开时反馈策略（抖动 / 高亮）是否阻塞当前迭代启动

### B. 不阻塞开发启动，但影响验收

1. 能量溢出值展示精度（取整 / 小数位 / 百分比规则）  
2. 右侧收起/展开 icon 方向、动效、样式  
3. 喜欢卡与选中态并存时 icon 排布、层级、避让  
4. 保底标记 icon 视觉样式  
5. 即将过期能量 hover 弹层内容、字段、样式、触发延迟  
6. 保底结果态展示时长、消失方式、过渡动效  
7. 保底 info 的位置、宽度、样式与动效细节

## 8. 验收口径

- 三栏职责清晰：左概览、中判断、右选卡；
- 保底模块仅作为辅助展示，不参与满足度；
- 能量溢出提示只在右侧全展开底部固定汇总区；
- X1/X5 切换后，已选态与溢出提示同步清空；
- 待确认项在评审前保持待确认状态，不按已确认实现。
