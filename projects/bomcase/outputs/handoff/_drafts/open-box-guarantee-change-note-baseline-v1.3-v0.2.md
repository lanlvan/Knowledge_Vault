## 2. 本次变化摘要

本次变化聚焦 Open Box 页面保底模块：保底模块不再放在中间核心区作为保底进度模块表达，而是调整为页面左下方「本期保底」运营活动卡牌。

| 关注项 | 结论 |
|---|---|
| 本次变化重点 | 位置、前台表达、触发后奖励发放链路、焦点区范围与两类标记关系发生调整。 |
| 是否影响前端 | 影响。前端需要按服务端配置和返回展示本期保底卡片、状态、文案、范围和两类标记，不自行推导保底结果。 |
| 是否影响后端 / 服务端 | 影响协作边界。服务端需要返回是否配置保底玩法、进度、剩余次数、焦点区范围、两类标记、开启后结算结果和批量开启最终结果。 |
| 是否影响 UI / 设计 | 影响。本期保底卡片、info、固定奖励视觉资源、焦点区标记和保底奖励标记存在视觉承接边界。 |
| 是否影响测试 | 影响。需要覆盖展示条件、状态文案、直接发奖链路、批量开启、两类标记和范围控制。 |
| 是否阻塞开发启动 | 当前未发现阻塞开发启动的事实层缺口；模块相关后续补充边界见第 10 节。 |

“历史背景输入”仅用于帮助理解本次变化，不作为当前事实源；开发、联调与验收以“当前确认口径”为准。

| 项目 | 历史背景输入（非当前事实源） | 当前确认口径（以此为准） | 研发影响 |
|---|---|---|---|

| 前台标题 | 原表达中曾存在“保底进度”标题和保底奖励名称相关表达。 | 前台标题为「本期保底」。 | 不沿用旧标题或旧奖励名称表达。 |

## 4. 当前模块口径

### 4.1 模块位置

| 项目 | 当前口径 |
|---|---|
| 区域名称 | 保底模块 / 本期保底卡片 |
| 区域位置 | 页面左下方区域 |
| 展示形式 | 运营活动卡牌 |
| 已确认说明 | 常态展示本期保底状态、焦点区卡牌范围、累计进度与结果状态；完整规则放入 info。 |

### 4.2 前台表达

| 项目 | 当前口径 |
|---|---|
| 前台标题 | 本期保底 |
| 范围标签 | 焦点区卡牌 |
| 包含元素 | 固定奖励视觉资源、模块标题、范围标签、info icon、进度文案、进度条、状态文案 |
| 固定奖励视觉资源 | 仅用于运营活动卡牌表达，不代表最终实际发放奖励，不根据实际奖励内容动态替换。 |

### 4.3 展示条件

| 场景 | 当前口径 |
|---|---|
| 当前军需箱配置保底玩法 | 展示页面左下方「本期保底」卡片。 |
| 当前军需箱未配置保底玩法 | 不展示本期保底卡片，不占位。 |
| 保底模块空状态 | 本次未确认保底模块空状态；除“不展示、不占位”外，不额外设计空状态。 |

### 4.4 数据 / 后端边界

- 保底进度、目标进度、剩余开启次数、焦点区卡牌范围、焦点区标记、保底奖励标记、批量开启最终保底结果，均以服务端配置与接口返回为准。
- 自然命中、触发保底及新一轮进度等结果判断以服务端结算返回为准，前端不自行推导。
- 触发保底后不再发放军需箱，不再要求用户二次开启。
- 触发保底后的奖励在后端结算完成时即发放。
- 保底奖励发放后自动开启新一轮保底进度。
- 保底奖励发放不依赖前端结果页播放状态；结果页仅作为告知展示，不作为奖励发放的触发节点。
- X5 等批量开启场景下，前端不逐次计算命中与进度变化，最终展示以服务端返回结果为准。

### 4.5 前端展示边界

- 前端按服务端配置展示本期保底卡片、焦点区范围、进度、状态和两类标记。
- 前端不通过左侧账号资源与右侧已选卡牌自行推导中间开启满足度、资源构成 icon 或保底结果。
- 前端不得根据卡牌位置、稀有度、图片或名称自行判断是否属于焦点区或保底奖励范围。
- 前端不得将焦点区标记与保底奖励标记强行合并为同一标记。
- 前端不得只展示其中一个标记来替代另一个标记，除非产品和 UI 另行确认。

### 4.6 状态边界

| 结果类型 | 当前口径 |
|---|---|
| 自然命中焦点区 | 用户开启结果中获得任一焦点区卡牌，视为命中保底范围，本轮保底完成，进度重置，并进入新一轮，不额外触发保底奖励发放。 |
| 未命中焦点区 | 用户开启结果中未获得任一焦点区卡牌，本轮保底进度 +1。 |
| 触发保底 | 连续未命中焦点区达到保底门槛后，触发保底；奖励由后端按当前箱子的保底配置，在指定范围卡牌中随机发放 1 个奖励；本轮保底完成，进度重置，并进入新一轮。 |
| 批量开启 | 如同时发生自然命中焦点区、触发保底奖励、新一轮进度变化等结果，前端不自行判断展示优先级。 |
| 结算触发态 | 仅作为瞬时态，不作为长期稳定态；产品建议最多展示 2 秒，如后端结果返回较快，可不强制停满 2 秒。 |

### 4.7 不覆盖内容

- 本期保底卡片不参与开启满足度计算。
- 不影响一键填充逻辑。
- 不影响开启军需按钮状态。
- 不进入中间资源构成 icon 区。
- 不直接影响 X1 / X5 切换。
- 不影响右侧卡牌背包。
- 不定义接口字段、数据表、服务拆分和后端计算方式。
- 不定义后端奖励随机算法、保底配置后台流程、奖励库存、资产到账、异常补偿或发放失败规则。

## 6. 状态与文案

| 状态 | 触发条件 | 前台文案 | 说明 |
|---|---|---|---|
| 未配置保底玩法 | 当前军需箱未配置保底玩法 | 不展示 | 不占位；不额外设计空状态。 |
| 常规累计态 | 配置保底玩法且处于常规累计 | 进度 X/Y | X 表示当前本轮已累计次数，Y 表示当前保底门槛，展示以服务端返回为准。 |
| 临近完成态 | 服务端返回剩余开启次数 | 再开启 N 次触发 | N 以服务端返回或服务端计算结果为准。 |
| 自然命中焦点区态 | 本次开启获得任一焦点区卡牌 | 已获得焦点区卡牌，新一轮已开启 | 不额外触发保底奖励发放。 |
| 结算触发瞬时态 | 连续未命中达到保底门槛并触发保底 | 奖励发放中... | 瞬时态；奖励在后端结算完成时即发放。 |
| 新一轮延续态 | 本轮完成后进入新一轮 | 上一轮已完成，新一轮已开启 | 新一轮开启后，展示服务端返回的新一轮进度。 |
| info | hover 保底 info icon | 焦点区卡牌为本期保底范围。<br>未获得焦点区卡牌时，保底进度增加。<br>获得焦点区卡牌或触发保底后，本轮保底完成并开启新一轮。<br>触发保底后，将直接获得焦点区范围内的 1 个奖励。 | info 内容用于前台规则说明，不替代后端实际结算规则；实际保底进度、门槛、奖励、范围以服务端配置与接口返回为准。 |

进度条展示规则：Y <= 10 时展示分段进度条；Y > 10 时展示连续进度条。
