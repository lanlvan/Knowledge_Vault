- 开箱页承接军需开启主流程，核心为三栏职责稳定：左侧资源概览、中间主判断与主操作、右侧选卡填充。

- 批量开启、保底进度、奖励结果、最终状态以服务端返回为准。

## 7. 核心页面口径【本次延续】

### 7.1 关键概念定义

| 概念 | 主表达 | 研发侧理解 | 关系说明 |
|---|---|---|---|
| 焦点区卡牌 | 焦点区卡牌 | 保底范围饰品 | 指向同一组配置对象，不是两个奖励池 |
| 焦点区标记 | 焦点区标记 | 命中范围标识 | 用于标识保底命中范围 |
| 保底奖励标记 | 保底奖励标记 | 触发保底后奖励范围标识 | 与焦点区标记可同指一批卡牌，但含义不同 |

### 7.2 研发边界

- 前端不自行推导焦点区命中、保底触发、新一轮进度或批量开启跨轮结果。
- 中间开启满足度与资源构成展示以服务端返回为准。
- 产品定义页面口径与验收边界，不定义接口字段、表结构和服务拆分。
- 最终页面展示以后端确认范围与服务端返回结果为准。

### 7.3 能量口径

| 位置 | 口径 |
|---|---|
| 左侧能量余额 | 账号已有能量 |
| 右侧填充 `+X%` | 本次已选卡牌新增能量，不含左侧已有能量 |
| 右侧溢出 `+Y%` | 超出当前开启所需补足能量的部分，开启后沉淀至账号能量余额 |
| 中间 `卡牌能量 +X%` | 本次计入满足度的卡牌能量值，以服务端返回为准 |

- `溢出 +Y%` 仅在右侧全展开底部固定汇总区展示。
- `溢出 +Y%` 不进入左侧资源概览、中间满足度区、中间资源 icon、本期保底卡片。
- 本文档不扩展能量计算公式、扣减顺序与有效期系统规则。

### 7.4 保底模块口径

| 项目 | 口径 |
|---|---|
| 是否参与开启满足度计算 | 不参与 |
| 是否影响一键填充 / 开启军需按钮状态 | 不影响 |
| 是否进入中间资源构成 icon 区 | 不进入 |
| 展示条件 | 仅配置保底玩法的指定军需箱展示 |
| 未配置处理 | 不展示、不占位 |
| 数据依据 | 后端配置与服务端返回 |

- 自然命中焦点区：本轮保底完成并进入新一轮，不额外触发保底奖励发放。
- 触发保底：后端按当前箱子保底配置，在指定范围卡牌中随机发放 1 个奖励，随后进入新一轮。
- 触发保底后不发放军需箱、不要求用户二次开启，结果页仅作告知展示。

### 7.5 X1 / X5 切换口径

- X1/X5 切换后：清空已选卡牌、重算满足度、重算中间资源 icon、重置右侧状态、清空溢出提示。
- 本期保底卡片不因 X1/X5 切换直接变化；保底进度在开启结算后按服务端返回刷新。

- 多结果并发（自然命中、触发保底、新一轮进度变化）时，前端不自行排优先级，以返回结果为准。
- 焦点区标记与保底奖励标记不新增点击或 hover 交互。

## 12. 验收口径【本次延续】

### 12.1 页面结构验收

- 页面包含左侧资源概览、中间核心区、右侧卡牌背包三个主区域。
- 本期保底卡片位于页面左下方，未配置保底时不展示不占位。
- 右侧支持半展开与全展开，且为同一组件骨架。
- 全展开为 2 列卡牌布局；仅列表区滚动，顶部与底部固定。

### 12.2 字段与文案验收

- 左侧赠品可展示 0，能量余额仅表示账号已有能量。
- 右侧 `填充 +X%` 表示本次新增能量；中间 `卡牌能量 +X%` 以服务端返回为准。
- 已确认用户可见文案完整覆盖：按钮文案、状态文案、底部汇总文案、info/hover 文案、轻提示文案、空状态文案。
- 本期保底文案覆盖：`进度 X/Y`、`再开启 N 次触发`、`已获得焦点区卡牌，新一轮已开启`、`奖励发放中...`、`上一轮已完成，新一轮已开启`。

### 12.3 状态与交互验收

- X1/X5 切换后必须清空已选并重算满足度，同时清空溢出提示。
- 半展开不展示溢出提示；全展开已选且有溢出时展示 `溢出 +Y%` 文案。
- 一键填充跳过喜欢卡时展示轻提示；非该场景不展示。
- 开启条件满足可开启；不满足时触发已有功能。

### 12.4 服务端返回与范围控制验收

- 保底进度、自然命中、触发保底、批量开启结果均以服务端返回为准。
- 前端不得自行推导跨轮保底结果、展示优先级或奖励发放时机。
- `溢出 +Y%` 不得进入左侧、中间或本期保底卡片。
- 焦点区标记与保底奖励标记不得混用或以单一标记替代。

### 12.5 不作为已确认规则的项

- 第 11 章后续补充边界仅影响 UI 表达、交互细节和验收细节，未补充前不作为固定实现与验收依据。

## 13. 不覆盖范围【本次延续】

- 不扩展完整能量计算、资源扣减、资产入账、奖励发放、库存、异常补偿规则。
- 不定义后端随机算法、保底后台配置流程、跨系统结算机制。
- 不新增保底奖励领取按钮、二次开启流程、奖励详情页。
- 不展开顶部导航、掉落信息 Tab 等既有功能内部规则。
- 不把示例值、示意图、后续补充边界写成固定系统规则。
