## 2. One-screen Summary

本次变化聚焦 Open Box 页面左下方「本期保底」运营活动卡牌。保底模块不再作为中间核心区的保底进度模块表达，也不参与开启满足度、一键填充、开启军需按钮状态或中间资源构成 icon 区。

研发需要关注：前台固定展示标题「本期保底」与范围标签「焦点区卡牌」；是否展示、进度、剩余次数、焦点区范围、两类标记、批量开启后的最终保底结果均以后端配置与接口返回为准；触发保底后直接由后端按当前箱子的保底配置，在指定范围卡牌中随机发放 1 个奖励，不再发放军需箱，也不要求用户二次开启。

当前 active source 已覆盖本模块变更，未发现阻塞研发启动的事实层缺口。

## 4. Current Confirmed Rule

### Module Position

- 模块名称为「保底模块 / 本期保底卡片」。
- 区域位置为页面左下方区域。
- 展示形式为运营活动卡牌。
- 仅配置保底玩法的指定军需箱展示；未配置保底玩法的军需箱不展示该模块，不占位。

### User-facing Expression

- 前台标题：`本期保底`。
- 范围标签：`焦点区卡牌`。
- 包含元素：固定奖励视觉资源、模块标题、范围标签、info icon、进度文案、进度条、状态文案。
- 固定奖励视觉资源仅用于运营活动卡牌表达，不代表最终实际发放奖励，不根据实际奖励内容动态替换。

### Data / Backend Boundary

- 保底进度、目标进度、剩余开启次数、焦点区卡牌范围、焦点区标记、保底奖励标记、批量开启最终保底结果，均以后端配置与接口返回为准。
- 自然命中焦点区：用户开启结果中获得任一焦点区卡牌，视为命中保底范围，本轮保底完成，进度重置，并进入新一轮，不额外触发保底奖励发放。
- 未命中焦点区：用户开启结果中未获得任一焦点区卡牌，本轮保底进度 +1。
- 触发保底：连续未命中焦点区达到保底门槛后，由后端按当前箱子的保底配置，在指定范围卡牌中随机发放 1 个奖励；本轮保底完成，进度重置，并进入新一轮。
- 触发保底后不再发放军需箱，不再要求用户二次开启；奖励在后端结算完成时即发放，不依赖前端结果页播放状态，结果页仅作为告知展示。

### Frontend Display Boundary

- 前端按后端配置展示保底卡片、进度、状态、焦点区范围与两类标记。
- 前端不自行推导焦点区命中、保底触发、保底奖励、新一轮进度或批量开启跨轮结果。
- 前端不得根据卡牌位置、稀有度、图片或名称自行判断是否属于焦点区或保底奖励范围。

### Interaction Boundary

- 保底 info icon hover 后展示保底规则说明。
- 本期保底卡片本体不新增点击规则。
- 固定奖励视觉资源不新增点击规则。
- 进度条不新增点击规则。
- 焦点区标记不新增点击规则。
- 保底奖励标记不新增点击规则。

### State Boundary

- 常规累计态、临近完成态、自然命中焦点区态、结算触发态、新一轮延续态均需可识别。
- 结算触发态仅作为瞬时态，不作为长期稳定态。
- 批量开启时，保底最终展示以后端返回结果为准；如同时发生自然命中焦点区、触发保底奖励、新一轮进度变化等结果，前端不自行判断展示优先级。

### Non-goals

- 本期保底卡片不参与开启满足度计算。
- 不影响一键填充逻辑。
- 不影响开启军需按钮状态。
- 不进入中间资源构成 icon 区。
- 不直接影响 X1 / X5 切换。
- 不影响右侧卡牌背包。
