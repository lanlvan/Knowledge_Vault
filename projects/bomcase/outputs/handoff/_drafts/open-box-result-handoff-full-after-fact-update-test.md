# Open Box Result 研发交接文档

## 1. 文档说明

本文档面向研发、UI 与测试，用于交接 Open Box Result / 开箱结果页的当前实现口径。

本版本覆盖开箱完成后的结果反馈页：展示本次结算后用户实际获得的全部饰品，呈现本次结算后的最终保底状态，并在满足条件时展示水晶能量沉淀提示。

本文档只描述结果页展示、状态、交互、边界与验收口径；完整保底计算、奖励发放链路、资产入账、钱包、交易、发货、接口字段、数据库字段与后端服务拆分不在本文档中定义。

## 2. 版本更新说明

本版本已采用新的保底奖励展示口径：

| 项目 | 当前口径 |
|---|---|
| 保底奖励发放 | 触发保底后，系统按当前箱子的保底配置直接发放焦点区范围内 1 个奖励饰品 |
| 保底奖励展示 | 保底奖励饰品直接进入“本次获得饰品区” |
| 展示顺序 | 以后端返回的最终结算顺序为准 |
| 前端处理 | 前端不自行推导保底奖励饰品插入位置，不做本地重排、拆分列表或抽出展示 |
| 保底奖励标识 | 仅系统额外发放的保底奖励饰品展示“保底奖励”标识 |
| 自然获得焦点区卡牌 | 展示顶部状态时，该自然获得饰品不展示“保底奖励”标识 |

## 3. 页面目标

Open Box Result 页面用于在后端完成本次开启结算后，向用户反馈本次实际获得结果。

页面目标包括：

| 目标 | 说明 |
|---|---|
| 展示本次实际获得饰品 | 以“本次获得饰品区”为核心，展示本次结算后用户实际获得的全部饰品 |
| 展示最终保底状态 | 顶部仅展示本次结算后的最新 / 最高优先级保底状态 |
| 展示水晶沉淀结果 | 仅当本次存在多余能量转为水晶时，低权重展示沉淀提示 |
| 承接后端结算 | 焦点区命中、保底触发、奖励发放、批量开启最终结果与展示顺序均以后端最终结算结果为准 |

## 4. 页面结构

|---|---|

| 主内容 | 本次获得饰品区 |
| 既有功能 | 饰品卡片、返回按钮、再来 5 次按钮、关闭按钮为已有功能，本次不展开 |

### 4.2 顶部标题区

| 项目 | 说明 |
|---|---|
| 区域位置 | 结果页顶部居中 |

| 展示条件 | 仅当本次结果需要展示保底状态反馈时展示 |
| 不展示情况 | 普通保底进度 +1 且未达到“距离保底门槛仅差 1 次”条件时，不展示顶部保底反馈 |

### 4.3 本次获得饰品区

|---|---|
| 区域位置 | 顶部标题区下方，中部主视觉区域 |
| 包含元素 | 本次开启获得的饰品卡片 |
| 展示内容 | 本次开启结算后用户实际获得的全部饰品，包括正常开箱掉落饰品与触发保底后额外发放的保底奖励饰品 |
| 排序规则 | 按后端返回的最终结算顺序展示 |
| 前端边界 | 前端不自行推导保底奖励饰品插入位置 |
| 标识规则 | 仅触发保底后由系统额外发放的保底奖励饰品展示“保底奖励”标识 |
| 不展示标识情况 | 用户自然获得焦点区卡牌时，该自然获得饰品不展示“保底奖励”标识 |
| 既有内容 | 饰品卡片样式、品质色条、磨损信息、附赠物角标为已有功能，本次不展开 |

### 4.4 底部按钮区

| 项目 | 说明 |
|---|---|
| 区域位置 | 结果页底部 |
| 包含元素 | 返回按钮、再来 5 次按钮 |
| “剩余次数”语义 | “剩余次数 51 次”表示当前拥有可再次开启的次数 |
| 按钮点击 | 已有功能，本次不展开 |
| 保底相关边界 | 底部按钮区不新增保底进度文案，避免与剩余开启次数混淆 |

### 4.5 水晶能量沉淀提示

| 项目 | 说明 |
|---|---|
| 区域位置 | 底部按钮区下方或附近，低权重展示 |
| 包含元素 | 提示文案 |
| 展示条件 | 仅当本次能量开启存在多余能量转为水晶时展示 |
| 展示范围 | 只说明本次多余能量沉淀结果，不承接完整资源概览 |
| 不展示情况 | 本次没有多余能量沉淀时不展示 |
| 示意图边界 | 设计示意图中的该提示仅用于示意位置，实际按条件展示 |

## 5. 字段与数据依赖

| 区域 | 展示值 / 字段 | 说明 |
|---|---|---|
| 顶部标题区 | 恭喜获得 | 结果页标题，表示本次开启结果已生成 |
| 顶部保底反馈 | 保底已触发 | 触发保底时的顶部状态信号，仅表达本次发生保底事件 |
| 顶部保底反馈 | 已获得焦点区卡牌，新一轮已开启 | 本次自然获得焦点区卡牌后展示；该自然获得饰品不展示“保底奖励”标识 |
| 顶部保底反馈 | 保底进度已更新，当前 X/Y | 本次结算后距离保底门槛仅差 1 次时展示；X/Y 以后端最终结算结果为准 |
| 本次获得饰品区 | 饰品卡片列表 | 展示本次结算后用户实际获得的全部饰品，包括正常开箱掉落饰品与触发保底后额外发放的保底奖励饰品 |
| 本次获得饰品区 | 保底奖励标识 | 仅触发保底后由系统额外发放的保底奖励饰品展示；自然获得焦点区卡牌不展示 |
| 本次获得饰品区 | +3 扳手 | 当前军需掉落饰品的附赠物角标；为已有功能，本次仅说明其语义 |
| 本次获得饰品区 | 展示顺序 | 以后端返回的最终结算顺序为准；前端不自行推导保底奖励饰品插入位置 |
| 底部按钮区 | 再来 5 次（剩余次数 51 次） | “剩余次数”表示当前拥有可再次开启的次数 |
| 水晶能量沉淀提示 | 多余能量已转为水晶，下次开启本箱可继续使用 | 仅当本次能量开启存在多余能量沉淀时展示；“本箱”指当前军需箱 |

## 6. 状态与展示规则

### 6.1 默认展示规则

结果页默认展示“本次获得饰品区”。

| 内容 | 默认是否展示 | 说明 |
|---|---|---|
| 顶部保底反馈 | 否 | 仅在触发保底、自然获得焦点区卡牌并进入新一轮、距离保底门槛仅差 1 次时展示 |
| 本次获得饰品区保底奖励标识 | 否 | 仅触发保底后由系统额外发放的保底奖励饰品展示 |
| 水晶能量沉淀提示 | 否 | 仅本次存在多余能量转为水晶时展示 |

### 6.2 顶部保底反馈规则

| 项目 | 规则 |
|---|---|
| 展示位置 | “恭喜获得”标题下方 |
| 信息职责 | 仅作为本次最终保底状态信号 |
| 文案长度 | 保持轻量，不展示奖励详情 |
| 信息优先级 | 触发保底 > 自然获得焦点区卡牌并进入新一轮 > 距离保底门槛仅差 1 次 |
| 最终状态规则 | 结果页只展示本次结算后的最新 / 最高优先级状态，不同时展示多个保底过程状态 |
| info icon | 结果页不展示保底 info icon |
| 普通进度更新 | 未达到“距离保底门槛仅差 1 次”条件时，不展示顶部反馈 |

### 6.3 本次获得饰品区规则

| 项目 | 规则 |
|---|---|
| 饰品卡片 | 展示本次结算后用户实际获得的全部饰品 |
| 正常掉落饰品 | 作为本次获得饰品区的一部分展示 |
| 保底奖励饰品 | 触发保底后由系统额外发放，直接并入本次获得饰品区 |
| 展示顺序 | 以后端返回的最终结算顺序为准；不做本地重排、拆分列表或抽出展示 |
| 保底奖励标识 | 仅系统额外发放的保底奖励饰品展示 |
| 自然获得焦点区卡牌 | 不展示“保底奖励”标识 |
| +3 扳手 | 表示当前军需掉落饰品的附赠物，保持在饰品卡片内展示 |
| 焦点区标记关系 | 焦点区标记与保底奖励标记指向同一批卡牌但含义不同，前端按后端配置展示，不自行合并或推导 |

### 6.4 水晶能量沉淀提示规则

| 项目 | 规则 |
|---|---|
| 展示条件 | 仅当本次能量开启存在多余能量转为水晶时展示 |
| 展示位置 | 底部按钮区下方或附近 |
| 视觉权重 | 低于本次获得饰品区和保底反馈 |
| 图标规则 | 不使用 VIP 图标，避免误解为 VIP 专属权益 |
| 展示内容 | 只说明本次多余能量沉淀结果，不展示完整水晶余额 |
| 状态关系 | 水晶能量沉淀提示与保底状态无关，任何状态下均按是否存在多余能量沉淀独立判断 |
| 示意说明 | 设计示意图中的水晶提示仅用于示意位置，实际按条件展示 |

### 6.5 主次信息层级

| 层级 | 信息 | 说明 |
|---|---|---|
| L1 | 本次获得饰品 | 结果页核心内容 |
| L2 | 顶部保底反馈与保底奖励标识 | 仅触发或关键状态时展示 |
| L3 | 底部操作按钮 | 返回 / 再来 5 次 |
| L4 | 水晶能量沉淀提示 | 低权重资源结算提示，仅条件展示 |
| L5 | 已有附赠物角标 | 饰品卡片内已有信息 |

## 7. 状态文案

| 状态 | 触发条件 | 前台展示文案 | 展示说明 |
|---|---|---|---|
| 未配置保底 | 当前军需箱未配置保底玩法 | 不展示 | 不展示顶部保底反馈，不展示保底奖励标识 |
| 普通进度更新 | 本次未获得焦点区卡牌，保底进度更新，但未达到“距离保底门槛仅差 1 次”条件 | 不展示 | 普通进度更新不在结果页顶部展示，返回开箱页后由保底进度模块展示最新进度 |
| 临近保底节点 | 本次未获得焦点区卡牌，结算后距离保底门槛仅差 1 次 | 保底进度已更新，当前 X/Y | X/Y 以后端最终结算结果为准；普通进度更新未达到该条件时不展示顶部反馈 |
| 自然获得焦点区卡牌 | 本次自然获得焦点区卡牌，保底进度进入新一轮 | 已获得焦点区卡牌，新一轮已开启 | 自然获得的焦点区饰品不展示“保底奖励”标识 |
| 触发保底 | 本次达到保底门槛，并由系统额外发放保底奖励饰品 | 保底已触发 | 顶部仅展示事件信号；保底奖励饰品并入本次获得饰品区，并展示“保底奖励”标识 |
| 批量开启未触发保底 | X5 等批量开启后未触发保底，且未达到“距离保底门槛仅差 1 次”条件 | 不展示 | 不展示顶部保底反馈，不展示保底奖励标识 |
| 批量开启达到临近保底节点 | X5 等批量开启后最终进度距离保底门槛仅差 1 次 | 保底进度已更新，当前 X/Y | X/Y 以后端最终结算结果为准，前端不自行推导 +N |
| 批量开启触发保底 | X5 等批量开启过程中触发保底 | 保底已触发 | 如批量开启中先自然获得焦点区卡牌、后触发保底，顶部最终只展示“保底已触发”；饰品顺序以后端返回为准 |
| 多余能量沉淀 | 本次能量开启存在多余能量转为水晶 | 多余能量已转为水晶，下次开启本箱可继续使用 | 底部低权重提示；仅发生沉淀时展示，不常驻；与保底状态无关 |
| 无多余能量沉淀 | 本次没有多余能量转为水晶 | 不展示 | 不展示水晶能量沉淀提示 |

前台状态文案不得使用“清零 / 重置”表达。

## 8. 交互规则

### 8.1 结果页顶部保底反馈

| 项目 | 说明 |
|---|---|
| 触发对象 | 顶部保底反馈文案 |
| 触发条件 | 本次结果满足触发保底、自然获得焦点区卡牌并进入新一轮、距离保底门槛仅差 1 次之一 |
| 展示结果 | 在“恭喜获得”标题下方展示一行轻量状态文案 |
| 优先级 | 触发保底 > 自然获得焦点区卡牌并进入新一轮 > 距离保底门槛仅差 1 次 |
| 是否所有状态适用 | 否 |
| 不适用处理 | 普通进度更新、未配置保底、未达到“距离保底门槛仅差 1 次”条件时不展示 |

### 8.2 本次获得饰品区展示

| 项目 | 说明 |
|---|---|
| 触发对象 | 本次获得饰品区 |
| 触发条件 | 本次开启结算完成 |
| 展示结果 | 展示后端返回的本次实际获得饰品列表 |
| 保底奖励饰品处理 | 触发保底后额外发放的保底奖励饰品并入本次获得饰品区展示 |
| 排序规则 | 最终展示顺序以后端返回的结算顺序为准 |
| 不适用处理 | 不单独展示额外奖励区域，不展示箱体奖励，不要求二次开启 |

### 8.3 保底奖励标识

| 项目 | 说明 |
|---|---|
| 触发对象 | 饰品卡片上的“保底奖励”标识 |
| 触发条件 | 该饰品为触发保底后由系统额外发放的保底奖励饰品 |
| 展示结果 | 该饰品卡片展示“保底奖励”标识 |
| 不展示情况 | 用户自然获得焦点区卡牌时，该自然获得饰品不展示“保底奖励”标识 |
| 标记边界 | 焦点区标记与保底奖励标记指向同一批卡牌但含义不同，前端按后端配置展示，不自行合并或推导 |

### 8.4 批量开启保底反馈

| 项目 | 说明 |
|---|---|
| 触发对象 | X5 等批量开启结果 |
| 触发条件 | 当前军需箱配置保底玩法，且用户进行批量开启 |
| 展示结果 | 根据后端最终结算结果映射到对应最高优先级状态文案 |
| 是否所有状态适用 | 仅配置保底玩法的指定军需箱适用 |
| 不适用处理 | 未配置保底玩法时不展示保底相关反馈 |
| 规则说明 | 批量开启不是单独展示状态；最终保底状态、奖励结果、饰品顺序均以后端返回为准，前端不自行推导 +N 或插入位置 |

### 8.5 水晶能量沉淀提示

| 项目 | 说明 |
|---|---|
| 触发对象 | 水晶能量沉淀提示 |
| 触发条件 | 本次能量开启存在多余能量转为水晶 |
| 展示结果 | 在底部按钮区下方或附近展示“多余能量已转为水晶，下次开启本箱可继续使用” |
| 是否所有状态适用 | 否 |
| 不适用处理 | 本次无多余能量沉淀时不展示 |
| 展示限制 | 不展示完整水晶余额，不使用 VIP 图标 |
| 状态关系 | 水晶能量沉淀提示与保底状态无关，任何状态下均按是否存在多余能量沉淀独立判断 |
| 示意说明 | 设计示意图中的水晶提示仅用于示意位置，实际按条件展示 |

| 前端禁止行为 | 前端不自行推导焦点区命中、保底触发、奖励发放、批量开启最终保底结果或保底奖励饰品插入位置 |

结果页只展示本次结算结果，不替代完整结算系统规则。

## 13. 研发实现注意事项

| 项目 | 注意事项 |
|---|---|
| 数据来源 | 结果页展示以后端最终结算结果为准 |
| 保底状态 | 顶部只展示本次结算后的最新 / 最高优先级状态 |
| 饰品列表 | 本次获得饰品区展示本次实际获得的全部饰品 |
| 保底奖励饰品 | 触发保底后由系统额外发放的饰品直接进入本次获得饰品区 |
| 展示顺序 | 前端按后端返回的结算顺序渲染，不自行调整保底奖励饰品位置 |
| 保底奖励标识 | 仅系统额外发放的保底奖励饰品展示“保底奖励”标识 |
| 自然获得焦点区卡牌 | 不展示“保底奖励”标识 |
| 批量开启 | 最终保底状态、奖励结果、饰品顺序均以后端返回为准 |
| 水晶提示 | 仅本次存在多余能量转为水晶时展示；无沉淀时不展示 |
| 既有功能 | 饰品卡片、返回按钮、再来 5 次按钮、关闭按钮等保持已有功能，本次不展开 |
| 不应新增 | 不新增保底奖励领取按钮、二次开启流程、完整资源概览、保底 info、接口字段或后端结算规则 |
