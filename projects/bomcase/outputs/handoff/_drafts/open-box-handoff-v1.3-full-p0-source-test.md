# Open Box 研发交接文档

## 1. 文档版本说明

本版本为 open box 当前页面口径的完整研发交接草稿，面向研发实现、联调、UI 承接和测试验收使用。

本版本不引入新的产品事实变化，仅整理当前已确认页面口径，用于研发实现、联调和验收校验。

本页面用于用户开启军需。开启条件可由赠品次数、账号已有能量、卡牌填充能量及其他实际参与资源共同满足。页面核心是稳定左侧资源概览、中间主判断与主操作、右侧选卡填充，以及页面左下方「本期保底」卡片表达。

最终开启满足度、资源构成、保底进度、保底状态与奖励结果以服务端返回为准。

## 4. 页面目标

open box 页面为军需开启主流程页面，当前确认范围覆盖：

1. 左侧「资源概览」区域：展示用户当前与开箱相关的账号资源状态。
2. 中间「猎户军需」核心操作区域：展示军需箱主视觉、开启次数选择、开启满足度、资源构成与主操作按钮。
3. 右侧「卡牌背包」区域：作为当前开箱流程中的卡牌填充选择器，用于支持卡牌填充能量操作。
4. 页面左下方「本期保底」运营活动卡牌：展示当前军需箱的保底状态、焦点区卡牌范围、累计进度与结果状态。
5. 下方饰品区域中的焦点区标记与保底奖励标记：仅说明其与本期保底卡片的关联，不展开饰品区域其它已有规则。

本次优化重点：

- 明确左侧资源概览、中间主操作、右侧卡牌填充选择器的职责分离。
- 强化中间区域作为当前页面主判断区和主操作区。
- 在页面左下方展示「本期保底」运营活动卡牌。
- 优化右侧卡牌背包半展开摘要态与全展开操作态。
- 明确卡牌填充、喜欢卡保护、资源构成 icon、本期保底卡片与按钮推荐态之间的边界。
- 在右侧卡牌背包全展开态底部固定汇总区，轻量提示本次填充能量溢出后的余额沉淀结果。

右侧面板分为两种展示态：

| 展示态 | 定义 | 结构要求 |
| --- | --- | --- |
| 半展开态 | 当前状态摘要。 | 只展示标题、摘要信息、状态说明和入口文案。 |

半展开与全展开应使用同一组件骨架，区别只在内容展示，不应表现为两个不同组件。

## 7. 核心页面口径

### 7.1 关键概念

| 概念 | 口径 |
| --- | --- |
| 焦点区卡牌 | 面向用户的保底命中范围表达。研发口径中，焦点区卡牌对应当前箱子内带保底标记的指定范围卡牌 / 饰品。 |
| 保底范围饰品 | 研发 / 后端理解口径，表示当前箱子中被配置为保底命中范围的指定卡牌 / 饰品。 |
| 焦点区标记 | 用于标识当前箱子中属于本期保底命中范围的卡牌。 |
| 保底奖励标记 | 用于标识触发保底后可能发放的奖励范围。 |
| 填充 +X% | 本次已选卡牌带来的新增能量，不包含账号已有能量。 |
| 卡牌能量 +X% | 本次开箱计入满足度的卡牌能量值，具体数值以服务端返回为准。 |
| 溢出 +Y% | 本次已选卡牌能量超过当前开启所需补足能量时，超出部分在开启后沉淀至账号能量余额。 |

焦点区卡牌与保底范围饰品指向同一组配置对象，不是两个奖励池。前台主表达统一使用「焦点区卡牌」，研发 / 后端口径可使用「保底范围饰品」。

### 7.2 卡牌数量口径

左侧「卡牌背包 N张」与右侧半展开态「共 N 张卡牌」表示用户当前拥有的卡牌总数。

右侧全展开列表是卡牌选择器，展示数量可受筛选、可用状态影响，不等同于用户拥有卡牌总数。喜欢卡保护不影响右侧列表展示，仅影响一键填充自动选择逻辑。

### 7.3 能量口径

| 位置 | 展示内容 | 口径说明 |
| --- | --- | --- |
| 左侧资源概览 | 能量余额 18% | 账号已有能量余额。 |
| 右侧底部汇总 / 半展开填充态 | 填充 +X% | 本次已选卡牌带来的新增能量，不包含账号已有能量。 |
| 右侧底部汇总补充说明 | 溢出 +Y% 存入能量余额 | 本次已选卡牌能量超过当前开启所需补足能量时，超出部分在开启后沉淀至账号能量余额；仅存在溢出时展示。 |
| 中间资源构成 icon | 卡牌能量 +X% | 本次开箱计入满足度的卡牌能量值，具体数值以服务端返回为准。 |

前端不通过「左侧账号已有能量 + 右侧本次选卡新增能量」自行计算中间「卡牌能量」展示值。中间「开启满足度」与资源构成均以服务端返回的本次开箱判断结果为准。

右侧「填充 +X%」表示本次已选卡牌带来的新增能量，不包含账号已有能量，也不因是否溢出而改变其展示口径。

「溢出 +Y%」仅在存在溢出能量时展示；不存在溢出时，仍展示「已选卡牌将在开启时消耗」。「溢出 +Y%」不进入中间资源构成 icon 区，不展示在中间开启满足度区域，不展示在左侧资源概览中。

### 7.4 资源参与口径

左侧资源概览展示账号资源总览；中间资源构成 icon 区只展示本次实际参与本次开启判断的资源。

| 路径 | 中间显示 icon | 中间不显示 |
| --- | --- | --- |
| 纯赠品路径 | 赠品次数 ×N | 卡牌能量、扳手、VIP |
| 纯能量路径 | 卡牌能量 +X%、扳手 ×N、VIP权益 +X%（如有） | 赠品 ×0 |
| 混合路径 | 赠品次数 ×N、卡牌能量 +X%、扳手 ×N、VIP权益 +X%（如有） | 未参与资源 |
| 都没有资源参与 | icon 区不显示任何内容 | 全部资源 icon |

赠品 ×0 不展示，未参与资源不展示，不显示空占位。当仅有 1 个资源 icon 时，使用完整资源名称文案，例如「赠品次数 ×1」。

### 7.5 本期保底口径

本期保底卡片仅作为当前军需箱的保底状态展示，不参与本次开启条件判断。

| 项目 | 口径 |
| --- | --- |
| 是否参与开启满足度计算 | 不参与 |
| 是否影响一键填充逻辑 | 不影响 |
| 是否影响开启军需按钮状态 | 不影响 |
| 是否进入资源构成 icon 区 | 不进入 |
| 是否直接影响 X1 / X5 切换 | 不直接影响 |
| 是否影响右侧卡牌背包 | 不影响 |
| 是否影响卡牌填充能量 | 不影响 |
| 是否影响资源消耗优先级 | 不影响 |
| 数据来源 | 以服务端配置与接口返回为准 |

| 类型 | 说明 |
| --- | --- |
| 自然命中焦点区 | 用户开启结果中获得任一焦点区卡牌，视为命中保底范围，本轮保底完成，进度重置，并进入新一轮，不额外触发保底奖励发放。 |
| 未命中焦点区 | 用户开启结果中未获得任一焦点区卡牌，本轮保底进度 +1。 |
| 触发保底 | 连续未命中焦点区达到保底门槛后，触发保底；奖励由后端按当前箱子的保底配置，在指定范围卡牌中随机发放 1 个奖励；本轮保底完成，进度重置，并进入新一轮。 |

保底奖励发放边界：

- 触发保底后不再发放军需箱，不再要求用户二次开启；
- 触发保底后的奖励在后端结算完成时即发放；
- 保底奖励发放后自动开启新一轮保底进度；
- 保底奖励发放不依赖前端结果页播放状态；
- 结果页仅作为告知展示，不作为奖励发放的触发节点。

批量开启说明：

- X5 等批量开启场景下，前端不逐次计算命中与进度变化；
- X5 等批量开启的保底最终展示以服务端结算结果为准；
- 前端不自行推导跨轮保底结果；
- 批量开启中如同时发生自然命中焦点区、触发保底奖励、新一轮进度变化等结果，前端不自行判断展示优先级，最终结果态文案与新一轮进度以服务端返回为准。

达到保底门槛后的展示说明：

- 达到保底门槛后的展示以最终结算结果为准；
- 不要求前台停留展示 `Y/Y` 过程态；
- 触发保底、自然命中焦点区等结果态用于本次开启后的结果反馈或刷新后的短期展示；
- 页面最终稳定态以结算后的新一轮保底进度为准；
- 结果态具体展示时长、消失方式、是否有过渡动效需后续确认。

### 7.6 X1 / X5 与赠品部分抵扣

X1 / X5 为开启次数选择。当选择 X5 且赠品次数不足 5 次时，赠品次数可计入本次开启满足度，剩余部分可通过卡牌能量等资源补足。

切换 X1 / X5 后：

- 已选卡牌清空；
- 开启满足度重新计算；
- 中间资源构成 icon 重新计算；
- 右侧状态重新判断；
- 右侧底部固定汇总区清空已选态与能量溢出提示；
- 本期保底卡片不因 X1 / X5 切换直接变化，保底进度仅在开启结算后按服务端结果刷新。

### 7.7 已有功能边界

资源消耗优先级、实际扣减逻辑、能量计算规则、开启条件不足时的已有提示逻辑，均沿用现有开箱规则，本次优化不调整。

保底进度、目标进度、剩余开启次数、焦点区卡牌范围、焦点区标记、保底奖励标记、批量开启最终保底结果，均以服务端配置与接口返回为准。

本次填充能量溢出值用于前台轻提示展示，具体计算和返回方式以后端或既有开启判断结果为准。本文档不重新定义能量计算、扣减顺序、余额沉淀规则或有效期规则。

### 8.3 本期保底卡片状态

默认状态：

```text
本期保底
焦点区卡牌
进度 X/Y
```

卡片使用固定奖励视觉资源，仅用于活动卡片表达，不代表最终实际发放奖励。

进度展示规则：

- X/Y 中，X 表示当前本轮已累计次数，Y 表示当前保底门槛；
- 进度值以服务端返回为准，前端不自行推导保底进度；
- Y ≤ 10 时展示分段进度条；
- Y > 10 时展示连续进度条；
- 新一轮开启后，进度按服务端返回的新一轮 X/Y 展示。

| 状态 | 触发条件 | 前台展示文案 | 展示说明 |
| --- | --- | --- | --- |
| 未配置保底 | 当前军需箱未配置保底玩法 | 不展示 | 模块不展示、不占位。 |
| 常规累计态 | 配置保底，且当前未临近触发 | 进度 X/Y | 正常累计中，按当前 X/Y 展示。 |
| 临近完成态 | 距离保底触发较近，例如剩余 N 次 | 再开启 N 次触发 | 用于强化即将达成感，N 以服务端返回或后端计算结果为准。 |
| 自然命中焦点区态 | 本次开启结果获得任一焦点区卡牌 | 已获得焦点区卡牌，新一轮已开启 | 本轮保底完成并重置，不额外触发保底奖励发放。 |
| 结算触发态 | 连续未命中达到保底门槛，进入奖励发放过程 | 奖励发放中... | 瞬时态，产品建议最多展示 2 秒；如服务端结果返回较快，可不强制停满 2 秒，直接进入新一轮延续态。 |
| 新一轮延续态 | 上一轮已完成，新一轮已开启 | 上一轮已完成，新一轮已开启 | 承接自然命中或触发保底后的重置说明，展示新一轮初始进度。 |

前台不使用「重置 / 清零」作为主表达。常态保持轻表达，完整规则放入 info。达到保底门槛后，不要求前台停留展示 `Y/Y` 过程态。

- 全展开态仅卡牌列表区域可滚动；顶部标题 / 筛选区与底部固定汇总区不随列表滚动。

卡牌选中态：

| 状态 | 视觉表现 | 说明 |
| --- | --- | --- |
| 普通卡 · 未选 | 未选中态正常展示，需与已选中态有明确视觉区分。 | 点击直接选中，无需确认。 |
| 普通卡 · 已选 | 展示已选中状态标识，需与未选中态有明确视觉区分。 | 再次点击反选撤回。 |
| 当前操作项 | 选中瞬间需有可识别反馈。 | 稳定后回到已选中态。 |
| 喜欢卡 · 未选 | 常驻喜欢卡标识，正常展示，不弱化；具体 icon 样式以最终设计稿为准。 | 点击触发二次确认。 |
| 喜欢卡 · 已选 | 需要同时表达喜欢保护与已选中状态。 | 已确认后正常计入填充结果，可反选。 |

喜欢卡与选中态共存时，需要同时表达两种状态。具体 icon 排布、层级与避让方式以最终设计稿为准，不允许因选中态覆盖而丢失喜欢保护表达。

喜欢卡轻提示：

```text
喜欢卡已保护，未自动填充
```

该提示不是常驻信息，仅在用户触发一键填充后，系统跳过喜欢卡时出现。如果未发生一键填充跳过喜欢卡，不展示该提示。

### 12.1 页面结构验收

- 页面应包含左侧资源概览、中间核心操作区、右侧卡牌背包三个主要区域。
- 左侧资源概览区域应包含：赠品开箱次数、能量余额、机械扳手、卡牌背包入口、会员等级。
- 中间核心区域应包含：军需箱主视觉、X1 / X5、开启满足度、资源构成 icon、一键填充、开启军需。
- 本期保底卡片应位于页面左下方区域。
- 本期保底卡片应以运营活动卡牌形式表达。
- 仅配置保底玩法的指定军需箱展示本期保底卡片。
- 未配置保底玩法时，不展示本期保底卡片，不保留空位。
- 右侧卡牌背包区域应支持半展开态和全展开态。
- 半展开和全展开应使用同一组件骨架，不应表现为两个不同组件。
- 全展开态应使用 2 列卡牌布局。
- 全展开态仅卡牌列表区域可滚动；顶部标题 / 筛选区与底部固定汇总区不随列表滚动。
- 能量溢出提示仅归属右侧卡牌背包全展开态底部固定汇总区，不新增独立模块。
- 下方卡牌 / 饰品区需要支持展示焦点区标记。
- 下方卡牌 / 饰品区需要支持展示保底奖励标记。

### 12.2 字段展示验收

- 左侧赠品开箱次数展示为 L1 主资源，可展示 0。
- 左侧能量余额展示为 L2 次主资源，仅展示账号已有能量。
- 左侧即将过期提示保留在能量余额卡片内。
- 左侧机械扳手、卡牌背包、会员等级展示为 L3 信息。
- 左侧会员等级不展示 info，不新增点击交互。
- 左侧不展示会员权益加成数值。
- 左侧卡牌背包展示为「卡牌背包 N张 ›」形式。
- 左侧「N张」表示用户当前拥有卡牌总数。
- 右侧半展开「共 N 张卡牌」表示用户当前拥有卡牌总数。
- 右侧全展开列表展示数量可受筛选、可用状态影响，不等同于用户拥有卡牌总数。
- 喜欢卡保护不影响右侧列表展示，仅影响一键填充自动选择逻辑。
- 中间开启满足度应展示百分比大字和进度条。
- 中间资源构成 icon 只展示本次实际参与资源。
- 中间「卡牌能量 +X%」以服务端返回的本次计入满足度的卡牌能量值为准，前端不自行根据左侧与右侧相加推导。
- 中间未参与资源不展示，不显示 ×0。
- 本期保底卡片应展示前台标题「本期保底」。
- 本期保底卡片应展示范围标签「焦点区卡牌」。
- 常规累计态展示轻文案「进度 X/Y」。
- 临近完成态展示「再开启 N 次触发」。
- 自然命中焦点区态展示「已获得焦点区卡牌，新一轮已开启」。
- 结算触发瞬时态展示「奖励发放中...」。
- 新一轮延续态展示「上一轮已完成，新一轮已开启」。
- 本期保底卡片应展示 info icon。
- Y ≤ 10 时展示分段进度条，Y > 10 时展示连续进度条。
- 固定奖励视觉资源仅用于活动卡片表达，不代表最终实际发放奖励。
- 保底结果态文案应以服务端返回为准展示，不由前端自行推导。
- 右侧顶部展示收起 / 展开切换 icon，不展示 `[0/3]`、`3/300` 等未确认口径计数。
- 右侧半展开填充态必须展示「已选 N 张，填充 +X%」。
- 右侧半展开态不展示「溢出 +Y% 存入能量余额」。
- 右侧全展开已选态必须展示「已选 N 张｜填充 +X%｜清除已选」。
- 右侧全展开已有选择时，底部固定汇总区应展示「已选卡牌将在开启时消耗」。
- 当本次已选卡牌存在能量溢出时，右侧全展开底部固定汇总区应展示「已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额」。
- 当不存在能量溢出时，不展示「溢出 +Y% 存入能量余额」。
- 「溢出 +Y%」表示本次开启消耗后沉淀至账号能量余额的部分。
- 「溢出 +Y%」不应展示在中间开启满足度、资源构成 icon 区或左侧资源概览中。
- 右侧「填充 +X%」表示本次已选卡牌带来的新增能量，不包含左侧账号已有能量。
- 卡牌能量值选中态与未选中态需有明确视觉区分，具体颜色以最终设计稿为准。
- 下方卡牌 / 饰品区应按服务端配置展示焦点区标记与保底奖励标记。

### 12.3 状态验收

- 有卡牌且赠品次数满足时，右侧半展开展示「赠品次数已满足，暂无需填充」。
- 有卡牌且开启能量已满时，右侧半展开展示「开启能量已满足，暂无需填充」。
- 有卡牌且已有填充时，右侧半展开展示「已选 N 张，填充 +X%」。
- 无可用卡牌时，右侧半展开展示「暂无可用卡牌」和「前往获取」。
- 无可用卡牌时，右侧半展开应展示空状态插图。
- 一键填充跳过喜欢卡时，右侧展示轻提示「喜欢卡已保护，未自动填充」。
- 非一键填充跳过喜欢卡场景，不展示该提示。
- 喜欢卡提示不是默认常驻说明，示例图为一键填充后的提示态。
- 可填充但未满足时，一键填充为推荐态。
- 已满足开启条件时，开启军需为推荐态。
- 都没有资源时，中间资源构成 icon 区不展示任何内容。
- 都没有资源时，中间进度条下方不展示文案。
- 无可填充卡牌时，一键填充不作为当前推荐操作，但状态仍需可识别；具体视觉表现以最终设计稿为准。
- 当选择 X5 且赠品次数不足 5 次时，赠品次数可计入本次开启满足度，剩余部分可通过卡牌能量等资源补足。
- X1 / X5 切换后，开启满足度和资源构成 icon 应重新计算。
- X1 / X5 切换后，已选卡牌清空，右侧底部固定汇总区不再展示已选态、消耗说明或能量溢出提示。
- 已选卡牌且不存在能量溢出时，右侧全展开态底部展示「已选卡牌将在开启时消耗」。
- 已选卡牌且存在能量溢出时，右侧全展开态底部展示「已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额」。
- 清除已选后，右侧全展开态底部不展示「已选卡牌将在开启时消耗」或「溢出 +Y% 存入能量余额」。
- 未配置保底玩法时，不展示本期保底卡片。
- 常规累计态、临近完成态、自然命中焦点区态、结算触发态、新一轮延续态均需可识别。
- 本次开启未获得任一焦点区卡牌时，本轮保底进度 +1。
- 本次开启获得任一焦点区卡牌时，本轮保底完成，进度重置，并进入新一轮。
- 自然命中焦点区态不触发额外保底奖励。
- 连续未命中焦点区达到保底门槛时，由后端按当前箱子的保底配置，在指定范围卡牌中随机发放 1 个奖励。
- 触发保底后，本轮保底完成，进度重置，并进入新一轮。
- 批量开启时，保底最终展示以服务端返回结果为准。
- 批量开启中如同时发生自然命中焦点区、触发保底奖励、新一轮进度变化等结果，前端不自行判断展示优先级。
- 结算触发态仅作为瞬时态，不作为长期稳定态。
- 结算触发态产品建议最多展示 2 秒；如服务端结果返回较快，可不强制停满 2 秒。
- 新一轮开启后，展示服务端返回的新一轮进度。

### 12.4 交互验收

- info 类统一 hover 触发。
- 左侧「卡牌背包 N张 ›」点击后，右侧应进入全展开态。
- 当右侧已处于全展开态时，再点击左侧「卡牌背包 N张 ›」，右侧应通过抖动或高亮等方式进行区域提示，具体方式待确认。
- X1 / X5 切换后，已选卡牌应清空，并重新计算开启满足度。
- X1 / X5 切换后，能量溢出提示应同步清空。
- 右侧「前往获取 ›」点击打开 CTA 下方建议弹窗。
- 普通卡点击后直接选中，并立即计入当前填充结果。
- 已选普通卡再次点击后反选撤回。
- 喜欢卡未选中时点击应触发二次确认，已有功能，本次不展开。
- 喜欢卡确认加入后正常计入填充结果。
- 一键填充时应跳过喜欢卡。
- 一键填充跳过喜欢卡时，右侧应展示轻提示。
- 一键填充后如存在能量溢出，右侧全展开态底部固定汇总区应展示溢出提示。
- 清除已选后，右侧已选状态清空。
- 清除已选后，右侧底部不再展示能量溢出提示。
- 开启条件满足时，点击开启军需执行开启。
- 开启条件不满足时，点击开启军需触发已有功能，本次不展开。
- 离开页面后，已选卡牌不持久化。
- 离开页面后，能量溢出提示不持久化。
- 即将过期能量提示 hover 后展示说明弹层，该弹层为后续迭代能力。
- 保底 info icon hover 后展示保底规则说明。
- 本期保底卡片本体不新增点击规则。
- 固定奖励视觉资源不新增点击规则。
- 进度条不新增点击规则。
- 焦点区标记不新增点击规则。
- 保底奖励标记不新增点击规则。

### 12.5 保底 info 验收

保底 info 至少包含以下说明：

```text
焦点区卡牌为本期保底范围。
未获得焦点区卡牌时，保底进度增加。
获得焦点区卡牌或触发保底后，本轮保底完成并开启新一轮。
触发保底后，将直接获得焦点区范围内的 1 个奖励。
```

info 内容用于前台规则说明，不替代后端实际结算规则。实际保底进度、门槛、奖励、范围以服务端配置与接口返回为准。

### 12.6 标记验收

- 焦点区标记用于命中判断范围识别。
- 保底奖励标记用于触发保底后的奖励范围识别。
- 焦点区标记与保底奖励标记指向同一批卡牌，但表达含义不同。
- 前端不得将焦点区标记与保底奖励标记强行合并为同一标记。
- 前端不得只展示其中一个标记来替代另一个标记，除非产品和 UI 另行确认。
- 前端不得根据卡牌位置、稀有度、图片或名称自行判断是否属于焦点区或保底奖励范围。
- 标记展示以服务端配置为准。

### 12.7 范围控制验收

- 不在左侧展示会员权益加成数值。
- 不在左侧会员等级上新增 info 或点击交互。
- 不把右侧卡牌背包做成资产管理区。
- 不在半展开态展示筛选、卡牌列表和底部固定汇总。
- 不在半展开态展示「溢出 +Y% 存入能量余额」。
- 不在右侧筛选区展示「清除」。
- 不在右侧顶部展示未确认口径的 `[0/3]`、`3/300` 等数量。
- 不在无确认情况下新增 loading、分页、权限、异常状态。
- 不重新定义资源消耗优先级、实际扣减逻辑、能量计算规则。
- 不因能量溢出提示新增独立能量沉淀模块、弹窗、领取按钮或结算前确认流程。
- 不把能量溢出提示放入中间开启满足度、资源构成 icon 区、本期保底卡片或左侧资源概览。
- 不新增累计返赠、任务、领取按钮、活动中心入口等未确认功能。
- 历史讨论中曾出现「累计返赠」命名，本次统一确认模块前台标题为「本期保底」，不使用「累计返赠」。
- 不展开资源概览、卡牌背包、底部掉落 Tab、CTA 按钮等已有功能规则。
- 不额外设计保底 loading、异常态、无权限态、按钮置灰态、未确认空状态。
- 保底常态保持轻表达，规则说明放入 info。
- 本期保底卡片不放回中间开启满足度区域。
- 本期保底卡片不参与开启满足度计算，不影响一键填充、开启军需按钮状态。
- 本期保底卡片不进入中间资源构成 icon 区。
- 不新增保底奖励领取按钮。
- 不新增保底奖励二次开箱流程。
- 不展示「获得军需箱后再开启」的旧规则表达。
- 不把固定设计资源当作实际发放奖励。
- 不把「焦点区卡牌」与「保底范围饰品」理解为两个奖励池。
- 不将「焦点区标记」与「保底奖励标记」混用。
- 能量溢出轻提示不影响一键填充、开启军需按钮状态。
- 对已有功能但本次未说明的内容，不展开流程和规则。

## 13. 不覆盖范围

- 不补完整资源扣减、库存、订单、支付、发货、资产、权益到账、结算、交易、钱包系统规则。
- 不把示例值写成固定系统规则。
- 不把待补充边界写成已确认产品事实。
- 不定义后端奖励随机算法。
- 不定义后端保底配置后台流程。
- 不定义奖励库存、资产到账、异常补偿、发放失败等后端规则。
- 不定义批量开启中每一次开箱的逐步进度动画。
- 不新增保底奖励领取按钮、弹窗或二次确认流程。
- 不新增保底奖励详情页。
