## 3. 【本次调整】不变更范围声明

- 不改变开箱页三栏职责（左侧资源概览 / 中部主判断与主操作 / 右侧选卡填充）。
- 不改变保底模块展示条件与职责边界（仅配置保底玩法时展示；不参与满足度计算，不影响按钮主状态）。
- 不改变能量溢出提示展示位置（仅右侧卡牌背包全展开态底部固定汇总区）。
- 不改变 X1/X5 切换后的“已选清空 + 状态重算”行为。
- 不改变既有验收基线与已有能力边界。
- 未特别标注 `【本次调整】` / `【本次新增】` / `【本次延续】` / `【待确认】` 的章节，均沿用上一版口径。

## 4. 页面目标

- 左侧展示资源总览；
- 中部承接主判断与主操作；
- 右侧承接卡牌填充选择；
- 中部展示保底进度与规则说明入口；
- 下方饰品区表达保底范围标记关联。

## 5. 页面结构

1. 顶部导航（已有功能范围）。
2. 左侧资源概览区。
3. 中部猎户军需核心区（X1/X5、满足度、资源构成、CTA）。
4. 中部保底进度模块（条件展示）。
5. 右侧卡牌背包区（半展开摘要态 / 全展开操作态 / 全展开底部固定汇总区）。
6. 下方饰品区保底标记与底部掉落信息 Tab（已有功能边界）。

## 6. 研发影响标记

### 6.1 【本次延续】高影响模块最小实现口径：能量溢出提示

- 模块职责：在右侧卡牌背包全展开态底部固定汇总区，展示本次填充能量超出后可沉淀至能量余额的结果提示。
- 已确认展示口径：
  - 存在溢出时展示“溢出 +Y% 存入能量余额”；
  - 不存在溢出时不展示该提示；
  - 该提示不进入中部满足度、资源构成 icon 区或左侧资源概览。
- 关键状态 / 交互：
  - 已选卡牌变化触发底部汇总与溢出提示重算；
  - 切换 X1/X5、清除已选后，溢出提示同步更新或清空；
  - 半展开态不展示溢出提示。
- 研发影响侧：前后端。
- 待确认阻塞点：
  - 溢出值来源（后端返回或前端计算）；
  - 溢出值刷新时机（选卡变化、X1/X5 切换、清除已选、开启后）。
- 验收关注点：
  - 展示/不展示条件正确；
  - 展示位置固定且与中部/左侧不混用；
  - 状态切换后提示与当前选择保持一致。

### 6.2 【本次调整】研发优先阅读顺序

建议优先阅读：第 2 节版本更新说明 -> 第 3 节不变更范围 -> 第 6.1 节高影响模块 -> 第 11 节待确认项分层 -> 第 12 节验收口径。

## 7. 组件与字段

- 左侧资源字段：赠品次数、能量余额、机械扳手、卡牌背包入口、会员等级；
- 中部关键字段：开启满足度、资源构成 icon、保底进度文案；
- 右侧关键字段：已选 N 张、填充 +X%、清除已选、溢出 +Y% 存入能量余额；
- 右侧顶部不展示未确认口径计数（如 `[0/3]`、`3/300`）。

## 8. 核心页面口径

### 8.1 三栏职责口径

- 左侧只做资源概览，不直接承担开启判断；
- 中部为开启判断与主操作区；
- 右侧为卡牌填充选择器，不作为资产管理区。

### 8.2 【本次延续】能量溢出提示口径

- 能量溢出提示属于页面展示口径补充，不构成资源计算规则变更；
- 展示位置限定在右侧卡牌背包全展开态底部固定汇总区；
- 不改变中部主操作区职责与开启按钮主状态。

### 8.3 保底模块口径

- 仅配置保底玩法时展示，未配置不占位；
- 不参与开启满足度计算，不进入资源构成 icon 区；
- 相关数据与结果以后端配置与接口返回为准。

## 9. 页面状态 / 交互规则

- 半展开与全展开共用同一组件骨架；
- 一键填充跳过喜欢卡时展示轻提示；
- 开启条件不足时，开启军需按钮保持可点击并触发已有能力承接；
- 切换 X1/X5 后，已选卡牌与底部汇总同步清空并重算。

## 10. 空状态 / 异常状态

- 无可用卡牌：右侧半展开显示空态插图与“前往获取”；
- 保底模块空状态：未配置保底玩法时不展示、不占位；
- 能量溢出提示空状态：不存在溢出时不展示该提示。

## 12. 验收口径

### 12.1 【本次调整】优先验收项

- 有溢出时：右侧全展开底部固定汇总区展示“溢出 +Y% 存入能量余额”；
- 无溢出时：不展示溢出提示；
- 溢出提示不得出现在中部满足度、资源构成 icon 区或左侧资源概览；
- 切换 X1/X5 或清除已选后，溢出提示状态与当前选择一致。

### 12.2 结构与行为验收

- 页面结构、按钮策略、保底展示条件、半/全展开切换、空状态策略沿用既有口径；
- 保底相关结果态与批量开启结算以后端结果为准。
