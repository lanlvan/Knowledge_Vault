## 4. 页面目标

Open Box 页面用于承接军需开启主流程，核心是三栏职责稳定：左侧资源总览、中间主判断与主操作、右侧选卡填充。开启条件可由赠品次数、账号已有能量、卡牌填充能量及其它实际参与资源共同满足。页面中所有与保底进度、结算结果、批量开启结果相关展示，均以后端配置与返回为准。

## 5. 页面结构

|区域|说明|
|---|---|
|顶部导航|已有功能，本次不展开内部规则|
|左侧资源概览|账号资源总览，不承担本次可开启判断，不承担选卡操作|
|中间猎户军需核心区|本次主判断与主操作区，含 X1/X5、满足度、资源构成、保底进度、CTA|
|右侧卡牌背包|开箱流程中的卡牌填充选择器，分半展开摘要态 / 全展开操作态|
|下方饰品区与底部掉落信息 Tab|已有功能；本次仅说明保底标记与保底模块关联|

### 5.1 左侧资源概览

- 三层信息层级：
  - L1 主资源：赠品开箱次数
  - L2 次主资源：账号已有能量余额
  - L3 辅助资源/入口/身份：机械扳手、卡牌背包、会员等级
- 可展示数量为 0 的资源。
- 即将过期提示仅在能量余额卡片内展示。

### 5.2 中间核心区

- 展示当前开启次数、开启满足度、本次参与资源、保底进度、推荐操作。
- 不展示账号资源总览，不展示完整卡牌列表。
- 中间本次资源构成 icon 区只展示本次实际参与开启判断的资源。

### 5.3 保底进度模块

|项目|说明|
|---|---|
|位置|中间核心区内，资源构成 icon 区下方、CTA 区上方|
|展示条件|仅配置保底玩法的军需箱展示|
|未配置处理|不展示、不占位，CTA 区自然承接|
|职责|仅展示保底机制进度与规则，不参与开启满足度与按钮状态|

### 5.4 右侧卡牌背包

- 半展开：只展示摘要信息，不承载具体选卡操作。
- 全展开：展示筛选、2 列卡牌列表、动态轻提示、底部固定汇总区。
- 底部固定汇总区在已选状态下展示已选数量、填充能量、清除入口、消耗说明；存在溢出时补充展示溢出提示。

## 6. 组件与字段

### 6.1 关键概念定义

- 开启满足度：当前开启条件的满足百分比展示，以后端返回为准。
- 本次填充能量 `+X%`：本次已选卡牌带来的新增能量，不含账号已有能量。
- 溢出 `+Y%`：本次填充超过当前开启所需补足能量的部分，开启后沉淀至账号能量余额。
- 保底范围饰品：当前军需箱配置中被纳入保底范围、在下方饰品区带保底标记的饰品。
- 自然命中保底范围饰品：正常开启结果获得任一保底范围饰品，本轮保底完成并进入新一轮，不额外触发保底奖励发放。
- 触发保底：连续未获得保底范围饰品达到门槛后，由后端结算发放保底奖励。

## 7. 核心页面口径

### 7.1 卡牌数量口径

- 左侧 `卡牌背包 N张` 与右侧半展开 `共 N 张卡牌` 均表示用户拥有卡牌总数。
- 右侧全展开列表是选择器，展示数量可受筛选与可用状态影响，不等同于用户拥有总数。
- 喜欢卡保护不影响列表展示，只影响一键填充自动选择逻辑。

### 7.2 能量口径

|位置|展示内容|口径说明|
|---|---|---|
|左侧资源概览|能量余额 18%|账号已有能量余额|
|右侧底部汇总 / 半展开填充态|填充 +X%|本次已选卡牌新增能量，不含账号已有能量|
|右侧底部汇总补充说明|溢出 +Y% 存入能量余额|存在溢出时展示，表示开启后沉淀至账号能量余额部分|
|中间资源构成 icon|卡牌能量 +X%|本次计入满足度的卡牌能量值，以后端返回为准|

- 前端不通过左侧已有能量与右侧新增能量自行推导中间值。
- 中间满足度与资源构成以后端返回结果为准。
- 溢出提示仅在右侧全展开底部固定汇总区展示。

### 7.3 资源参与口径

|路径|中间显示 icon|中间不显示|
|---|---|---|
|纯赠品路径|赠品次数 ×N|卡牌能量、扳手、VIP|
|纯能量路径|卡牌能量 +X%、扳手 ×N、VIP权益 +X%（如有）|赠品 ×0|
|混合路径|赠品次数 ×N、卡牌能量 +X%、扳手 ×N、VIP权益 +X%（如有）|未参与资源|
|都没有资源参与|icon 区不显示任何内容|全部资源 icon|

### 7.4 保底机制口径

|项目|口径|
|---|---|
|是否参与开启满足度计算|不参与|
|是否影响一键填充逻辑|不影响|
|是否影响开启军需按钮状态|不影响|
|是否进入资源构成 icon 区|不进入|
|数据来源|以后端配置与接口返回为准|

- 达到保底门槛后的展示以最终结算结果为准，不要求停留 `Y/Y` 过程态。
- 结果态用于开启后的反馈或短期展示；稳定态回到新一轮进度。
- 批量开启中多结果并发时，前端不自行判断优先级，最终以后端返回为准。

### 7.5 X1 / X5 切换口径

- 切换后清空已选卡牌。
- 重新计算开启满足度与中间资源构成。
- 右侧状态重判，底部汇总与溢出提示清空。
- 保底进度不因切换直接变化，仅在开启结算后刷新。

## 12. 验收口径

### 12.1 页面结构验收

- 页面包含左侧资源概览、中间核心操作区、右侧卡牌背包三大区域。
- 保底进度模块位于中间资源构成 icon 区下方、CTA 区上方。
- 未配置保底玩法时模块不展示、不占位。
- 右侧支持半展开与全展开，且为同一组件骨架。
- 全展开态为 2 列卡牌布局，仅列表区滚动。
- 能量溢出提示仅归属右侧全展开底部固定汇总区。

### 12.2 字段展示验收

- 左侧赠品次数为 L1，能量余额为 L2，机械扳手/卡牌背包/会员等级为 L3。
- 左侧会员等级不展示 info，不新增点击交互，不展示会员权益加成值。
- 左侧 `N张` 与右侧半展开 `共 N 张卡牌` 均表示用户拥有总数。
- 中间资源构成仅展示本次参与资源，未参与资源不展示，不显示 `×0`。
- 保底模块常态主文案为：`当前 X/Y，再积累 N 次可获「保底奖励名称」`。
- 右侧半展开填充态必须展示：`已选 N 张，填充 +X%`。
- 右侧全展开已选态必须展示：`已选 N 张｜填充 +X%｜清除已选`。
- 存在溢出时必须展示：`已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额`。
- 不存在溢出时不得展示溢出文案。

### 12.3 状态验收

- 有卡牌且赠品满足：半展开展示 `赠品次数已满足，暂无需填充`。
- 有卡牌且能量已满：半展开展示 `开启能量已满足，暂无需填充`。
- 无卡牌：半展开展示 `暂无可用卡牌` + `前往获取`，并展示空状态插图。
- 一键填充跳过喜欢卡时展示：`喜欢卡已保护，未自动填充`。
- 可填充但未满足时，一键填充为推荐操作；已满足时开启军需为推荐操作。
- X1/X5 切换后，已选清空，右侧底部消耗说明与溢出提示同步清空。
- 保底结果态与批量开启结果以后端返回为准。

### 12.4 交互验收

- info 类统一 hover 触发。
- 左侧 `卡牌背包 N张 ›` 点击后右侧进入全展开。
- 右侧已全展开时再次点击左侧入口，右侧以抖动或高亮提示（具体方式待确认）。
- 普通卡点击直选中，再点反选；喜欢卡点击触发二次确认。
- 一键填充跳过喜欢卡并展示轻提示。
- 清除已选后，已选状态与溢出提示同步清空。
- 开启军需满足条件执行开启，不满足条件沿用已有处理。
- 离开页面后，已选卡牌与溢出提示不持久化。

### 12.5 保底 info 验收

保底 info 至少覆盖以下四行：

```text
本军需箱设有保底范围饰品，可在下方饰品区域通过保底标记识别
未获得任一保底范围饰品时，保底进度 +1
获得任一保底范围饰品后，本轮保底完成并开启新一轮
触发保底后自动发放对应奖励
```

### 12.6 范围控制验收

- 不在半展开态展示筛选、卡牌列表、底部固定汇总、溢出提示。
- 不在右侧筛选区展示“清除”。
- 不在右侧顶部展示未确认口径计数（如 `[0/3]`、`3/300`）。
- 不在左侧会员等级新增 info 或点击交互。
- 不重新定义资源扣减优先级、实际扣减逻辑、能量计算规则。
- 不新增独立能量沉淀模块、弹窗、领取按钮或结算前确认流程。
- 不将溢出提示放入中间满足度、资源构成、保底模块或左侧资源概览。

## 13. 不覆盖范围

- 不补完整资源扣减、库存、订单、支付、发货、资产、权益到账、结算、交易、钱包系统规则。
- 不把示例值写成固定系统规则。
- 不把后续补充边界写成已确认规则。
- 不扩展顶部导航、底部掉落信息 Tab、筛选能力等已有功能内部规则。
