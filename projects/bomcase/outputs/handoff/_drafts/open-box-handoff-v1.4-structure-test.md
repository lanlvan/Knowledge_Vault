## 4. 页面目标

- 页面用于承接开箱主流程，稳定三栏职责：左侧资源概览、中间主判断与主操作、右侧选卡填充。
- 页面左下方展示「本期保底」运营活动卡牌，表达保底范围、进度和结果态。
- 焦点区标记与保底奖励标记用于下方卡牌/饰品区范围识别，不扩展该区域其它规则。
- 最终开启满足度、保底结果、批量开启结果以服务端返回为准。
- 本页不扩展资源扣减、库存、发放、钱包等系统级规则。

## 7. 核心页面口径

### 7.1 关键概念定义

| 概念 | 定义 |
| --- | --- |
| 焦点区卡牌（主表达） | 面向用户的保底命中范围表达 |
| 保底范围饰品（研发理解口径） | 当前箱子内带保底标记的指定范围卡牌/饰品 |
| 二者关系 | 指向同一组配置对象，不是两个奖励池 |
| 焦点区标记 | 标识该卡牌属于本期保底命中范围 |
| 保底奖励标记 | 标识该卡牌属于触发保底后的可能奖励范围 |
| 两类标记关系 | 指向同一批卡牌，但表达含义不同，不可合并替代 |

### 7.2 资源与能量口径

| 位置 | 展示内容 | 口径说明 |
| --- | --- | --- |
| 左侧资源概览 | 能量余额 18% | 账号已有能量余额 |
| 右侧半展开/全展开汇总 | 填充 +X% | 本次已选卡牌新增能量，不包含账号已有能量 |
| 右侧全展开底部补充说明 | 溢出 +Y% 存入能量余额 | 本次填充超过当前开启所需补足能量时展示 |
| 中间资源构成 icon | 卡牌能量 +X% | 本次计入开启满足度的卡牌能量，以服务端返回为准 |

- 前端不通过左侧余额与右侧填充值相加，推导中间展示。
- `溢出 +Y%` 只在右侧全展开底部固定汇总区出现。
- `溢出 +Y%` 不进入中间满足度、不进入中间资源构成 icon、不进入左侧资源概览。

### 7.3 本期保底模块口径

| 项目 | 口径 |
| --- | --- |
| 是否参与开启满足度计算 | 不参与 |
| 是否影响一键填充逻辑 | 不影响 |
| 是否影响开启军需按钮状态 | 不影响 |
| 是否进入中间资源构成 icon 区 | 不进入 |
| 是否直接影响 X1/X5 切换 | 不直接影响 |
| 是否影响右侧卡牌背包 | 不影响 |
| 数据来源 | 以服务端配置与接口返回为准 |

| 类型 | 说明 |
| --- | --- |
| 自然命中焦点区 | 获得任一焦点区卡牌，本轮保底完成，进度重置并进入新一轮；不额外触发保底奖励 |
| 未命中焦点区 | 本轮保底进度 +1 |
| 触发保底 | 连续未命中达到门槛后触发；后端按当前箱子的保底配置，在指定范围卡牌中随机发放 1 个奖励；本轮完成并进入新一轮 |

- 触发保底后不再发放军需箱，不要求二次开启。
- 结果页只做告知展示，不作为奖励发放触发节点。
- 批量开启（X5）最终结果以服务端结算返回为准，前端不逐次推导。

### 7.4 X1 / X5 规则

- X1 / X5 是开启次数选择，不是规则模式切换。
- 切换后：已选卡牌清空、开启满足度重算、中间资源构成重算、右侧状态重算、底部已选态和溢出提示清空。
- 本期保底卡片不因切换直接变化；仅在开启结算后按服务端结果刷新。

### 7.5 已有功能边界

- 资源扣减优先级、实际扣减逻辑、能量计算规则沿用已有能力。
- 本文档不定义接口字段、表结构、服务拆分与后端算法。
- 本文档不扩展顶部导航、底部掉落信息 Tab 等已有功能内部规则。

## 8. 页面状态

### 8.1 左侧资源概览状态

| 层级 | 内容 | 状态表达要求 |
| --- | --- | --- |
| L1 | 赠品开箱次数 | 主资源层级，权重高于 L2/L3，可展示 0，保留 `+` 入口 |
| L2 | 账号已有能量余额 | 次主资源层级，默认展示总量 |
| L3 | 机械扳手、卡牌背包、会员等级 | 辅助层级，不抢占主视觉 |

| 状态项 | 触发条件 | 展示结果 |
| --- | --- | --- |
| 能量即将过期提示 | 存在即将过期能量 | 展示 `即将过期 X%` |
| 能量即将过期提示 | 无即将过期能量 | 不展示 |
| 会员等级 | 任意状态 | 展示 `会员等级 / Lv.N / X%`，不展示会员权益加成值 |

左侧 info / hover 文案：

```text
展示账号当前与开启军需相关的资源概览，具体开启判断以中间区域为准。
```

```text
能量余额为账号当前已有能量，可用于补足开启满足度；具体来源与有效期以能量明细为准。
```

```text
即将过期能量为近期可能失效的能量，请在有效期内使用。具体过期时间以系统记录为准。
```

## 12. 验收口径

### 12.1 页面结构验收

- 页面包含左侧资源概览、中间核心区、右侧卡牌背包三大区域。
- 本期保底卡片位于页面左下方，且为运营活动卡牌形式。
- 配置保底玩法时展示本期保底卡片；未配置不展示不占位。
- 右侧支持半展开和全展开，且为同一组件骨架。
- 全展开为 2 列卡牌布局，仅列表区域滚动，顶部与底部固定。
- 能量溢出提示仅归属右侧全展开底部固定汇总区。
- 下方卡牌/饰品区支持焦点区标记与保底奖励标记展示。

### 12.2 字段展示验收

- 左侧赠品开箱次数为 L1 主资源，可展示 0。
- 左侧能量余额为 L2 次主资源，仅展示账号已有能量。
- 左侧即将过期提示在能量余额卡片内展示，不额外新增底部区。
- 左侧会员等级不展示 info，不新增点击交互，不展示会员权益加成值。
- 左侧 `卡牌背包 N张 ›` 与右侧半展开 `共 N 张卡牌` 均表示用户拥有卡牌总数。
- 右侧全展开列表数量可受筛选和可用状态影响，不等同于拥有总数。
- 喜欢卡保护不影响列表展示，仅影响一键填充自动选择。
- 中间展示满足度百分比与进度条；资源构成只展示本次参与资源。
- 本期保底卡片展示：`本期保底`、`焦点区卡牌`、`进度 X/Y`、状态文案与 info icon。
- 本期保底状态文案需覆盖：`再开启 N 次触发`、`已获得焦点区卡牌，新一轮已开启`、`奖励发放中...`、`上一轮已完成，新一轮已开启`。
- Y ≤ 10 展示分段进度条，Y > 10 展示连续进度条。
- 右侧半展开填充态展示 `已选 N 张，填充 +X%`，不展示溢出提示。
- 右侧全展开已选态展示 `已选 N 张｜填充 +X%｜清除已选`。
- 已选且无溢出时展示 `已选卡牌将在开启时消耗`。
- 已选且有溢出时展示 `已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额`。
- 不存在溢出时不展示 `溢出 +Y%` 文案。
- `溢出 +Y%` 不出现在左侧、中间、本期保底卡片。

### 12.3 状态验收

- 有卡牌且赠品满足：半展开展示 `赠品次数已满足，暂无需填充`。
- 有卡牌且能量已满：半展开展示 `开启能量已满足，暂无需填充`。
- 有卡牌且已有填充：半展开展示 `已选 N 张，填充 +X%`。
- 无卡牌：半展开展示 `暂无可用卡牌` 与 `前往获取`，并显示空状态插图。
- 一键填充跳过喜欢卡时展示轻提示 `喜欢卡已保护，未自动填充`；其他场景不展示。
- 可填充但未满足时，一键填充为推荐态；已满足时，开启军需为推荐态。
- 都没有资源时，中间资源构成 icon 区不展示内容，进度条下方不展示文案。
- 切换 X1/X5 后，满足度与资源构成重算，已选与溢出提示清空。
- 未配置保底玩法时，本期保底卡片不展示。
- 保底状态覆盖常规累计、临近完成、自然命中、结算触发、新一轮延续。
- 未命中焦点区时保底进度 +1；自然命中时本轮完成并进入新一轮。
- 达到保底门槛时由后端按当前箱子的保底配置，在指定范围卡牌中随机发放 1 个奖励。
- 批量开启最终展示以后端结算结果为准。

### 12.4 交互验收

- info 类统一使用 hover 触发。
- 左侧 `卡牌背包 N张 ›` 点击后右侧进入全展开；若已全展开则触发区域提示。
- X1/X5 切换后，已选卡牌清空并重算状态，溢出提示同步清空。
- `前往获取 ›` 点击打开 CTA 下方建议弹窗。
- 普通卡点击直接选中；再次点击可反选。
- 喜欢卡点击触发二次确认；确认后计入填充结果。
- 一键填充时应跳过喜欢卡并展示轻提示。
- 全展开已选且存在溢出时，底部固定汇总区展示溢出提示。
- 清除已选后，已选态与溢出提示同时清空。
- 开启条件满足时点击开启军需执行开启；不满足时触发已有能力。
- 离开页面后，已选状态与溢出提示不持久化。
- 本期保底卡片本体、固定奖励视觉资源、进度条、焦点区标记、保底奖励标记均不新增点击规则。

### 12.5 保底 info 验收

- info 至少包含以下四句文案：

```text
焦点区卡牌为本期保底范围。
未获得焦点区卡牌时，保底进度增加。
获得焦点区卡牌或触发保底后，本轮保底完成并开启新一轮。
触发保底后，将直接获得焦点区范围内的 1 个奖励。
```

- info 用于前台规则说明，不替代后端结算规则。
- 实际保底进度、门槛、奖励与范围以服务端配置和返回为准。

### 12.6 标记验收

- 焦点区标记用于命中判断范围识别。
- 保底奖励标记用于触发保底后的奖励范围识别。
- 两类标记指向同一批卡牌但语义不同，不可强行合并。
- 未经确认不得只展示其中一个标记替代另一个。
- 前端不得根据卡牌位置、稀有度、图片或名称自行判断标记范围。
- 标记展示以服务端配置为准。

### 12.7 范围控制验收

- 不在左侧展示会员权益加成值。
- 不在左侧会员等级新增 info 或点击交互。
- 不将右侧背包改造成资产管理区。
- 不在半展开态展示筛选、列表、底部固定汇总、溢出提示。
- 不在右侧筛选区展示清除入口。
- 不在右侧顶部展示 `[0/3]`、`3/300` 等未确认计数。
- 不新增未确认 loading、分页、权限、异常状态。
- 不重定义资源消耗优先级、扣减逻辑、能量计算规则。
- 不因溢出提示新增独立模块、弹窗、领取按钮、结算前确认流程。
- 不将溢出提示放入中间区域、左侧区域或本期保底卡片。
- 不新增累计返赠、任务、领取按钮、活动中心入口等未确认功能。
- 模块前台标题统一为 `本期保底`，不使用 `累计返赠`。
- 不新增保底奖励领取按钮、二次开箱流程、保底奖励详情页。

## 13. 不覆盖范围

- 不补充资源扣减、库存、订单、支付、发货、资产、权益到账、钱包系统规则。
- 不把示例值写成固定系统规则。
- 不把后续补充边界写成已确认事实。
- 不定义后端奖励随机算法与保底配置后台流程。
- 不定义发放失败、异常补偿等后端处理规则。
- 不定义批量开启逐次动画过程。
- 不扩展顶部导航、底部掉落信息 Tab 等既有模块内部规则。
