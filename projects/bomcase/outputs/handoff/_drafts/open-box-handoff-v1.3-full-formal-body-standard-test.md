# Open Box 研发交接文档

## 1. 文档版本说明

本版本为 Open Box 当前页面口径的 Full Mode 研发交接测试稿，面向研发实现、联调、UI 承接和测试验收使用。

本版本不新增产品规则，不定义接口字段、表结构、服务拆分或后端技术方案。页面最终展示、开启满足度、保底进度、保底结果、批量开启结果与奖励发放结果，均以服务端配置与接口返回为准。

页面目标是稳定三栏职责：左侧资源概览、中间主判断与主操作、右侧卡牌填充选择器；同时在页面左下方展示「本期保底」运营活动卡牌，并在下方饰品区域保留焦点区标记与保底奖励标记的识别关系。

## 4. 页面目标

Open Box 页面用于承接军需开启主流程，当前确认范围覆盖：

1. 左侧「资源概览」区域：展示用户当前与开箱相关的账号资源状态。
2. 中间「猎户军需」核心操作区域：展示军需箱主视觉、开启次数选择、开启满足度、资源构成与主操作按钮。
3. 右侧「卡牌背包」区域：作为当前开箱流程中的卡牌填充选择器，用于支持卡牌填充能量操作。
4. 页面左下方「本期保底」运营活动卡牌：展示当前军需箱的保底状态、焦点区卡牌范围、累计进度与结果状态。
5. 下方饰品区域中的焦点区标记与保底奖励标记：仅说明其与本期保底卡片的关联，不展开饰品区域其它已有规则。

本页面不改变资源扣减优先级、能量计算规则、库存、资产到账、奖励随机算法或后端保底配置后台流程。

右侧卡牌背包有两种展示态：

| 展示态编号 | 展示态 | 定义 | 结构要求 |
|---|---|---|---|
| RS-1 | 半展开态 | 当前状态摘要 | 只展示标题、摘要信息、状态说明和入口文案；不展示筛选、卡牌列表、底部固定汇总和溢出提示 |
