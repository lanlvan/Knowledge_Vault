## 4. 页面目标

Toy Collection 页面用于展示用户潮玩图鉴收藏信息，包括全站潮玩收藏进度、全部卡牌范围内的 SSR 顶级单卡统计、分类下 SSR 顶级单卡数量、系列与卡组信息、卡组内单卡收藏状态、用户已解锁 / 未解锁的单卡展示，以及隐藏款与全站唯一单卡展示。

页面支持用户按分类、系列、单卡拥有状态进行筛选查看。

关键概念定义：

| 概念 | 定义 |
| --- | --- |
| SSR / 顶级单卡 | 本页中“顶级单卡”等于 SSR。 |
| 全站收藏完成度 | 基于全站潮玩数量、全站系列数量和用户已收集数量展示的收藏进度，整数百分比按向下取整展示。 |
| 已解锁 | 用户曾经获得过该卡牌；用户曾经拥有卡牌但后续已兑换或消耗后，仍归类为“已解锁”。 |
| 未解锁 | 用户尚未获得过该卡牌。 |
| 拥有状态筛选 | 筛选对象为单卡解锁状态，不是卡组状态。 |
| 隐藏款 | 单卡展示属性，不是单卡状态；未获得时表现为“隐藏展示 + 未解锁状态”。 |
| 全站唯一 SSR | 展示“仅此一张”标签的 SSR 单卡。 |
| 集齐兑换 | 卡组内占位区域，当前为未开放状态，不支持点击。 |

### 5.3 分类 Tab 与筛选区

位置：顶部统计区下方。

包含分类 Tab、系列筛选、拥有状态筛选。

分类 Tab 包含：

| 分类 |
| --- |
| 枪械 |
| 手套 |
| 刀具 |

分类 Tab 规则：

- 默认选中第一个 Tab。
- 用户切换 Tab 后，对下方卡组内容进行筛选。
- 分类数量固定展示该分类下的 SSR 顶级单卡数量。
- 当前统计对象为 SSR。
- 分类数量不受系列筛选、拥有状态筛选影响。

系列筛选包含：

| 内容 | 说明 |
| --- | --- |
| 当前系列名称 | 展示当前选中系列。 |
| 下拉箭头 | 表示当前为下拉单选。 |

系列筛选规则：

- 当前为下拉单选。
- 最多支持 9 个系列。
- 系列由管理端配置。
- 系列支持后台排序配置。
- 分类与系列筛选联动；分类切换后，系列筛选列表随分类变化。
- 每张卡牌归属一个分类（枪 / 手套 / 道具）。
- 不同卡牌可组合成卡组。
- 多个卡组可归属同一系列。

拥有状态筛选包含：

| 内容 | 说明 |
| --- | --- |
| 当前状态名称 | 展示当前选中的单卡解锁状态筛选。 |
| 下拉箭头 | 表示可展开选择状态。 |

拥有状态筛选规则：

- 筛选对象为单卡解锁状态，不是卡组状态。
- 当前支持“全部”“已解锁”“未解锁”。
- 解锁状态筛选不与分类 / 系列筛选项联动，即不会改变分类或系列筛选列表。
- 解锁状态筛选仍作为筛选条件参与下方卡牌 / 卡组列表过滤。
- 分类、系列、解锁状态筛选同时存在时，下方列表按多条件 AND 过滤。

## 7. 核心页面口径

### 7.1 全站统计与筛选分层

- 顶部收藏进度按全站数据展示。
- 收藏完成度整数百分比按向下取整展示。
- 顶部 SSR 统计按全部卡牌范围计算。
- “顶级单卡”等于 SSR。
- “全站共 XX 张顶级单卡”为全部卡牌范围内的 SSR 顶级单卡累计数量。
- “你已拥有 X / XX”为当前用户在全部卡牌范围内已拥有的 SSR 顶级单卡数量。
- 顶部 SSR 统计不随分类 Tab、系列筛选、拥有状态筛选变化。

### 7.2 分类、系列、状态筛选

- 分类 Tab 默认选中第一个分类。
- 当前截图默认选中“枪械”。
- 分类数量固定展示该分类下的 SSR 顶级单卡数量。
- 分类数量不受系列筛选、拥有状态筛选影响。
- 分类与系列筛选联动，分类切换后系列筛选列表随分类变化。
- 系列筛选当前为下拉单选。
- 系列最多支持 9 个。
- 系列由管理端配置，并支持后台排序配置。
- 单卡状态筛选对象为单卡解锁状态，不是卡组状态。
- 单卡状态筛选支持“全部”“已解锁”“未解锁”。
- 解锁状态筛选不改变分类或系列筛选列表。
- 解锁状态筛选仍作为筛选条件参与下方卡牌 / 卡组列表过滤。
- 分类、系列、解锁状态筛选同时存在时，下方列表按多条件 AND 过滤。

### 7.3 单卡状态与展示属性

- 单卡状态仅包含“已解锁 / 未解锁”。
- “已解锁”表示用户曾经获得过该卡牌。
- 用户曾经拥有卡牌但后续已兑换或消耗后，仍归类为“已解锁”。
- “拥有”不作为单卡状态。
- “隐藏”不作为单卡状态，而是展示属性。
- 隐藏款未获得时表现为“隐藏展示 + 未解锁状态”。
- 隐藏款获取后与普通卡牌展示逻辑一致。
- 隐藏款保留“隐藏款”标签。
- 全站唯一 SSR 展示“仅此一张”标签。

### 7.4 卡组与获取渠道边界

- 一个系列下可存在多个卡组。
- 每张卡牌归属一个分类（枪 / 手套 / 道具）。
- 不同卡牌可组合成卡组。
- 多个卡组可归属同一系列。
- 卡牌来源于不同盲盒卡包。
- 不同卡包掉落不同卡牌。
- 获取渠道不同，但不影响单卡交互逻辑。
- 示例值用于表达字段形态，不直接固化为完整系统规则。

### 7.5 集齐兑换边界

- 集齐兑换区域当前为未开放状态。
- 固定展示“集齐兑换：敬请期待”和“更多兑换内容后续开放”。
- 当前不支持点击。
- 不新增兑换功能逻辑。

| TC-SCOPE-004 | 查看集齐兑换区域 | 不新增兑换功能逻辑 | 当前仅未开放占位。 |
| TC-SCOPE-005 | 查看或点击隐藏款 | 不新增隐藏款额外交互逻辑 | 隐藏款与普通单卡点击逻辑一致。 |

## 13. 不覆盖范围

- 不扩展完整资产、交易、兑换、养成、背包系统规则。
- 不新增 loading / error / empty / reset 规则。
- 不把截图示意或示例值写成固定系统规则。
- 不补充导航消息、库存、弹窗等已有能力内部规则。
- 不扩写未确认异常恢复流程。
- 不新增兑换功能逻辑。
- 不新增隐藏款额外交互逻辑。
