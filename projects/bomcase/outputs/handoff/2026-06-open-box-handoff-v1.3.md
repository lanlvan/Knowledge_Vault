## 一、页面目标

> 本模块说明：页面目标定义开箱页三栏职责，核心是左侧总览、中间判断、右侧选卡，不互相替代。

本页面为开箱页局部优化交接文档，当前确认范围覆盖：

1. 左侧「资源概览」区域：展示用户当前与开箱相关的账号资源状态。
    
2. 中间「猎户军需」核心操作区域：展示军需箱主视觉、开启次数选择、开启满足度、资源构成、保底进度与主操作按钮。
    
3. 右侧「卡牌背包」区域：作为当前开箱流程中的卡牌填充选择器，用于支持卡牌填充能量操作。
    
4. 下方饰品区域中的「保底范围饰品标记」：仅说明其与保底进度模块的关联，不展开饰品区域其它已有规则。
    

本页面用于用户开启军需。开启条件可由赠品次数、账号已有能量、卡牌填充能量及其他实际参与资源共同满足。

> 本文档为最终规则口径。所有 UI 示意图仅用于辅助说明，不作为像素级规范；如示意图与本文档存在差异，以本文档为准。

本次优化重点是：

- 明确左侧资源概览、中间主操作、右侧卡牌填充选择器的职责分离。
    
- 强化中间区域作为当前页面主判断区和主操作区。
    
- 在中间主操作区补充保底进度模块，展示当前军需箱的保底进度与规则说明。
    
- 优化右侧卡牌背包在开箱流程中的半展开摘要态与全展开操作态。
    
- 明确卡牌填充、喜欢卡保护、资源构成 icon、保底进度与按钮推荐态之间的边界。
    
- 在右侧卡牌背包全展开态底部固定汇总区，轻量提示本次填充能量溢出后的余额沉淀结果。

本次 v1.2 新增的“能量溢出轻提示”属于前台展示口径补充。

- 产品只确认页面展示口径、状态边界和验收口径。

### 2. 左侧资源概览区域

|项目|说明|
|---|---|
|区域名称|资源概览|
|区域位置|页面左侧常驻面板|
|包含元素|区域标题、副说明、资源概览 info、赠品开箱次数、能量余额、能量余额 info、即将过期能量提示、机械扳手、卡牌背包入口、会员等级|
|已确认说明|左侧区域只做账号资源概览，不承担本次是否可开启判断，也不承担卡牌选择操作|

左侧资源概览展示账号当前资源总览，可展示数量为 0 的资源。

左侧信息层级分为三层：

1. **L1 主资源**：赠品开箱次数，最高优先级展示。
    
2. **L2 次主资源**：账号已有能量余额，中等优先级展示。
    
3. **L3 辅助资源 / 入口 / 身份信息**：机械扳手、卡牌背包、会员等级。
    

会员等级只展示身份和等级进度，不展示会员权益加成数值，不展示会员等级 info，不新增会员等级点击交互。

能量即将过期提示保留在「能量余额」卡片内展示，不额外增加左侧底部状态提示区。

### 3. 中间猎户军需核心区域

> 本模块说明：中间区域负责本次开启判断和主操作，核心是展示后端确认的满足度，不自行推导资源结果。

|项目|说明|
|---|---|
|区域名称|猎户军需核心操作区|
|区域位置|页面中间主视觉与主操作区|
|包含元素|页面标题、军需箱主视觉、X1 / X5 开启次数选择、开启满足度、本次资源构成 icon 区、保底进度模块、CTA 按钮区|
|已确认说明|中间区域是当前页面的主判断区和主操作区|

中间区域负责表达：

- 当前选择开启次数。
    
- 当前开启满足度。
    
- 本次实际参与资源。
    
- 当前军需箱的保底进度。
    
- 当前推荐操作。
    

中间区域不做账号资源总览，不展示完整卡牌列表，不展示本次卡牌填充溢出说明。

中间「本次资源构成 icon 区」仅展示本次实际参与本次开启判断的资源；未参与资源、数量为 0 的资源不在中间 icon 区展示。

中间区域展示的开启满足度与资源构成 icon 以后端返回的本次计入结果为准。前端不通过左侧账号资源与右侧已选卡牌自行推导中间展示结果。

### 4. 保底进度模块

> 本模块说明：保底进度模块展示当前进度和奖励，核心是不参与开启满足度，也不影响按钮状态。

|项目|说明|
|---|---|
|区域名称|保底进度模块|
|区域位置|中间核心区内，本次资源构成 icon 区下方、CTA 按钮区上方|
|包含元素|小军需箱 icon、模块标题、主文案、info icon、分段进度条、规则 info|
|展示条件|仅配置保底玩法的指定军需箱展示|
|未配置处理|未配置保底玩法的军需箱不展示该模块，不占位；CTA 按钮区按正常布局自然承接上方内容|
|已确认说明|常态主模块展示保底进度、剩余积累次数、可获得保底奖励；保底范围与计算规则放入 info|

保底进度模块为中间核心区的辅助机制展示模块，仅展示当前军需箱的保底进度与规则说明。

保底进度模块不参与开启满足度计算，不影响一键填充逻辑，不影响开启军需按钮状态，不进入本次资源构成 icon 区。

保底进度、目标进度、剩余积累次数、保底奖励名称、保底范围饰品标记均以后端配置与接口返回为准。

---

### 5. 右侧卡牌背包区域

> 本模块说明：右侧卡牌背包用于本次选卡填充，核心是临时选择器，不是完整资产管理区；半展开态只展示摘要，溢出提示仅在全展开态底部展示。

|项目|说明|
|---|---|
|区域名称|卡牌背包|
|区域位置|页面右侧面板|
|包含元素|半展开态摘要面板、全展开态卡牌选择面板、右侧顶部收起 / 展开切换 icon、卡牌筛选、2 列卡牌列表、卡牌选中态、喜欢卡标识、一键填充跳过喜欢卡轻提示、底部固定汇总区|
|已确认说明|右侧「卡牌背包」保留资产语义，但实际定位是当前开箱流程中的卡牌填充选择器，不是资产管理区|

右侧面板分为两种展示态：

1. **半展开态**：当前状态摘要。
    
2. **全展开态**：操作详情。
    

半展开与全展开应使用同一组件骨架，区别只在内容展示，不应表现为两个不同组件。

右侧顶部展示收起 / 展开切换 icon，不展示 `[0/3]`、`3/300` 等未确认口径计数。icon 方向、动效和样式以最终设计稿为准。

右侧底部固定汇总区在已选卡牌状态下，负责展示本次已选卡牌数量、填充能量、清除入口与消耗说明。当本次存在填充能量溢出时，右侧底部固定汇总区轻量补充展示溢出能量沉淀提示。

## 五、核心口径说明

### 1. 卡牌数量口径

左侧「卡牌背包 N张」与右侧半展开态「共 N 张卡牌」表示：

> 用户当前拥有的卡牌总数。

右侧全展开列表是卡牌选择器，展示数量可受筛选、可用状态影响，不等同于用户拥有卡牌总数。

喜欢卡保护不影响右侧列表展示，仅影响一键填充自动选择逻辑。

设计图中的右侧列表展示数量仅作为布局示例，不作为卡牌数量口径。

### 2. 能量口径

|位置|展示内容|口径说明|
|---|---|---|
|左侧资源概览|能量余额 18%|账号已有能量余额。|
|右侧底部汇总 / 半展开填充态|填充 +X%|本次已选卡牌带来的新增能量，不包含账号已有能量。|
|右侧底部汇总补充说明|溢出 +Y% 存入能量余额|本次已选卡牌能量超过当前开启所需补足能量时，超出部分在开启后沉淀至账号能量余额。仅存在溢出时展示。|
|中间资源构成 icon|卡牌能量 +X%|本次开箱计入满足度的卡牌能量值，具体数值以后端返回为准。|

说明：

- 前端不通过“左侧账号已有能量 + 右侧本次选卡新增能量”自行计算中间「卡牌能量」展示值。
    
- 中间「开启满足度」与资源构成均以后端返回的本次开箱判断结果为准。
    
- 右侧「填充 +X%」表示本次已选卡牌带来的新增能量，不包含账号已有能量，也不因是否溢出而改变其展示口径。
    
- 当本次已选卡牌能量超过当前开启所需补足能量时，超出部分作为溢出能量沉淀至账号能量余额。
    
- 右侧底部汇总区可轻量展示：`已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额`。
    
- “溢出 +Y%”仅在存在溢出能量时展示；不存在溢出时，仍展示原文案：`已选卡牌将在开启时消耗`。
    
- “溢出 +Y%”不进入中间资源构成 icon 区，不展示在中间开启满足度区域，不展示在左侧资源概览中。
    
- 示意图中的 `+18%`、`+68%`、`+95%` 均为状态示例，具体值以后端返回为准。
    
- 本文档仅约束展示口径，不展开已有能量计算规则、实际扣减逻辑或资源消耗优先级。

### 3. 资源参与口径

左侧资源概览展示账号资源总览；中间资源构成 icon 区只展示本次实际参与本次开启判断的资源。

|---|---|---|
|纯赠品路径|赠品次数 ×N|卡牌能量、扳手、VIP|
|纯能量路径|卡牌能量 +X%、扳手 ×N、VIP权益 +X%（如有）|赠品 ×0|
|混合路径|赠品次数 ×N、卡牌能量 +X%、扳手 ×N、VIP权益 +X%（如有）|未参与资源|
|都没有资源参与|icon 区不显示任何内容|全部资源 icon|

说明：

- 赠品 ×0 不展示。
    
- 未参与资源不展示。
    
- 不显示空占位。
    
- 当仅有 1 个资源 icon 时，使用完整资源名称文案，例如：`赠品次数 ×1`。
    
- 右侧底部的“溢出 +Y% 存入能量余额”不属于中间资源构成 icon 区内容。

### 4. 保底模块口径

保底进度模块仅作为当前军需箱的机制进度展示，不参与本次开启条件判断。

|项目|口径|
|---|---|
|是否参与开启满足度计算|不参与|
|是否影响一键填充逻辑|不影响|
|是否影响开启军需按钮状态|不影响|
|是否进入资源构成 icon 区|不进入|
|是否影响卡牌填充能量|不影响|
|是否影响资源消耗优先级|不影响|
|数据来源|以后端配置与接口返回为准|

保底范围饰品定义：

> 保底范围饰品指当前军需箱配置中被纳入保底范围、并在下方饰品区域展示保底标记 icon 的饰品。

自然命中与触发保底的区别：

|类型|说明|
|---|---|
|获得保底范围饰品|指用户在正常开启结果中获得任一保底范围饰品，本轮保底完成并进入新一轮，不额外触发保底奖励发放。|
|触发保底|指连续未获得保底范围饰品达到配置门槛后，系统按配置发放保底奖励。|

保底奖励发放边界：

- 保底奖励在后端结算完成时即发放。
    
- 保底奖励发放后自动开启新一轮保底进度。
    
- 保底奖励发放不依赖前端结果页播放状态。
    
- 结果页仅作为告知展示，不作为奖励发放的触发节点。
    

批量开启说明：

- X5 等批量开启的保底最终展示以后端结算结果为准。
    
- 前端不自行推导跨轮保底结果。
    
- 批量开启中如同时发生自然命中保底范围饰品、触发保底奖励、新一轮进度变化等结果，前端不自行判断展示优先级，最终结果态文案与新一轮进度以后端返回为准。
    

达到保底门槛后的展示说明：

- 达到保底门槛后的展示以最终结算结果为准。
    
- 不要求前台停留展示 `Y/Y` 过程态。
    
- 触发保底、获得保底范围饰品等结果态用于本次开启后的结果反馈或刷新后的短期展示。
    
- 页面最终稳定态以结算后的新一轮保底进度为准。
    
- 结果态具体展示时长、消失方式、是否有过渡动效待确认。

### 5. X1 / X5 与赠品部分抵扣规则

X1 / X5 为开启次数选择。

当选择 X5 且赠品次数不足 5 次时，赠品次数可计入本次开启满足度，剩余部分可通过卡牌能量等资源补足。

切换 X1 / X5 后：

- 已选卡牌清空。
    
- 开启满足度重新计算。
    
- 中间资源构成 icon 重新计算。
    
- 右侧状态重新判断。
    
- 右侧底部固定汇总区清空已选态与能量溢出提示。
    
- 保底进度模块不因 X1 / X5 切换直接变化；保底进度仅在开启结算后按后端结果刷新。

### 6. 已有功能边界

资源消耗优先级、实际扣减逻辑、能量计算规则、开启条件不足时的已有提示逻辑，均沿用现有开箱规则，本次优化不调整。

保底进度、目标进度、剩余积累次数、保底奖励名称、保底范围饰品标记、批量开启最终保底结果，均以后端配置与接口返回为准。

本次填充能量溢出值用于前台轻提示展示，具体计算和返回方式以后端或既有开启判断结果为准。本文档不重新定义能量计算、扣减顺序、余额沉淀规则或有效期规则。

本文档只描述本次页面展示、状态、交互和视觉规则，不重新定义已有资源扣减规则。

- 本次能量溢出提示不进入本次资源构成 icon 区。
    

#### 2.5 保底进度模块

默认状态：

```text
保底进度
当前 X/Y，再积累 N 次可获「保底奖励名称」 info
```

下方展示分段进度条。

分段进度条规则：

- 总段数与目标进度 Y 一致。
    
- 当前进度 X 决定点亮段数。
    
- 示例：`3/7` 表示总段数 7 段，点亮 3 段。
    
- 目标进度 Y 以后端配置返回为准。
    

展示规则：

|状态|触发条件|前台展示文案|展示说明|
|---|---|---|---|
|未配置保底|当前军需箱未配置保底玩法|不展示|模块不展示、不占位；CTA 按钮区按正常布局自然承接上方内容。|
|累计中 / 新一轮稳定态|当前军需箱配置保底玩法，且处于正常累计展示状态|当前 X/Y，再积累 N 次可获「保底奖励名称」|包括初始累计中、自然命中保底范围饰品后进入新一轮、触发保底后进入新一轮等稳定展示场景。|
|未获得保底范围饰品后|本次开启未获得任一保底范围饰品|当前 X/Y，再积累 N 次可获「保底奖励名称」|保底进度 +1 后刷新模块展示，属于稳定态刷新。|
|获得保底范围饰品|本次开启获得任一保底范围饰品|已获得保底范围饰品，本轮保底完成，新一轮已开始|结果态用于本次开启后的反馈或短期展示；页面最终稳定态回到「累计中 / 新一轮稳定态」。|
|触发保底|连续未获得保底范围饰品达到保底门槛|保底已触发，已获得「保底奖励名称」|保底奖励由后端结算完成时发放；结果态用于本次开启后的反馈或短期展示；页面最终稳定态回到「累计中 / 新一轮稳定态」。|
|批量开启跨门槛|X5 等批量开启过程中触发保底|保底已触发，已获得「保底奖励名称」，新一轮进度 X/Y|最终展示以后端结算结果为准，前端不自行判断展示优先级。|

说明：

- 前台不使用“重置 / 清零”作为主表达。
    
- 常态主文案不使用“未中”作为主表达，规则说明放入 info。
    
- 达到保底门槛后，不要求前台停留展示 `Y/Y` 过程态。
    
- 自然命中保底范围饰品或触发保底奖励后，页面最终稳定态均回到「累计中 / 新一轮稳定态」，展示后端返回的新一轮进度。
    
- 保底进度模块权重低于开启满足度和 CTA，不抢主操作权重。
    
- 批量开启中如同时发生自然命中保底范围饰品、触发保底奖励、新一轮进度变化等结果，前端不自行判断展示优先级，最终结果态文案与新一轮进度以后端返回为准。
    
- 结果态具体展示时长、消失方式、是否有过渡动效待确认。

### 3. 右侧卡牌背包半展开态

- 半展开态不展示“溢出 +Y% 存入能量余额”，避免摘要态信息过重。

- 副标题「选卡填充能量」
    
- 筛选项
    
- 卡牌列表
    
- 底部固定汇总
    
- 能量溢出轻提示
    

半展开展示内容只包括：

- 卡牌背包标题
    
- 当前摘要信息
    
- 当前状态说明
    
- 入口文案

### 4. 右侧卡牌背包全展开态

    - 全部等级

- 轻提示展示在右侧全展开态卡牌列表区域底部附近、底部固定汇总区上方；具体位置以最终设计稿为准。
    

#### 4.6 底部固定汇总区

> 本模块说明：底部汇总展示已选卡牌和填充能量，核心是溢出提示仅在全展开态展示，不进入半展开态和中间区域。

底部汇总区固定在右侧面板底部，不随卡牌列表滚动。

|场景|底部文案|
|---|---|
|需要补能量，未选卡|已选 0 张｜选择卡牌后填充能量|
|赠品已满足，未选卡|赠品次数已满足，暂无需填充|
|能量已满足，未选卡|开启能量已满足，暂无需填充|
|已有选择|已选 N 张｜填充 +X%｜清除已选|
|已有选择 · 无溢出补充说明|已选卡牌将在开启时消耗|
|已有选择 · 有溢出补充说明|已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额|

说明：

- “填充 +X%”表示本次已选卡牌带来的新增能量，不包含左侧账号已有能量。
    
- “清除已选”仅在已选卡牌数量大于 0 时展示。
    
- “已选卡牌将在开启时消耗”仅在已选卡牌数量大于 0 且不存在能量溢出时展示。
    
- 当本次已选卡牌存在能量溢出时，底部补充说明展示“已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额”。
    
- “溢出 +Y%”表示本次开启消耗后沉淀至账号能量余额的部分。
    
- “溢出 +Y% 存入能量余额”为轻提示文案，不使用强按钮、弹窗或独立模块表达。
    
- “溢出 +Y%”不在半展开态默认展示，避免半展开摘要过重。
    
- “清除已选”为弱文字按钮，不抢主操作权重。
    
- 顶部筛选区不展示“清除”。

### 3. 保底进度变化

|项目|说明|
|---|---|
|触发对象|用户开启当前军需箱|
|触发条件|当前军需箱配置保底玩法|

## 九、验收口径

### 1. 页面结构验收

- 页面应包含左侧资源概览、中间核心操作区、右侧卡牌背包三个主要区域。
    
- 左侧资源概览区域应包含：赠品开箱次数、能量余额、机械扳手、卡牌背包入口、会员等级。
    
- 中间核心区域应包含：军需箱主视觉、X1 / X5、开启满足度、资源构成 icon、保底进度模块、一键填充、开启军需。
    
- 保底进度模块应位于本次资源构成 icon 区下方、CTA 按钮区上方。
    
- 仅配置保底玩法的指定军需箱展示保底进度模块。
    
- 未配置保底玩法时，不展示保底进度模块，不保留空位，CTA 按钮区按正常布局自然承接上方内容。
    
- 右侧卡牌背包区域应支持半展开态和全展开态。
    
- 半展开和全展开应使用同一组件骨架，不应表现为两个不同组件。
    
- 全展开态应使用 2 列卡牌布局。
    
- 全展开态仅卡牌列表区域可滚动；顶部标题 / 筛选区与底部固定汇总区不随列表滚动。
    
- 能量溢出提示仅归属右侧卡牌背包全展开态底部固定汇总区，不新增独立模块。
    
- 下方饰品区域中，属于保底范围饰品的饰品应展示保底标记 icon。

### 2. 字段展示验收

- 左侧赠品开箱次数展示为 L1 主资源，可展示 0。
    
- 左侧能量余额展示为 L2 次主资源，仅展示账号已有能量。
    
- 左侧即将过期提示保留在能量余额卡片内。
    
- 左侧机械扳手、卡牌背包、会员等级展示为 L3 信息。
    
- 左侧会员等级不展示 info，不新增点击交互。
    
- 左侧不展示会员权益加成数值。
    
- 左侧卡牌背包展示为“卡牌背包 N张 ›”形式。
    
- 左侧“N张”表示用户当前拥有卡牌总数。
    
- 右侧半展开“共 N 张卡牌”表示用户当前拥有卡牌总数。
    
- 右侧全展开列表展示数量可受筛选、可用状态影响，不等同于用户拥有卡牌总数。
    
- 喜欢卡保护不影响右侧列表展示，仅影响一键填充自动选择逻辑。
    
- 中间开启满足度应展示百分比大字和进度条。
    
- 中间资源构成 icon 只展示本次实际参与资源。
    
- 中间「卡牌能量 +X%」以后端返回的本次计入满足度的卡牌能量值为准，前端不自行根据左侧与右侧相加推导。
    
- 中间未参与资源不展示，不显示 ×0。
    
- 保底模块应展示模块标题“保底进度”。
    
- 保底模块应展示常态主文案：`当前 X/Y，再积累 N 次可获「保底奖励名称」`。
    
- 保底奖励名称按当前军需箱配置动态展示。
    
- 保底奖励名称支持 hover，hover 内容以原有内容为主。
    
- 保底模块应展示 info icon。
    
- 保底模块应展示分段进度条；总段数与目标进度 Y 一致，当前进度 X 决定点亮段数。
    
- 保底范围规则不在常态主文案中展示，放入 info。
    
- 保底结果态文案应以后端返回为准展示，不由前端自行推导。
    
- 右侧顶部展示收起 / 展开切换 icon，不展示 `[0/3]`、`3/300` 等未确认口径计数。
    
- 右侧半展开填充态必须展示“已选 N 张，填充 +X%”。
    
- 右侧半展开态不展示“溢出 +Y% 存入能量余额”。
    
- 右侧全展开已选态必须展示“已选 N 张｜填充 +X%｜清除已选”。
    
- 右侧全展开已有选择时，底部固定汇总区应展示“已选卡牌将在开启时消耗”。
    
- 当本次已选卡牌存在能量溢出时，右侧全展开底部固定汇总区应展示“已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额”。
    
- 当不存在能量溢出时，不展示“溢出 +Y% 存入能量余额”。
    
- “溢出 +Y%”表示本次开启消耗后沉淀至账号能量余额的部分。
    
- “溢出 +Y%”不应展示在中间开启满足度、资源构成 icon 区或左侧资源概览中。
    
- 右侧“填充 +X%”表示本次已选卡牌带来的新增能量，不包含左侧账号已有能量。
    
- 卡牌能量值未选为蓝色，已选为金色。
    
- 保底范围饰品应展示保底标记 icon。

### 3. 状态验收

- 有卡牌且赠品次数满足时，右侧半展开展示：`赠品次数已满足，暂无需填充`。
    
- 有卡牌且开启能量已满时，右侧半展开展示：`开启能量已满足，暂无需填充`。
    
- 有卡牌且已有填充时，右侧半展开展示：`已选 N 张，填充 +X%`。
    
- 无可用卡牌时，右侧半展开展示：`暂无可用卡牌` 和 `前往获取`。
    
- 无可用卡牌时，右侧半展开应展示空状态插图。
    
- 一键填充跳过喜欢卡时，右侧展示轻提示：`喜欢卡已保护，未自动填充`。
    
- 非一键填充跳过喜欢卡场景，不展示该提示。
    
- 喜欢卡提示不是默认常驻说明，示例图为一键填充后的提示态。
    
- 可填充但未满足时，一键填充为推荐态。
    
- 已满足开启条件时，开启军需为推荐态。
    
- 都没有资源时，中间资源构成 icon 区不展示任何内容。
    
- 都没有资源时，中间进度条下方不展示文案。
    
- 无可填充卡牌时，一键填充按钮为灰度无特效，不作为当前推荐操作。
    
- 当选择 X5 且赠品次数不足 5 次时，赠品次数可计入本次开启满足度，剩余部分可通过卡牌能量等资源补足。
    
- X1 / X5 切换后，开启满足度和资源构成 icon 应重新计算。
    
- X1 / X5 切换后，已选卡牌清空，右侧底部固定汇总区不再展示已选态、消耗说明或能量溢出提示。
    
- 已选卡牌且不存在能量溢出时，右侧全展开态底部展示“已选卡牌将在开启时消耗”。
    
- 已选卡牌且存在能量溢出时，右侧全展开态底部展示“已选卡牌将在开启时消耗，溢出 +Y% 存入能量余额”。
    
- 清除已选后，右侧全展开态底部不展示“已选卡牌将在开启时消耗”或“溢出 +Y% 存入能量余额”。
    
- 未配置保底玩法时，不展示保底进度模块。
    
- 配置保底玩法且处于「累计中 / 新一轮稳定态」时，保底模块展示当前进度、剩余积累次数、可获得保底奖励。
    
- 本次开启未获得任一保底范围饰品时，保底进度 +1，并刷新为对应进度文案。
    
- 本次开启获得任一保底范围饰品时，本轮保底完成并开启新一轮。
    
- 连续未获得保底范围饰品达到保底门槛时，由后端在结算完成后自动发放对应保底奖励。
    
- 保底奖励发放不依赖前端结果页播放状态。
    
- 批量开启时，保底最终展示以后端返回结果为准。
    
- 批量开启中如同时发生自然命中保底范围饰品、触发保底奖励、新一轮进度变化等结果，前端不自行判断展示优先级。
    
- 达到保底门槛后，不要求前台停留展示 `Y/Y` 过程态。
    
- 触发保底、获得保底范围饰品等结果态用于本次开启后的结果反馈或短期展示；页面最终稳定态以结算后的新一轮保底进度为准。
    
- 保底结果态展示时长、消失方式、过渡动效详见待补充项，本次暂不纳入验收。

- 清除已选后，右侧底部不再展示能量溢出提示。
    
- 开启条件满足时，点击开启军需执行开启。
    
- 开启条件不满足时，点击开启军需触发已有功能，本次不展开。
    
- 离开页面后，已选卡牌不持久化。
    
- 离开页面后，能量溢出提示不持久化。
    
- 即将过期能量提示 hover 后展示说明弹层，该弹层为后续迭代能力。
    
- 保底 info icon hover 后展示保底规则说明。
    
- 保底奖励名称支持 hover，hover 内容以原有内容为主。
    
- 保底范围饰品标记 icon 不新增点击或 hover 交互。
    

---

### 5. 保底 info 验收

保底 info 至少包含以下说明：

```text
本军需箱设有保底范围饰品，可在下方饰品区域通过保底标记识别
未获得任一保底范围饰品时，保底进度 +1
获得任一保底范围饰品后，本轮保底完成并开启新一轮
触发保底后自动发放对应奖励
```

说明：

- info 内容用于前台规则说明，不替代后端实际结算规则。
    
- 实际保底进度、门槛、奖励、范围以后端配置与接口返回为准。

### 6. 范围控制验收

- 不在左侧展示会员权益加成数值。
    
- 不在左侧会员等级上新增 info 或点击交互。
    
- 不把右侧卡牌背包做成资产管理区。
    
- 不在半展开态展示筛选、卡牌列表和底部固定汇总。
    
- 不在半展开态展示“溢出 +Y% 存入能量余额”。
    
- 不在右侧筛选区展示“清除”。
    
- 不在右侧顶部展示未确认口径的 `[0/3]`、`3/300` 等数量。
    
- 不在无确认情况下新增 loading、分页、权限、异常状态。
    
- 不在本次文档中重新定义资源消耗优先级、实际扣减逻辑、能量计算规则。
    
- 不因能量溢出提示新增独立能量沉淀模块、弹窗、领取按钮或结算前确认流程。
    
- 不把能量溢出提示放入中间开启满足度、资源构成 icon 区、保底进度模块或左侧资源概览。
    
- 不新增累计返赠、任务、领取按钮、活动中心入口等未确认功能。
    
- 历史讨论中曾出现“累计返赠”命名，本次统一确认模块名称为“保底进度”，不使用“累计返赠”。
    
- 不展开资源概览、卡牌背包、底部掉落 Tab、CTA 按钮等已有功能规则。
    
- 不额外设计保底 loading、异常态、无权限态、按钮置灰态、未确认空状态。
    
- 保底常态主文案不使用“未中”作为主表达，规则说明放入 info。
    
- 保底模块不参与开启满足度计算，不影响一键填充、开启军需按钮状态。
    
- 能量溢出轻提示不影响一键填充、开启军需按钮状态。
    
- 对已有功能但本次未说明的内容，不展开流程和规则。
    
- UI 示意图仅作为辅助理解，不作为最终规则口径；最终以本文档为准。
