# Open Box Result Active Source

## 1. Source Scope
<!-- section_layer: F -->
<!-- layer_note: source 身份、事实范围、后端结算结果展示边界和不替代完整结算系统规则的约束按 F 读取。 -->

本文件是 Open Box Result 页面当前 active source（`source_type: handoff_document`），用于维护页面事实、字段、状态、交互与验收口径。

页面展示以后端结算结果为准，本文件不替代完整结算系统规则。

## 2. Page Goal
<!-- section_layer: F+E -->
<!-- layer_note: 结果页职责、后端结算结果承接、本次获得饰品为核心、保底奖励饰品并入饰品列表、水晶沉淀提示条件和已有功能不展开边界为 F；顶部保底反馈、本次获得饰品区、水晶能量沉淀提示等反馈名称和提示表达为 E，读取时必须局部拆分。 -->

本页面为军需箱开箱完成后的结果反馈页，用于在后端完成本次开启结算后，展示用户本次实际获得的全部饰品，并补充呈现本次结算后的最终保底状态与水晶能量沉淀提示。

本次结果页保留两类反馈：

1. 顶部保底反馈：按后端最终结算结果，提示本次最高优先级保底状态；
2. 水晶能量沉淀提示：当本次存在多余能量转为水晶时，低权重提示用户后续可继续使用。

结果页核心内容为本次获得饰品区。该区域展示本次开启结算后用户实际获得的全部饰品，包括正常开箱掉落饰品，以及触发保底后由系统额外发放的保底奖励饰品。

触发保底后不再发放箱体奖励，不要求用户二次开启；后端按当前箱子的保底配置，直接发放焦点区范围内 1 个奖励饰品。保底奖励饰品不单独展示为独立奖励区域，不固定置顶、置尾或独立抽出展示，最终展示顺序以后端返回的结算顺序为准。

饰品卡片、返回按钮、再来 5 次按钮、关闭按钮等为已有功能，本次不展开；保底计算、资源扣减、奖励发放与批量开启结算规则不在本文档重新定义，结果页仅展示后端最终结算结果。
