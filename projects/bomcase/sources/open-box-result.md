# Open Box Result Active Source

## 1. Source Scope
<!-- section_layer: F -->
<!-- layer_note: source 身份、事实范围、后端结算结果展示边界和不替代完整结算系统规则的约束按 F 读取。 -->

本文件是 Open Box Result 页面当前 active source（`source_type: handoff_document`），用于维护页面事实、字段、状态、交互与验收口径。

页面展示以后端结算结果为准，本文件不替代完整结算系统规则。

## 2. Page Goal
<!-- section_layer: F+E -->
<!-- layer_note: 结果页职责、后端结算结果承接、正常掉落为核心、保底奖励额外展示、水晶沉淀提示条件和已有功能不展开边界为 F；顶部保底反馈、保底奖励区、水晶能量沉淀提示等反馈名称和提示表达为 E，读取时必须局部拆分。 -->

本页面为军需箱开箱完成后的结果反馈页，用于在后端完成本次开启结算后，展示用户本次正常掉落饰品，并补充呈现本次结算产生的保底状态、额外保底奖励与水晶能量沉淀提示。

本次结果页新增内容聚焦三类反馈：

1. 顶部保底反馈：提示本次是否触发保底、完成本轮保底或进入临近保底节点；
2. 保底奖励区：当本次触发保底并额外发放奖励军需箱时，单独展示奖励信息；
3. 水晶能量沉淀提示：当本次存在多余能量转为水晶时，低权重提示用户后续可继续使用。

结果页核心内容仍为正常掉落饰品展示。保底奖励不混入正常掉落饰品卡片，不作为第 6 张掉落卡片展示；水晶提示不承接完整资源概览，仅作为本次结算后的条件提示。

正常掉落饰品卡片、返回按钮、再来 5 次按钮、关闭按钮等为已有功能，本次不展开；保底计算、资源扣减、奖励发放与批量开启结算规则不在本文档重新定义，结果页仅展示后端最终结算结果。
